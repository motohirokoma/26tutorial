基本コマンドと会話の流れ

起動のしかた、スラッシュコマンド、権限モード、ショートカットキー。日常的に使う操作を一通り学びます。

所要時間: 1.5h Phase: Basics 難易度: 入門

起動方法は3つある

状況によって claude の呼び方を使い分けます。

形式コマンド例こういう時に
対話モードclaude普段使い。話しながら作業を進める
ワンショットclaude -p "ファイル一覧を要約して"シェルスクリプトに組み込む。CIで使う
セッション再開claude --continue / claude --resume前回の続きから話す
起動形式の選び方
graph TD
  A["どう起動する?"] --> Q1{"対話したい?"}
  Q1 -- はい --> Q2{"前回の続き?"}
  Q2 -- はい --> R1["claude --continue"]
  Q2 -- いいえ --> R2["claude"]
  Q1 -- いいえ --> R3["claude -p '指示'"]
対話 → 普通起動。1発結果が欲しい → -p。続き → --continue。

対話モードの流れ

セッションのライフサイクル
起動 → やり取り → 終了の3段。途中でいつでも /clear や /compact で履歴を整理できる。

対話例:

$ claude

> プロジェクトの構成を1分で説明して

[Reading project files...]
このプロジェクトは Express + TypeScript の REST API です。
ルーティングは src/routes/、コントローラは src/controllers/、
DB は Prisma + PostgreSQL を使用…

> /users エンドポイントにページネーション機能を追加して

[Editing src/routes/users.ts...]
[Adding tests...]
追加しました。?page=1&limit=20 形式でアクセスできます。

スラッシュコマンド一覧

会話中に / から始まる文を打つと、Claudeへの指示ではなく クライアント自身への命令 になります。

コマンド意味
/helpコマンド一覧表示
/clear履歴を全部消す(新規セッション化)
/compact履歴を要約圧縮(コンテキスト節約)
/model使用モデルの切り替え (Opus/Sonnet/Haiku)
/login / /logout認証の切り替え
/initカレントディレクトリにCLAUDE.mdを作る
/permissions権限モードの確認・変更
/costこのセッションの推定コスト表示
/exit / /quit終了
迷ったら

/help で全部出ます。最初は /clear /compact /exit の3つだけ覚えれば日常使いには困りません。

権限モード(ここ重要)

Claudeが「ファイルを書く」「シェルコマンドを実行する」のは、あなたの環境に対する 変更行為 です。安全のため、操作の都度確認するか、事前に許可するかを選べます。

権限モードの強さ比較
緑→赤になるほど自動化される代わりに「うっかり実行」のリスクが上がる。
モード挙動使う場面
default毎回 [y/n] で確認初学者・本番環境・新しいプロジェクト
acceptEditsファイル編集は自動承認、シェル実行は確認慣れた個人プロジェクトの作業中
bypassPermissionsすべて自動承認独立したサンドボックス環境のみ

切り替え方:

# 起動時にフラグで指定
claude --permission-mode acceptEdits

# セッション中に
> /permissions
bypassPermissions について

「全部YES」モード。本番環境では絶対に使わない。Docker内・使い捨てVMなど、壊れても困らない環境専用。「rm -rf ~」のような取り返しのつかない操作も自動実行されます。

承認プロンプトの仕組み

default モードで作業すると、こういうプロンプトが出ます。

Claude wants to Bash: npm test
[y] Yes  [n] No  [a] Always allow npm test  [d] Deny + tell why
  • y — 1回だけ許可
  • n — 拒否
  • a — このコマンド(または同類)を以後は自動許可。.claude/settings.local.json に記録される
  • d — 拒否+理由を伝える(Claudeが代替案を考える)

コンテキストとのつきあい方

コンテキストウィンドウ は有限です。長いセッションでは履歴を整理する習慣を。

セッション中のコンテキスト推移
放置すると累積。compactで段階的に圧縮 → セッション末まで持たせるのが理想。
/compact

要約圧縮

  • 過去の会話を要約に置き換える
  • 「やった内容」は覚えたまま縮む
  • 長丁場のセッションで便利
/clear

全削除

  • 履歴ごとリセット
  • 新しい話題に切り替えるとき
  • 「何を話したか」も消える

覚えておくと楽なショートカットキー

Ctrl+C
実行中のタスクを止める
Ctrl+D
セッション終了(/exit と同じ)
↑ / ↓
過去の入力を呼び出す
Esc
入力中の指示をキャンセル
Shift+Enter
改行(複数行入力)

CLIフラグ - 起動時のオプション

  • -p "指示" — 1発で結果を返してすぐ終わる
  • --continue — 前回のセッションの続きから
  • --resume — セッションを選んで再開
  • --model claude-opus-4-5 — モデル指定
  • --permission-mode acceptEdits — 権限モード指定
  • --add-dir ../shared — 別ディレクトリも参照可能に
もっと詳しく: ヘッドレスでの活用

-p モードは「人間の対話なし」で動くので、シェルスクリプトに組み込めます。例:

# 毎朝、未マージPRを要約してSlackに投げるスクリプト
claude -p "gh pr list を見て、未マージPRをタイトルだけ列挙して" \
  | curl -X POST -d @- https://hooks.slack.com/...

これで「自動レポーター」が完成します。CIに組み込むと、コードレビューの一次対応も無人化できます(Ch07-2で扱う)。

Exercise 1
入門

スラッシュコマンドを5つ実際に使ってみる

セッションを起動して、以下を順に実行してみましょう。

  1. /help — 一覧を眺める
  2. 軽く話しかけて雑談
  3. /cost — ここまでの推定費用を見る
  4. /compact — 履歴を圧縮する
  5. /exit — 終了
模範解答を見る

すべて期待通りに動けば成功。/cost の表示は、対話の長さによって数セント〜数十セント程度。月額プランで使っているなら金額表示はされず「Plan usage」になることもあります。

意外と便利なのが /compact。セッションが長くなって応答が遅くなってきたら、これを叩くだけで急に軽くなります。

Exercise 2
応用

自動承認リストを設計する

あなたが毎日叩くコマンド(npm test, git status など)で、Claudeに毎回確認されるとうるさいものを思い浮かべてください。.claude/settings.local.json の permissions.allow に書く形で設計しましょう。

  1. 自動許可してOKなコマンドを5つ選ぶ
  2. 「絶対に毎回確認したい」コマンドを3つ選ぶ
  3. JSON で permissions.allow / permissions.deny を書いてみる
模範解答を見る 
{
  "permissions": {
    "allow": [
      "Bash(npm test:*)",
      "Bash(npm run lint:*)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(ls:*)",
      "Read(*)"
    ],
    "deny": [
      "Bash(rm:*)",
      "Bash(git push --force:*)",
      "Bash(npm publish:*)"
    ]
  }
}

原則: 「読む系・テスト系」は自動許可してOK。「破壊系・公開系」は必ず確認。:* はサブコマンドや引数のワイルドカードです。Ch10 で hooks と組み合わせて、さらに細かい制御もできます。
