ようこそ&セットアップ再確認

なぜClaude Code / インストール確認 / 最初のスラッシュコマンドで土台を整える

所要時間: 30分 Phase: Basics 難易度: 入門
この章のゴール

この24時間カリキュラムを走り切ると、あなたは Claude Code 利用者の上位20%(パワーユーザー)に到達します。そのスタート地点がこの章。なぜClaude Codeなのかを腹落ちさせ、インストールと初回起動を自分の手で確認し、/help /cost /clear /compact の4つのスラッシュコマンドを触る。ここまでを30分で。読むより触るを合言葉に、章の中に「今すぐ試す」を3箇所仕込みました。

バージョン注記

本章の記述は 2026年1月時点 の Claude Code CLI の挙動に基づきます。コマンドやフラグは進化するため、違和感があれば claude --help と 公式ドキュメント を必ず確認してください。

なぜ Claude Code — 「ターミナルの中の同僚」

Claude Code は、あなたのターミナル(CLI)に住み着いて、ファイルを読み書きし、コマンドを実行し、git を回し、テストを走らせ、PRを出すところまで丸ごと手伝う agentic coding 環境です。Web の ChatGPT や Claude.ai との最大の違いは、あなたの実リポジトリと実環境に直接アクセスできること。

Web版チャット vs Claude Code
同じ「Claude」でも、CLI に入ることで手作業の往復が消え、手が止まらなくなる。

もう一つ重要なのは、Claude Code は 「何でも自動でやってくれる魔法」ではない という点です。良い頼み方(Ch02)、適切な道具選び(Ch03-04)、進捗の管理(Ch05)など、任せ方のスキル が結果を何倍にもブレさせます。同じAIでも、使う人で3-5倍の生産性差が出る — ここにパワーユーザーの鉱脈があります。

  • 手が止まらない — ファイル編集・シェル実行・git操作までCLI内で完結。コピペの往復がゼロに。
  • 確認しながら進む — 破壊的操作には許可プロンプトが挟まる。安全網が内蔵されている。
  • 拡張できる — Subagents / MCP / Hooks / Skills など、自分用に育てる仕掛けが豊富(Ch16-20で)。

始める前に — 自分のロールを決める

Claude Code を最大限使うための最初の一歩は、インストールではなく 自分のロール定義。「どんな仕事の、どんな役割で、誰と働くために」使うのかを言語化しておくと、毎回のプロンプトが自然に濃くなります。

ロール未定義

ぼんやりと使う

  • 「なんかできそう」で開く
  • 毎回前提を説明し直す
  • 同じミスが繰り返される
  • 結局Webのチャットで済ませがち
ロール定義済み

役割が明確

  • 「Next.js のフロント担当者」
  • 前提は CLAUDE.md に集約(Ch06)
  • 制約・好み・禁忌が共通認識
  • 手が止まらない

ロール例は下記。1つ選ぶ or 自分用に言語化してみてください。Ch06で CLAUDE.md に書き起こします。

個人開発者
副業/OSS。手数を増やしたい。品質より速度寄り
チーム所属エンジニア
既存リポに差分を入れる。テスト・レビューが必須
SRE/インフラ
CLI/スクリプト中心。破壊的操作が多くPlan重視

ここがプロの境目

/cost を毎日チェックして、使い方を最適化する。トークン消費が見えれば、「無駄にReadしてないか」「並列化しすぎてないか」を即時に修正できる。月末の請求で驚かないための基本動作であり、上位20%全員がやっていること。

インストール確認 — バージョン番号まで見る

すでに Claude Code を入れた状態でこのチュートリアルを読んでいる想定ですが、バージョン番号を確認する癖は最初に作ってしまいましょう。本書が前提にしているのは 2026/1 時点の CLI です。

# 1. Claude Code が入っているか確認
claude --version

# 2. ヘルプでサブコマンド一覧を確認
claude --help

# 3. (任意)最新化
npm i -g @anthropic-ai/claude-code
公式ドキュメント必読

インストール方法・推奨環境・サポートOSは公式に書いてある内容が一次情報です。ここは公式ドキュメントにも明記されており、ズレると認証エラー等で詰まるので必ずマスターしてください。Claude Code 公式 →

今すぐ試す(60秒)

ターミナルを開いて、下記3つを順に実行してください。全部通れば準備完了です。

claude --version
claude --help
cd ~/your-project && claude

最後のコマンドで対話セッションが立ち上がります。いったん /help と打って、使えるスラッシュコマンド一覧を眺めてから次に進みましょう。

もっと詳しく: グローバルインストールを避けたい人へ(npx / volta / mise)

Node.js のバージョン管理を volta / mise / fnm 等で行っている場合、global install よりも プロジェクト固有にピン止めする方が安全なケースがあります。たとえば npx -y @anthropic-ai/claude-code で起動時ごとに最新を呼ぶ方式、あるいは volta install @anthropic-ai/claude-code でツールチェインに入れる方式。どちらもチームに導入する時の再現性が上がります。詳細は公式docs参照。

覚える4つのスラッシュコマンド

初日にまず使うのは 4つだけ。/help /cost /clear /compact。これで「困ったら戻れる」「無駄がない」を両立できます。

コマンド役割使うタイミング
/help使えるコマンド一覧を出す迷ったとき。とりあえず打つ
/costセッションのトークン消費・料金1日の終わり、または重い作業の直後
/clear会話履歴をリセット(新セッション)コンテキストの切れ目(別タスクへ移行時)
/compact履歴を要約して圧縮履歴が膨らんできたが文脈は残したい時
/clear vs /compact の使い分け
graph TD
  Start(["履歴が重い"]) --> Q1{"文脈を
残したい?"}
  Q1 -- "Yes" --> C1["/compact で要約圧縮"]
  Q1 -- "No・新タスク" --> C2["/clear でリセット"]

迷ったら「このあと前提を使うか?」で判断。使うなら /compact、使わないなら /clear。
日次コストの「見える化」イメージ
/cost を毎日チェックしていると、使い方の変化(重い作業の日・軽い日)が可視化される。
今すぐ試す(30秒)

いま起動している claude セッションで、4つすべてを順に打ってみてください。

/help
/cost
/compact
/clear

/clear を打つと履歴がリセットされます。怖がらず押してみてOK — いつでも戻せる安心感を身体で覚えるのがこの章のもう一つの狙い。

もっと詳しく: /cost の読み方と月末に驚かないコツ

/cost の表示は「入力トークン / 出力トークン / キャッシュヒット / 合計見積もり」の4要素。注目すべきはキャッシュヒット率で、これが高いほど同じ前提で繰り返し使えていることを意味します。逆に毎回低いなら CLAUDE.md や既存コンテキストが活用されていない可能性。Ch06 と Ch14 で深掘りしますが、まずは 毎日1回見ることで肌感覚が付きます。

最初の1問を投げる — 体感をつかむ

準備が整ったら、最初の依頼を投げます。ポイントは いきなり本番業務ではなく、リスクのない「小さな依頼」から始めること。この1発で、Claude Code がどこまで自発的に動くかの肌感覚が掴めます。

おすすめの「最初の1問」テンプレ:

このディレクトリに何があるか、ls と head で概要を把握して、
このプロジェクトは何をするものか、100字以内で説明してください。
コードは触らないでください。

注目ポイントは3つ。(A) Claude が Bash や Read を自分で選んで使うこと。(B) 最後の1行「コードは触らないで」という 制約 が効くこと。(C) 100字以内という 成功条件 があると、Claude は余計な推測を書かないこと。

ペースは3つ。(A) 合宿型: 週末2日で10時間ずつ。没入感が強く一気にレベルアップするが、Project章での自社への応用は別日になる。(B) 日次30分型: 平日30分ずつ、48日で完走。毎日手を動かすので定着が高い。(C) 週次3時間型: 週末に1章分ずつ、8週で完走。家庭と両立しやすい。自分のリズムで選んでください。どれを選んでも /cost は毎日チェック。

Exercise 1
入門

初回セットアップと4コマンドの動作確認

自分の環境で以下を順に実行し、スクショ or メモを1つ残してください。

  1. claude --version でバージョンを確認(数字を控える)
  2. 適当なディレクトリで claude を起動
  3. /help /cost /compact /clear を順に実行
  4. 各コマンドの出力から、1行ずつ「気になった表示」をメモ
模範解答を見る

典型的なメモ例:

- claude --version: 2.x.x (2026/1時点の最新系列)
- /help: 思ったよりコマンドが多い(/model, /resume, /mcp など)
- /cost: 入力トークン数 / 出力トークン数 / 合計 $x.xx で表示される
- /compact: 「履歴を要約しました」というメッセージが出て応答が軽くなる
- /clear: 完全リセット。履歴が空になる

観察ポイント: /cost の数字の桁感(ドル単位? セント単位?)が自分の感覚とズレていないか確認。ズレているなら Ch14 のコンテキスト管理がすぐ効く合図です。

Exercise 2
応用

「自分のロール」を1段落で言語化する

Ch06で書く CLAUDE.md の原型を作ります。下記3つを含めて200字以内の1段落にまとめてください。

  1. 自分の立場(所属・職種・関わる主なリポジトリのタイプ)
  2. Claude Code に任せたい主なタスク(上位3つ)
  3. 絶対に任せたくないこと/禁忌(本番DB・機密ファイル・強制push 等)
模範解答を見る

例:

Next.js と TypeScript で個人開発している副業エンジニア。
主にフロント実装・UIの微調整・簡易APIを依頼したい。
任せる上位3つ:(1)差分を小さく刻む実装、
(2)PR本文の下書き、
(3)エラー原因の仮説立案。
禁忌:本番DBへの直接接続、.env / credential ファイルの編集、
force push、main への直pushは絶対にしない。
口調は簡潔に、英語の技術用語は日本語訳しない。

ポイント:「禁忌」を必ず書くこと。Claude Code は基本は従順ですが、言葉にして渡しておくことで確認プロンプトの頻度が下がり、やりたい作業に集中できるようになります。これが Ch06 で CLAUDE.md に昇格します。
