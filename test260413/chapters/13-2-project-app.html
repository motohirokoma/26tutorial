Step 1 — Server Componentで初期データ取得

// app/(app)/[view]/page.tsx
import { getServerSession } from 'next-auth';
import { authOptions } from '@/lib/auth';
import { prisma } from '@/lib/db';
import TaskList from '@/components/TaskList';
import AddBar from '@/components/AddBar';

export default async function ViewPage({ params }: { params: { view: string } }) {
  const session = await getServerSession(authOptions);
  if (!session?.user?.id) return <p>Sign in required</p>;

  const tasks = await prisma.task.findMany({
    where: { userId: session.user.id /* + view ごとの条件 */ },
    orderBy: [{ priority: 'desc' }, { dueAt: 'asc' }, { createdAt: 'desc' }],
  });

  return (
    <section>
      <h1 className="text-2xl font-bold mb-4 capitalize">{params.view}</h1>
      <TaskList initial={tasks} view={params.view} />
      <AddBar />
    </section>
  );
}
App Router の強み

サーバー側で prisma.task.findMany を直接呼べる。APIを経由しないのでネットワーク往復ゼロ、初期表示が爆速になります。

Step 2 — 楽観的UI で「サクサク感」を出す

チェックを入れた瞬間に取り消し線。サーバ応答を待たない。失敗したら戻す。これだけで体感速度が劇的に変わります。

通常 vs 楽観的UI のレスポンス時間 (体感)

useOptimistic を使う最小例

'use client';
import { useOptimistic, startTransition } from 'react';

export default function TaskList({ initial }: { initial: Task[] }) {
  const [tasks, setTasks] = useState(initial);
  const [optimistic, applyOptimistic] = useOptimistic(
    tasks,
    (state, action: { id: string; done: boolean }) =>
      state.map(t => t.id === action.id ? { ...t, done: action.done } : t)
  );

  async function toggle(id: string, done: boolean) {
    startTransition(() => applyOptimistic({ id, done }));
    const res = await fetch(`/api/tasks/${id}`, {
      method: 'PATCH',
      body: JSON.stringify({ done }),
    });
    if (!res.ok) {
      // 失敗時はサーバー再取得で戻す
      const fresh = await fetch('/api/tasks?view=today').then(r => r.json());
      setTasks(fresh.data);
    }
  }

  return (
    <ul className="space-y-1">
      {optimistic.map(t => <TaskItem key={t.id} task={t} onToggle={toggle} />)}
    </ul>
  );
}

Step 3 — キーボードショートカット

タスク管理アプリの肝は キーボードで完結すること。マウスに頼らないUXを Claude に作らせます。

Esc編集をキャンセル

// hooks/useHotkeys.ts
import { useEffect } from 'react';
export function useHotkeys(map: Record<string, () => void>) {
  useEffect(() => {
    function onKey(e: KeyboardEvent) {
      const tgt = e.target as HTMLElement;
      if (tgt.matches('input, textarea, [contenteditable]')) return; // 入力中は無視
      const handler = map[e.key];
      if (handler) { e.preventDefault(); handler(); }
    }
    window.addEventListener('keydown', onKey);
    return () => window.removeEventListener('keydown', onKey);
  }, [map]);
}

export default function TaskItem({ task, onToggle }: Props) {
  return (
    <li className="flex items-center gap-3 px-3 py-2 rounded hover:bg-zinc-50">
      <input
        type="checkbox"
        checked={task.done}
        onChange={(e) => onToggle(task.id, e.target.checked)}
      />
      <span
        className={`flex-1 transition-all duration-200 ${
          task.done ? 'line-through text-zinc-400' : 'text-zinc-900'
        }`}
      >
        {task.title}
      </span>
      {task.dueAt && <span className="text-xs text-zinc-500">{format(task.dueAt)}</span>}
    </li>
  );
}

注意: useOptimistic 経由で渡された task は「楽観的に更新済み」の状態。失敗時に巻き戻る挙動も自動。

Exercise 2
応用

キーボードショートカット ? でモーダル表示

? を押すと、ショートカット一覧のモーダルがフェードインで開く機能を追加してください。Server Component で初期に隠した状態でレンダーし、Client Component でトグルします。

  1. components/HotkeyHelp.tsx を Client Component で作成
  2. useHotkeys で ? をトグルにバインド
  3. Tailwindで fixed inset-0 backdrop-blur オーバーレイ
  4. Esc でも閉じられるようにする
  5. Lighthouse 90+ をキープ (アクセシビリティ aria-modal 等を忘れずに)
模範解答を見る 
'use client';
import { useState } from 'react';
import { useHotkeys } from '@/hooks/useHotkeys';

export default function HotkeyHelp() {
  const [open, setOpen] = useState(false);
  useHotkeys({
    '?': () => setOpen(o => !o),
    'Escape': () => setOpen(false),
  });
  if (!open) return null;
  return (
    <div role="dialog" aria-modal
         className="fixed inset-0 bg-black/30 backdrop-blur-sm flex items-center justify-center z-50"
         onClick={() => setOpen(false)}>
      <div className="bg-white rounded-xl p-6 w-[420px] shadow-xl" onClick={e => e.stopPropagation()}>
        <h2 className="font-bold mb-4">Shortcuts</h2>
        <dl className="grid grid-cols-[80px_1fr] gap-y-2 text-sm">
          <dt><kbd>n</kbd></dt><dd>新規追加</dd>
          <dt><kbd>j</kbd> <kbd>k</kbd></dt><dd>移動</dd>
          <dt><kbd>x</kbd></dt><dd>完了切替</dd>
          <dt><kbd>?</kbd></dt><dd>このヘルプ</dd>
        </dl>
      </div>
    </div>
  );
}

アクセシビリティのポイント: role="dialog" + aria-modal、フォーカストラップ、Escで閉じる。Claudeに「a11y要件: WCAG AA で...」とCLAUDE.mdに書いておくと最初から考慮してくれます。
