Project: 個人ポートフォリオサイト (Setup)

所要時間: 1.5h Phase 4: Projects 難易度: 実践

Step 1-2: Astro雛形を作る

まずは手動で1コマンド。Claudeに任せず、こちらでベースラインを敷く方が早いです。

mkdir -p ~/projects/portfolio && cd ~/projects/portfolio
npm create astro@latest .
# → "Empty" template, TypeScript: Strict, install deps: Yes, git init: Yes
npx astro add tailwind

Step 3: CLAUDE.md を最初に書く

これが本章の最重要ポイント。Claudeに丸投げする前に「やってほしくない事」を含めて伝えます。

# Project: Personal Portfolio

## 概要
個人ポートフォリオ静的サイト。GitHub Pages 公開。
コンテンツは src/content/ 配下の Markdown を使う。

## 技術スタック
- Astro 4 + TypeScript (strict)
- Tailwind CSS
- Content Collections (型安全Markdown)

## ディレクトリ
- src/pages/        — ルーティング (.astro)
- src/components/   — 再利用UI (PascalCase)
- src/content/      — works/*.md と blog/*.md
- src/layouts/      — ページ骨格

## デザイン原則
- ライト系ミニマル。アクセント1色 (#18181b)
- フォント: Inter + Noto Sans JP
- レイアウト最大幅 720px、余白多め

## NG
- インラインCSS は最終手段。原則 Tailwind ユーティリティ
- 外部APIへのfetch は禁止 (静的サイトのため)
- public/ 直下に巨大画像は置かない (1MB超は警告)
CLAUDE.md ファースト

「あとで書こう」ではなく最初に書く。これがあれば「Tailwind使って」「Markdownでコンテンツ書ける構造で」と毎回言わずに済みます。

Step 4: Claudeに最初のページを書かせる

ここからClaude登場。一気に全部頼まず、1ページずつ進めるのが鉄則です。

Claudeに src/pages/index.astro を作って。仕様: ヘッダ(名前/職業/SNS), Recent Works 3件のグリッド (Content Collectionsから取得), フッタ。Tailwindで Inter+Noto Sans JP。最大幅720px。 — 推奨プロンプトの一例
もっと詳しく: 1ページずつ進める理由

「全ページ一気に作って」と頼むと:

  • 修正したい時にどのファイルを直すか曖昧
  • Claudeのコンテキストが膨れて精度劣化
  • レビューしきれずバグが混入

1ページ作る → 動作確認 → commit → 次のページ、のルーティンが結局速いです。

Content Collections — 型安全な Markdown

Astroの強み。src/content/config.ts でスキーマを定義すると、Markdownのfrontmatterに型がつき、ミスを防げます。

// src/content/config.ts
import { defineCollection, z } from 'astro:content';

const works = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    summary: z.string().max(120),
    tech: z.array(z.string()),
    publishedAt: z.date(),
    cover: z.string().optional(),
  }),
});

export const collections = { works };

このスキーマがあれば、summary を120字以上書いた瞬間にビルドエラー。Claudeにもこのスキーマを伝えれば、整った Markdown を吐いてくれます。

Exercise 1
入門

Astro雛形を起こして "Hello, Portfolio" を表示

本章の Step 1-2 を実際に実行し、トップページに自分の名前を表示してください。

  1. npm create astro@latest portfolio で雛形作成 (Empty テンプレ)
  2. cd portfolio → npm run dev
  3. src/pages/index.astro を編集して「Hello, <あなたの名前>」と出す
  4. ブラウザで確認
模範解答を見る 
---
// src/pages/index.astro
const name = 'Motohiro Koma';
---
<html lang="ja">
  <head><meta charset="UTF-8" /><title>Hello</title></head>
  <body>
    <main style="font-family: sans-serif; padding: 40px;">
      <h1>Hello, {name} 👋</h1>
      <p>ポートフォリオ準備中です。</p>
    </main>
  </body>
</html>

注意: --- で囲まれた部分が frontmatter (サーバー側JS)、それ以下がHTML出力です。

Exercise 2
応用

Content Collectionsを定義し、Worksを2件Markdownで追加

本章の src/content/config.ts スキーマを書き、自分の作品を2件 Markdown で追加して、トップページに一覧表示してください。

  1. src/content/config.ts に works コレクションを定義 (上のサンプル参照)
  2. src/content/works/work-01.md と work-02.md を作成
  3. frontmatter に title/summary/tech/publishedAt を埋める
  4. index.astro から getCollection('works') で取得し、3件並べる
  5. storyへ「Tech は z.array(z.string()) でしたよね」と聞いて型確認
模範解答を見る

Markdownファイル例:

---
title: "Recipe Share"
summary: "自炊レシピをシェアできるWebアプリ"
tech: ["Next.js", "Prisma", "PostgreSQL"]
publishedAt: 2025-09-15
---

ユーザーが投稿したレシピを材料・カロリーで検索できる...

index.astro 側:

---
import { getCollection } from 'astro:content';
const works = (await getCollection('works'))
  .sort((a, b) => b.data.publishedAt.valueOf() - a.data.publishedAt.valueOf())
  .slice(0, 3);
---
<ul>
  {works.map(w => (
    <li>
      <h3>{w.data.title}</h3>
      <p>{w.data.summary}</p>
      <small>{w.data.tech.join(', ')}</small>
    </li>
  ))}
</ul>

スキーマ違反 (例: summary が120字超) があると npm run dev が即エラーになります。これがContent Collectionsの効用です。
