Step 1 — TaskListView の実装

// Views/TaskListView.swift
import SwiftUI
import SwiftData

struct TaskListView: View {
    let view: String
    @Query(sort: [SortDescriptor(\.priority, order: .reverse), SortDescriptor(\.dueAt)])
    private var allTasks: [Task]
    @Environment(\.modelContext) private var ctx
    @StateObject private var sync: SyncService

    init(view: String, ctx: ModelContext) {
        self.view = view
        _sync = StateObject(wrappedValue: SyncService(context: ctx))
    }

    var filtered: [Task] {
        let now = Date()
        switch view {
        case "today":    return allTasks.filter { !$0.done && ($0.dueAt ?? .distantFuture) <= now.endOfDay }
        case "upcoming": return allTasks.filter { !$0.done && ($0.dueAt ?? .distantPast) > now.endOfDay }
        case "done":     return allTasks.filter { $0.done }
        default:         return allTasks
        }
    }

    var body: some View {
        List {
            ForEach(filtered) { task in
                TaskRowView(task: task) { toggle(task) }
            }
            .onDelete(perform: delete)
        }
        .refreshable { await sync.pull(view: view) }
        .task { await sync.pull(view: view) }
        .overlay(alignment: .bottomTrailing) {
            Button { /* AddSheet 表示 */ } label: { Image(systemName: "plus.circle.fill").font(.system(size: 56)) }
                .padding()
        }
    }
}
@Query の魔法

SwiftDataの @Query は、データ変更を自動検知してView更新します。手動でreloadする必要なし。React の useState よりも宣言的。

Step 2 — TaskRowView と Haptic フィードバック

// Views/TaskRowView.swift
import SwiftUI

struct TaskRowView: View {
    let task: Task
    let onToggle: () -> Void
    @State private var animating = false

    var body: some View {
        HStack(spacing: 12) {
            Button(action: {
                withAnimation(.spring(duration: 0.25)) {
                    onToggle()
                    animating.toggle()
                    UIImpactFeedbackGenerator(style: .light).impactOccurred()
                }
            }) {
                Image(systemName: task.done ? "checkmark.circle.fill" : "circle")
                    .font(.title2)
                    .foregroundStyle(task.done ? .green : .secondary)
            }
            VStack(alignment: .leading, spacing: 2) {
                Text(task.title)
                    .strikethrough(task.done, color: .secondary)
                    .foregroundStyle(task.done ? .secondary : .primary)
                if let due = task.dueAt {
                    Text(due, style: .date).font(.caption).foregroundStyle(.secondary)
                }
            }
            Spacer()
            if task.priority >= 3 { Image(systemName: "exclamationmark").foregroundStyle(.red) }
        }
        .contentShape(Rectangle())
    }
}
Haptic は必ず

iOS の体験を一段引き上げるのが Haptic。完了タップにふさわしい .light impact を1行入れるだけで「気持ちいいアプリ」になります。

Archive & Upload

step 5
TestFlight

step 6
審査提出
説明文・プライバシーポリシー・年齢制限を埋めて Submit for Review。
ASWebAuthenticationSession のリダイレクトURL

App Store審査でハネられる頻出ポイント。tasqios://callback のような独自スキームを Info.plist の CFBundleURLTypes に登録し、Web側の OAuth Redirect URI にも一致させること。

Tasq iOS を Simulator で動かす

本章までのコードをXcodeでビルドし、iPhone 15 Simulator で起動。Tasqに3件タスクを追加して、Web側 (Ch13) のサーバとデータが同期することを目視確認してください。

  1. Xcodeで Cmd + R で Simulator 起動
  2. Sign in with Google → Web側のセッションと同じユーザでログイン
  3. iOSで「テスト1, テスト2, テスト3」を追加
  4. Webブラウザで https://<tasq-url>/today を開いて3件あるか確認
模範解答を見る

同期しない時のチェック:

  • iOS Simulator から本番URL (https) にアクセスできる? (Wi-Fi / proxy 問題)
  • Bearer トークンがHTTPヘッダに正しく入っている? (Charles Proxy で確認可)
  • Web側API がCORS等で iOS UA を蹴っていない?
  • ローカル開発時は http://192.168.x.x:3000 を使い、Info.plist の ATS 例外設定が必要

4週間の学習プランを設計して、CLAUDE.mdに書く

本章までで身についた力を、次の4週間でどう伸ばすか。あなた専用の学習プランを設計し、自分の ~/.claude/CLAUDE.md に追記してください。

  1. 「自分が今一番苦手な分野」を1つ言語化 (例: テスト / iOS / MCP自作)
  2. その分野を伸ばす具体的なミニプロジェクトを1つ考える
  3. 4週間を週単位で区切り、週ごとの到達点を3行で書く
  4. Claude に「私の今のスキルは <...> で、4週間後に <...> になりたい。具体プランをレビューして」とフィードバックを依頼
  5. レビューを反映して、確定版を ~/.claude/CLAUDE.md の末尾に # Learning Plan 2026Q2 として追記
