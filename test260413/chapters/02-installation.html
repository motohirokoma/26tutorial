インストールとセットアップ

この章を終えると、自分のマシンでClaude Codeが動く状態になります。ターミナル初心者でも大丈夫、一歩ずつ進めましょう。

所要時間: 1h Phase: Basics 難易度: 入門

インストール前のチェック

必要なものは3つだけ。順に確認しましょう。

OS
macOS / Linux / Windows(WSL)
Node.js
v18 以上
Anthropic アカウント
Pro / Max / API キー
インストール全体フロー
1
前提確認
OS / Node.js
2
Node.js 導入
nvm 経由が安全
3
Claude Code 導入
npm install -g

ログイン or APIキー
5

claude
トラブルが起きやすいのは 2 と 4。ここを丁寧にやれば後はスムーズ。

Node.jsのインストール

すでに node --version が v18 以上を返すなら、この節はスキップしてOKです。

方法A: nvm 経由(推奨)

将来 Node.js のバージョンを切り替えたくなる場面が必ず来ます。最初から nvm で入れておくと安心。

# macOS / Linux / WSL
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# シェルを開き直して
nvm install 22
nvm alias default 22
node --version  # v22.x.x

方法B: 公式インストーラ

「とりあえず動けばOK」なら nodejs.org から LTS 版をダウンロード。

Windows ユーザーへ

ネイティブの PowerShell でも動きますが、WSL (Ubuntu) を入れて Linux ベースで作業すると、Claude Code に限らずほぼすべての開発ツールがスムーズに動きます。

Claude Codeのインストール

1コマンドで完了します。

npm install -g @anthropic-ai/claude-code

確認:

claude --version  # 例: 1.x.x
もっと詳しく: -g フラグの意味

-g = global。プロジェクトのフォルダ内ではなく、システム全体にインストールする指定。Claude Code は「どのフォルダで起動しても呼べるコマンド」にしたいので、必ず -g を付けます。

権限エラー(EACCES)が出たら、それは「nvmじゃなくシステム標準のNodeに入れようとしている」サイン。本文の方法Aで nvm 経由で入れ直すと解決します。sudo で押し通すのは推奨されません。

認証の設定

3つの選択肢があります。個人で始めるなら方法A、会社で使うなら方法Bが王道です。

A: Subscription

Pro/Max でログイン

  • 初期設定が一番簡単
  • ブラウザ認証で完結
  • 毎月定額・上限あり
B: API Key

従量課金キーを使う

  • console.anthropic.com で発行
  • 使った分だけ課金
  • チーム/CI 向け
C: Bedrock/Vertex

クラウド経由

  • AWS/GCP のAPIから
  • 企業ガバナンス対応
  • 環境変数で切り替え

方法A: ログイン認証(個人推奨)

# 初回起動するとブラウザが開く
claude
# → 表示されたURLをブラウザで開いて、Anthropicアカウントでログイン
# → 表示された認証コードをターミナルに貼り付け

方法B: APIキー

# console.anthropic.com で「Create Key」してから
export ANTHROPIC_API_KEY="sk-ant-..."

# 永続化したい場合は ~/.zshrc などに書く
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.zshrc
超重要

APIキーを 絶対に Git にコミットしない。.env ファイルに書くなら必ず .gitignore に .env を入れる。間違えてpushしてしまったら、Anthropic Console で即座に「Revoke」してから新しいキーを発行し直す。

認証方式の選び方フロー
graph TD
  A["どれを使う?"] --> Q1{"個人 or チーム?"}
  Q1 -- 個人 --> Q2{"月額 or 従量?"}
  Q2 -- 月額固定 --> A1["A: Pro/Max ログイン"]
  Q2 -- 使った分だけ --> A2["B: API キー"]
  Q1 -- 会社 --> Q3{"クラウド指定ある?"}
  Q3 -- AWS --> A3["C: Bedrock 経由"]
  Q3 -- GCP --> A4["C: Vertex 経由"]
  Q3 -- なし --> A2
迷ったらA。学習目的ならProプラン+ログインで十分。

初回起動してみよう

適当な空フォルダを作って、その中で claude を起動します。

mkdir hello-claude && cd hello-claude
claude

プロンプト > が出たら、何でも話しかけてOK。試しに:

> こんにちは。あなたが何ができるか、3つだけ簡単に教えて。
うまくいったサイン

3行くらいの自己紹介が返ってくれば成功です。返答スタイルが「ちょっと冗長だな」と感じても、それは Ch04 のプロンプト設計で改善できます。

設定ファイルってどこにあるの?

Claude Code は2層の設定を持ちます。

場所パス用途
ユーザー全体~/.claude/全プロジェクトで共通の設定・履歴・Skill 等
プロジェクト固有./.claude/そのプロジェクト専用の設定・許可リスト

主な中身:

  • settings.json — モデル選択、permission, env 等
  • CLAUDE.md — Claude に常に読ませる「指示書」(Ch09 で詳述)
  • skills/ — 自作スキル(Ch10 で詳述)
  • memory/ — 過去の会話の自動メモリ

VS Code拡張も入れておこう

ターミナル単体でも動きますが、エディタ内で完結したいなら拡張機能を入れましょう。

  1. VS Code を起動
  2. Extensions(拡張機能)タブを開く
  3. 「Claude Code」で検索 → Anthropic 公式の拡張をインストール
  4. サイドバーに Claude アイコンが出れば成功

困ったときのチェックリスト

Troubleshooting
command not found: claude
npm の global インストールパスが PATH に入っていない。npm config get prefix で出たパスの /bin を ~/.zshrc に通す。nvm経由なら通常は不要。
EACCES: permission denied
システム標準Nodeに -g インストールしようとしている。nvm経由のNodeに切り替えてから再実行。
ブラウザ認証画面が開かない
SSH越しなど。--no-browser 起動 → 表示されたURLを手元ブラウザで開く。
「context window exceeded」
セッションの履歴が長すぎ。/clear で履歴クリアまたは /compact で要約圧縮(Ch03 で扱う)。
Exercise 1
入門

インストール完了確認をする

下記コマンドを順に実行して、すべて期待通りの出力になるか確認しましょう。

  1. node --version → v18 以上
  2. claude --version → 数字が表示
  3. claude 起動 → プロンプト > が出る
  4. 「Hello」と話しかけて返事が返ってくる
模範解答を見る

すべて通れば成功。途中で詰まったら、上の「困ったときのチェックリスト」を参照。

典型的な詰まりポイント:

  • Node が古い → nvm install 22 && nvm alias default 22
  • claude コマンドが認識されない → 新しいターミナルウィンドウを開き直す
  • 認証エラー → claude /logout 後にもう一度起動

動作確認できたら、/exit で終了します。

Exercise 2
応用

プロジェクト用の .env を安全に管理する

仮の API_KEY=sk-test-xxxx を .env に書いて、絶対にGitにpushされないように設定しましょう。

  1. 新規フォルダを作って git init
  2. .env に API_KEY=sk-test-xxxx を書く
  3. .gitignore に .env を追加
  4. git status で .env が追跡対象に含まれていないことを確認
  5. 確認できたら、Claude に 「.env.example だけ作って、実値はマスクして」と頼む
模範解答を見る 
mkdir env-demo && cd env-demo
git init
echo "API_KEY=sk-test-xxxx" > .env
echo ".env" > .gitignore
git status
# → .gitignore のみが Untracked、.env は出てこなければ成功

Claude を起動して:

> .env を読んで .env.example を作って。値は <your-key> のようにマスクしてください。

これで「キーの存在は記録するが、実値はリポに残さない」運用ができます。チーム参加時の標準パターンです。
