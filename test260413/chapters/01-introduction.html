Claude Codeって何?

まずは全体像を掴みましょう。難しい話は後回しで、「これ使えそう!」と感じてもらえればOKです。

所要時間: 1h Phase: Basics 難易度: 入門
前提知識が不安な方へ: はじめての方へ (Ch00) を先に読む →

一言で言うと

Claude Code は、ターミナルで動く「ペアプログラミングするAI相棒」です。Claude という賢いAIに、自分のコードベースを直接見せながら「ここを直して」「これ実装して」「テスト書いて」と頼める道具です。

3つの世界の中の Claude Code
「コードを編集できる」「AIと話せる」「コマンドを動かせる」の交点に立つツール、それがClaude Code。
エディタとチャットを行き来していた時代から、同じ画面で対話しながら作業できる時代へ。それがClaude Codeの体験。

実例で見てみよう

ターミナルで claude と打つと、こんな感じで会話が始まります。

$ claude
Welcome to Claude Code!

> README.md を読んで、このプロジェクトの目的を3行で要約して

[Reading README.md...]
このプロジェクトは…
1. Webサイトのアクセス解析を簡単に始める
2. ノーコードで設定可能なダッシュボード提供
3. プライバシー優先 (Cookie不使用)

> テストも追加してほしい。Jestで。

[Creating tests/index.test.js...]
テストを5件追加しました。npm test で実行できます。

ポイントは、Claude が自分でファイルを読み、自分で書き、自分でコマンドを実行すること。Web ChatGPT に「コピペして貼り付けて、結果を返して」と往復する手間がありません。

何ができるの?

1

コードを読む

大規模リポジトリでも構造を把握。「この関数どこで呼ばれてる?」と聞けば探してくれる。

2

コードを書く

新規ファイル作成、既存ファイル編集、リファクタリング。差分を見せて確認してから適用。

3

テスト・デバッグ

失敗するテストを書いて → 修正 → 通す、の TDD サイクルもこなす。

4

Git 操作

差分を見て意味のあるコミットメッセージ、PR文の自動生成、コードレビュー。

5

調査・調べ物

ライブラリの使い方、ドキュメント要約、エラーメッセージの解読、ベストプラクティス相談。

6

定型作業の自動化

命名規約のリネーム、設定ファイル整備、CIワークフロー生成 など。

どうやって動いてるの?

裏側の仕組みは「Agent ループ」。あなたの指示を受けたClaudeが、必要に応じてツール(Read/Edit/Bashなど)を呼び、結果を見て次の手を決める…という繰り返しです。

Agent ループのしくみ
sequenceDiagram
  participant U as あなた
  participant C as Claude
  participant T as ツール
  participant F as ファイル/シェル
  U->>C: 「テスト書いて」
  C->>T: Read(src/calc.js)
  T->>F: 読み取り
  F-->>C: ファイル内容
  C->>T: Write(tests/calc.test.js)
  T->>F: 書き込み
  C-->>U: 「5件のテストを書きました」
指示 → 必要な道具を自分で選ぶ → 実行 → 結果を確認 → 次の手 の繰り返し。
もっと詳しく: なぜ「ターミナル」がベストなのか

VS Code 拡張やGUIアプリも存在しますが、本体は CLI です。理由は単純で、「開発者の作業の中心はターミナル」だから。git, npm, テストランナー、デプロイコマンド、すべてターミナルで動く。そこにAIを置くのが最も自然で、最も速い。SSH越しのサーバーでも動くし、CIに組み込むこともできる。

他のAIツールと何が違うの?

ツール形態得意苦手
ChatGPT (Web)ブラウザ会話・調べもの・短いコード大規模コードへの直接アクセス
GitHub Copilotエディタ補完1行〜関数単位の補完複数ファイルにまたがる作業
Cursorエディタ統合ファイル編集 + チャットCLI/Git操作の自動化
Claude CodeCLI / IDE長文理解・複数ファイル編集・CLIタスクUIで派手な見た目が必要なタスク

「もう Copilot を使ってる」「Cursor 派です」という方も、使い分けでOK。Claude Code は「対話しながら、複数ファイルにまたがる中・大規模タスクを進める」のが特に強いです。

どこで使えるの?

  • ターミナル単体 — macOS/Linux/WSL のシェルで claude 起動
  • VS Code 拡張 — エディタ内のサイドパネルで対話できる
  • SSHログイン先のサーバー — リモート作業もそのまま
  • GitHub Actions などのCI — 自動でPRレビュー/issue対応も可能

Exercise 1
入門

Claude Code を使うイメージを言語化する

あなたが普段どんな開発作業に時間を取られているか、3つ書き出して、それぞれ Claude Code に何をやってもらえそうかメモしましょう。

  1. 普段の作業を3つ書き出す(例: バグ修正、テスト書き、PR文作成)
  2. 各作業を Claude にどう頼みそうかを 1 行で書く
  3. 「自分でやった方が早そうな部分」「Claudeに任せたい部分」を区別する
模範解答を見る

例:

作業Claudeへの頼み方判断
レビュー前のPR説明文を書く「このdiffを見てPR descriptionを書いて」任せる(下書き → 自分で調整)
軽微なリファクタ「foo関数を3つに分割して、テストも維持して」任せる
本番DBのデータ変更—自分でやる(危険な操作は手作業)

判断軸: 「失敗してもロールバック可能か」「自分の意思決定が要るか」。要らないルーチン作業ほどClaude向き。

Exercise 2
応用

競合ツールとの使い分け表を作ってみる

あなたが普段使っている AI ツール(ChatGPT/Copilot/Cursor 等)と、Claude Code を、自分の用途で比較する表を作ってみましょう。

  1. 横軸に「用途(調査/補完/複数ファイル編集/CLIタスク 等)」
  2. 縦軸に「ツール名」
  3. 各セルに ◎/○/△ を埋めてみる
  4. 「ここはどうしてもClaude Codeが必要」と思うセルを1つ見つける
模範解答を見る

判断は人それぞれだが、典型的にはこうなる。

ツール調査1行補完複数ファイル編集CLIタスク
ChatGPT◎△△×
Copilot△◎△×
Cursor○○○△
Claude Code○—◎◎

Claude Code が真価を発揮するセル: 「複数ファイル編集」「CLIタスク」。逆に「エディタ内の1行補完」は今でも Copilot が速い。併用こそ最強です。
