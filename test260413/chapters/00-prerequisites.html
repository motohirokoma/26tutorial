はじめての方へ

「ターミナル」「npm」「APIキー」「Git」など、Claude Codeを触るうえで前提となる用語をやさしく解説します。プログラミング未経験でも、ここを読めばCh01から進められます。読み飛ばしてOKな項目もあるので、気になるところだけ拾い読みしてください。

この章の進め方

進捗トラッキングの対象外です。「言葉を聞いたことがあるけど何かはよく知らない」を「ざっくりわかる」に変えるのが目的。完璧を目指さず、雰囲気を掴んだら次へ進みましょう。

そもそも「ターミナル」って何?

ターミナルは、文字でコンピューターを操作する黒い画面のことです。マウスでアイコンをクリックする代わりに、英単語(コマンド)を入力して指示します。Claude Code は ChatGPT のような Web画面ではなく、このターミナルから動くツールです。

GUI と CLI の違い
同じ「メモを開く」でも、GUIはクリック、CLIはコマンドを打つ。Claude Codeは右側の世界の住人。
OS標準のターミナル起動方法
macOSTerminal.appSpotlight (⌘+Space) で「terminal」と入力
WindowsWindows Terminalスタートメニューから「Terminal」or「PowerShell」
LinuxGNOME Terminal 等Ctrl+Alt+T など

最低限覚えておくと楽なコマンド

Claude Code は内部でこれらを使うので、自分で打つ機会は少ないですが、画面に出てきたとき意味がわかると安心です。

  • pwd — いま自分がどのフォルダにいるかを表示する(print working directory)
  • ls — 今のフォルダにあるファイル/フォルダを一覧表示する
  • cd フォルダ名 — フォルダに移動する。cd .. で1つ上に戻る
  • cat ファイル名 — テキストファイルの中身を表示する
  • Ctrl+C — 動いてるコマンドを止める。詰んだらコレ

「ディレクトリ」って「フォルダ」のこと?

はい、ほぼ同じ意味です。GUI で言う「フォルダ」を CLI 文化では「ディレクトリ」と呼びます。Claude Code のドキュメントには「ディレクトリ」と書かれていることが多いですが、頭の中では「フォルダ」と置き換えてOKです。

Node.js と npm って何?

Node.js は JavaScript をパソコンで動かすためのソフトです。Claude Code は中身が JavaScript で書かれているので、これがインストールされていないと動きません。

npm は「Node Package Manager」の略で、Node.js 用のソフトを インストール/アップデート/削除 する仕組みです。Apple App Store の開発者版だと思ってください。

Claude Code を入れるまでの依存関係
graph LR
  A["パソコン"] --> B["Node.js
(JS実行環境)"]
  B --> C["npm
(パッケージ管理)"]
  C --> D["Claude Code

下から順に依存している。Node.jsを入れると npm も付いてくるので、実質「Node.js → Claude Code」の2ステップ。
もっと詳しく: なぜ npm が必要なのか

世の中のJSツールは何万とあり、それぞれ依存しあっています。手作業で集めるのは現実的でないため、npm install -g @anthropic-ai/claude-code のように1コマンドで本体+依存を全部揃えてくれる仕組みが npm です。-g は「どこからでも呼べるように、グローバルに入れる」という意味のフラグです。

「APIキー」って何? なぜ必要?

APIキー は「あなたが Anthropic 社の AI を使う鍵」です。Web の ChatGPT が「ログインしてブラウザで使う」のに対して、Claude Code は「キーを持って外部ツール経由で AI を呼ぶ」スタイルもサポートしています。

取り扱い注意

APIキーは 家の鍵 と同じです。GitHub などに誤ってアップロードすると、知らない人が使い、課金が発生します。Ch02 で「.env ファイルに入れて gitignore する」やり方を扱います。

Claude Pro/Max プランで Claude Code を使う場合は、ブラウザでログインすればOK(キー不要)です。職場で会社のAPIキーを使うパターン、個人の Anthropic Console で発行するパターン、いろいろあります。Ch02 で詳しく扱うので、今は「鍵」というイメージだけ持っておいてください。

GitHubとGitの違いは?

Git

道具(ローカル)

  • 「いつ・誰が・何を変えた」を記録する仕組み
  • パソコンの中で動く
  • インターネット接続なしで使える
GitHub

場所(クラウド)

  • Gitで管理したコードを置くWebサービス
  • 共有・レビュー・ディスカッションの場
  • 履歴をブラウザで眺められる

例: Word が「Git」、OneDrive/Google Drive が「GitHub」。手元で書く道具と、共有する場所、と分けて覚えると混乱しません。Claude Code は両方を扱えます(Ch06 / Ch07)。

コードエディタって何を使えばいい?

Claude Code 単体でも完結しますが、生成されたコードを目で読む・微調整するには VS Code が最もおすすめです。無料・全OS対応・拡張機能豊富。Ch02 で拡張機能の入れ方も紹介します。

困ったときの心構え

エラーは「失敗」ではなく「コンピューターからのお知らせ」。恥ずかしいことではないし、Claudeに見せれば一緒に読んでくれる。
  • エラーが出たら、そのまま Claude にコピペ — 自分で原因を考えるより速い。Ch08 で型を学びます。
  • 詰まったら一回終了 — Ctrl+C で止めて再起動。多くの不具合はこれで直る。
  • 用語がわからない単語は、本サイトの 点線下線 にカーソルを当てる — 説明がポップアップで出ます。
