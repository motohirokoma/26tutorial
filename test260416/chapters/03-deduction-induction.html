演繹法と帰納法

2大推論パターンを身につけ、後の章で出てくる So What / Why So の往復運動の土台を作ります。30分で「論理の飛躍を見抜く目」を養います。

所要時間: 30分 Phase: Basics 難易度: ★★☆

1. 推論の2方向

論理的推論は方向で2つに分かれます。演繹法(deduction)は「上から下」、帰納法(induction)は「下から上」。両者は対立ではなく 補完関係 にあります。

演繹は降りる、帰納は登る
演繹は確実(前提が正なら結論も正)、帰納は蓋然的(反例が出ると崩れる)。

2. 演繹法 — 三段論法と現代的応用

演繹法の典型は三段論法です。「大前提 + 小前提 → 結論」の3行構造。

大前提: すべての人間は死ぬ
小前提: ソクラテスは人間である
結論:   ソクラテスは死ぬ

ビジネス文脈での例は次のとおり。

大前提: 市場成長率5%以下の業界では新規参入の成功確率は20%未満
小前提: 当社が検討している業界の成長率は2%
結論:   この業界への新規参入は成功確率が20%未満と推定される

演繹が崩れる典型パターン

パターン例何が問題か
大前提が偽「すべての顧客は安い方を選ぶ」+ 当社は安い → 売れる大前提に反例多数(高品質志向、ブランド志向の顧客)
小前提が偽「業界A→Bに転職した人は皆給与アップ」+ 私はAからBに転職する → アップ「皆」が偽のことも
関係に飛躍「タバコは健康に悪い」+ あなたは健康になりたい → コーヒーをやめろ大前提と小前提が同じ対象を扱っていない

3. 帰納法 — 事例からパターンを引く

帰納法は 複数の観察事実から共通点を見つけて一般則を導く 推論。マーケティングのインサイト発見、競合分析、リサーチの結論はほぼ全て帰納法です。

事例1: 顧客A → 「夜10時以降に購入が多い」
事例2: 顧客B → 「夜9時以降に購入が多い」
事例3: 顧客C → 「夜11時以降に購入が多い」
   ↓
パターン: 当サイトの主要購入は「夜の時間帯」
   ↓
一般則: 夜帯にプロモを打つと反応が良い (蓋然的)
So What / Why So の往復運動
sequenceDiagram
  participant F as 事実(下)
  participant C as 結論(上)
  F->>C: So What? (事実から何が言える?)
  C-->>F: Why So? (結論を支える事実は?)
  F->>C: So What?
  C-->>F: Why So?
  Note over F,C: 上下を往復することで論理が締まる
So What は帰納方向(下→上)、Why So は演繹方向(上→下)で検証。両方を行き来する。

帰納法の落とし穴

  • サンプル不足 — 3件で「これが法則だ」と言うのは早急。最低でも10件、できれば30件以上。
  • サンプル偏り — 自社顧客だけ見ても市場全体は語れない。母集団を意識する。
  • 反例無視 — 自分の仮説に合う事例だけ集めると確証バイアスに陥る。反例も探す。
  • 因果と相関の混同 — A と B が同時に起きても、A→B とは限らない。

4. 2つを比べる

観点演繹法帰納法
方向一般から個別(上→下)個別から一般(下→上)
結論の強さ確実(前提が真なら必ず真)蓋然的(反例で崩れる)
使う場面既存ルールの当てはめ新しいインサイトの発見
ビジネス例規制適合判定、社内ルール適用顧客インサイト、市場トレンド分析
典型的な失敗大前提が古い・偽サンプルが少ない・偏る
業務シーン別 — どちらを使うか
実務での使い分け

新しい何かを発見する局面では帰納、既存の枠組みに当てはめる局面では演繹。「事実集め→パターン発見→当てはめ→検証」を繰り返すと両方を自然に使うことになる。

Exercise 1
入門

演繹/帰納を見分ける

次の推論はそれぞれ演繹/帰納のどちらでしょうか。理由も短く。

  1. 「先月のセールで売上が30%増えた。今月もセールを打てば売上が伸びるはずだ」
  2. 「コスト削減10%を全部署に通達。営業部もコストを10%削減する必要がある」
  3. 「直近3社のM&A後3年で離職率が上昇している。次のM&Aでも離職率上昇が起きそうだ」
模範解答を見る
  • 1: 帰納 — 1事例(先月)から一般則(セール=売上増)を導いている。サンプル1なので非常に脆弱。
  • 2: 演繹 — 一般則(全部署に通達)を個別(営業部)に当てはめている。大前提が真なら結論も真。
  • 3: 帰納 — 3事例から一般則を導いている。3件はやや少ないので、業界別・規模別の追加データで検証したい。
Exercise 2
応用

So What / Why So を1往復する

あなたの直近の業務データ(売上、KPI、ユーザー指標、何でも可)を1つ取り上げ、次のステップで So What / Why So を実践してください。

  1. 事実を3〜5個書き出す
  2. So What? — 事実から言えるパターン・結論を書く(帰納)
  3. Why So? — その結論を支える事実は本当に十分か?反例はないか?を書く(検証)
  4. 必要なら結論を修正する
模範解答を見る

例:

事実:
- 月曜日の問い合わせ件数が他曜日の1.4倍
- 月曜の問い合わせ平均待ち時間 12分(他曜日平均 5分)
- 顧客満足度 月曜のみ 3.2/5 (他曜日 4.1/5)
- スタッフ配置は曜日均一

So What?
 月曜は需要 vs 供給のミスマッチが起きており、満足度を下げている。

Why So? (検証)
 - サンプルは何ヶ月分? → 過去3ヶ月の傾向。短い? → 季節性影響あるかも → 半年データで再確認
 - 月曜が高いのは祝日連休明けだけかも → 連休明けと通常月曜を分けて再集計
 - 問い合わせ内容が違う可能性 → 問い合わせ種別別に深掘り

修正後の結論:
 月曜の高需要は確認できたが、内訳と季節性を切り分けてからスタッフ増員を判断したい

このように Why So で結論をいったん疑い、修正することで論理の品質が劇的に上がります。
