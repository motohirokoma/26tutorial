Phase 2: Core > Chapter 04

ロジックツリー

「分解の道具」としてのツリー。What / Why / How の3型を使い分け、テーマを構造的に深掘りします。60分でツリーを書く手が動くようになります。

所要時間: 60分 Phase: Core 難易度: ★★☆

1. ロジックツリーとは

ロジックツリーは、1つのテーマを 階層的に MECE 分解した木構造。MECE(Ch.02)の実装形態であり、その後のフレームワーク全ての土台です。

ロジックツリーの基本構造
幹(テーマ)→枝(分解)→葉(細目)。各分岐がMECEなら全体もMECE。

2. 3つの型 — What / Why / How

ロジックツリーは「何を分解するか」で3型に分かれます。同じ見た目でも目的が違うので、最初に 「自分はどの型を作っているか」 を意識します。

  • What ツリー(構成要素分解) — 対象が「何で構成されているか」を MECE 分解。例: 売上=客数×単価、コスト=人件費+材料費+...
  • Why ツリー(原因分解) — 問題が「なぜ起きているか」を分解。例: 離職率高い → 給与/業務量/人間関係 → ...
  • How ツリー(手段分解) — 目的を「どうやって達成するか」を分解。例: 売上倍増 → 客数増/単価増 → 新規獲得/離脱防止...

What ツリーの例

売上 = 客数 × 単価 (Whatツリー)
graph TD
  S["売上"] --> N["客数"]
  S --> P["単価"]
  N --> N1["新規顧客"]
  N --> N2["既存顧客"]
  P --> P1["商品価格"]
  P --> P2["購入点数"]
  N1 --> N1a["広告経由"]
  N1 --> N1b["オーガニック"]
  N2 --> N2a["リピート率"]
  N2 --> N2b["アップセル"]
構成要素を数式と対応付けて分解。MECEが保証されやすい。

Why ツリーの例

なぜ離職率が高いのか (Whyツリー)
graph TD
  R["離職率が高い"] --> A["待遇への不満"]
  R --> B["業務への不満"]
  R --> C["人間関係への不満"]
  R --> D["将来不安"]
  A --> A1["給与水準"]
  A --> A2["評価制度"]
  B --> B1["業務量過多"]
  B --> B2["業務内容"]
  C --> C1["上司との関係"]
  C --> C2["同僚との関係"]
  D --> D1["キャリア"]
  D --> D2["会社の業績"]
「なぜ?」を繰り返して原因を深掘り。3-5階層で根本原因に届く。

How ツリーの例

どうやって離職率を下げるか (Howツリー)
graph TD
  H["離職率を下げる"] --> H1["待遇改善"]
  H --> H2["業務改善"]
  H --> H3["関係性改善"]
  H --> H4["キャリア支援"]
  H1 --> H1a["給与改定"]
  H1 --> H1b["評価制度見直し"]
  H2 --> H2a["業務量適正化"]
  H2 --> H2b["業務内容多様化"]
  H3 --> H3a["1on1強化"]
  H3 --> H3b["チームビルド"]
  H4 --> H4a["研修制度"]
  H4 --> H4b["異動機会"]
Whyツリーと対称形で、原因に対応する打ち手を網羅する。

3型を比べると、それぞれの用途と前提条件が見えてきます。

型問い使う場面注意点
What「何で構成されている?」現状把握、構造理解数式に落ちると最強
Why「なぜ起きている?」原因究明、診断3-5階層で根本に届く
How「どうやる?」打ち手立案Whyと対称構造にする

各ステップの実例

  1. 頂点定義:「離職率を下げる」(動詞+目的語で1文)
  2. 型選択: 打ち手を考えたい → How ツリー
  3. 第1分岐: 待遇/業務/関係/キャリアの4切り口で MECE 分解
  4. 深掘り: 各枝を更に2-3階層分解
  5. 検証: 「全枝を実行したら本当にゴールに届く?」(So What)、「各枝はゴールに必要?」(Why So)
枝数の目安

1階層あたり3-5本が読みやすい。2本だと「他にない?」と感じ、6本以上だと頭に入らない。3-5本に収まる切り口を選ぶと自然と読みやすいツリーになる。

最初の「頭の中」の感覚 — ラフスケッチ
慣れないうちは、まずこんな手書き風で十分。後で清書する。

4. よくあるアンチパターン

アンチパターン

「思いついた順」のツリー

  • 第1分岐がMECEでない
  • 階層レベルがバラバラ
  • 枝の太さが極端に偏る
ベストプラクティス

「切り口先取り」のツリー

  • 第1分岐の切り口を3つ試して選ぶ
  • 同階層の粒度を揃える
  • 枝の太さを意識して再分解

枝の「太さ」を意識する

ツリーの枝にはインパクトの大小があります。同じ深さで描いても、ある枝は売上の60%、別の枝は5%、ということはざらにあります。太い枝(影響大)から優先的に深掘りするのがコンサル的なツリーの読み方です。

同じツリーでも「枝の太さ」は均等ではない

このグラフを見れば、「既存リピート」と「既存アップセル」だけで7割超え。打ち手を考えるならまずこの2枝を深掘りすべき、と分かります。

もっと詳しく: 「均等深掘り」と「重点深掘り」

初学者は全枝を均等に深掘りしがちですが、コンサル熟練者はインパクトの大きい枝に時間を集中させます。これを 「80/20原則」 とも呼びます。

ただし、初期段階では 「無理やり全枝を1階層深掘りしてから、太い枝だけ更に深掘り」 という二段階アプローチが安全です。最初から太さを判断するのは経験が必要だからです。

5. ツリーを使い倒す3つのコツ

  1. 最初の1分は描かず考える — テーマを1文に落とす、型を選ぶ、第1切り口の候補3つを書き出す。これだけで質が劇的に上がる。
  2. 3階層描いたらズームアウト — 全体を眺めてMECE違反、レベル不揃い、太さの偏りをチェック。
  3. 結論を頂点に書き直す — 完成したツリーは ピラミッド構造(Ch.06) に変換できる。これが資料化への近道。
ツリーの本当の力は、「一人で考える」ためではなく「他者と思考を共有する」ためにある。 — 大前研一
Exercise 1
入門

3型を見分ける

次のツリーは What / Why / How のどの型でしょうか。

  1. 「コスト」 → 「固定費」+「変動費」 → ...
  2. 「採用が進まない」 → 「母集団少ない」+「歩留まり悪い」 → ...
  3. 「離職率を下げる」 → 「給与改善」「業務改善」「関係改善」 → ...
模範解答を見る
  • 1: What — コストを構成要素で分解。数式分解の典型。
  • 2: Why — 「なぜ進まないか」の原因分解。
  • 3: How — 打ち手(手段)を並べている。

見分けのコツは 「頂点と子の関係を一言で表すと?」。What なら「は」「で構成される」、Why なら「なぜなら」、How なら「するためには」。

Exercise 2
応用

同じテーマで3型ツリーを描く

「営業成績を伸ばす」というテーマについて、What / Why / How の3型でそれぞれ第2階層までツリーを描いてください。終わったら、3つを並べて「どれが今の自分の議論に最適か」を1文で書く。

  1. What ツリー(構成要素分解)
  2. Why ツリー(原因分解 — 「なぜ伸び悩んでいるか」)
  3. How ツリー(手段分解)
  4. 3つを比較した結論:
模範解答を見る

例(営業成績):

What: 成績 = 案件数 × 受注率 × 単価
   - 案件数 → 新規/既存
   - 受注率 → 提案品質/価格競争力
   - 単価 → 商品Mix/値引

Why: なぜ伸び悩んでいる?
   - 案件数不足 → リード獲得不足/既存深掘り不足
   - 受注率低い → 競合に負ける/提案不足
   - 単価低い → 値引きしすぎ/低単価品偏り

How: どう伸ばす?
   - 案件数増 → リード生成/既存営業
   - 受注率上げ → 提案力/競合分析
   - 単価上げ → アップセル/値引方針

3つを比較した結論:「現状把握ができていない段階なら What で構造を、原因が見えているなら Why、打ち手を決めるなら How」のように使い分け。多くの場合、What → Why → How の順に進めると無理がない。
