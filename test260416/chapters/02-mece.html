Phase 1: Basics > Chapter 02

MECE

「漏れなく、ダブりなく」考える技術。MECEは全てのフレームワークの基礎であり、ここを外すと後続の章が全部ぐらつきます。60分で「切り口を選ぶ目」を養います。

所要時間: 60分 Phase: Basics 難易度: ★★☆

1. MECE とは何か

MECE(Mutually Exclusive, Collectively Exhaustive)は、対象を 「ダブりなく(ME)、漏れなく(CE)」 に分類する原則。McKinsey出身のバーバラ・ミントが体系化し、今やコンサルの共通言語です。

MECE / 漏れあり / ダブりあり の3パターン
MECEは「全体を覆う、重ならない、隙間がない」3点セット。3つのうち1つでも崩れると分析が信用されない。

MECEを満たすことで得られるメリットは、「考え漏れによる致命的な失敗を防ぐ」ことと、「ダブり議論で時間を浪費しない」ことの2つに集約されます。

例えば、新規顧客獲得の打ち手を考える時、「広告」「営業」だけ列挙すると 口コミ・代理店・既存顧客紹介 という大きな選択肢が漏れます。MECEで切れば、これらが自然に視野に入ってきます。

2. 「切り口」の選び方

MECEを実践する最大のコツは、切り口の選び方にあります。同じ対象でも、切り口次第でMECEになったり、ぐちゃぐちゃになったりします。

定石となる切り口パターン

  • 属性で切る — 性別・年齢・地域・職種など。客観的でMECE化しやすい王道。
  • 時系列で切る — 過去/現在/未来、短期/中期/長期。動的な変化を扱う時に強い。
  • 階層で切る — 戦略/戦術/オペレーション、経営層/管理職/現場。意思決定レイヤー別に。
  • プロセスで切る — 認知 から 検討 から 購入 から 利用 から 推奨。顧客行動を流れで追う時の鉄板。
  • 対立軸で切る — 内部/外部、定量/定性、コスト/価値。2軸で全体を覆う最小構成。
同じケーキを「切り方」で変える
同じ対象でも切り口で見え方が変わる。最初は複数の切り口を試して、一番説明力が強いものを選ぶ。

3. ビジネス定番フレームをMECE視点で再解釈

ビジネスフレームワークの多くは、実は 「先人が発見したMECEな切り口」 です。フレームを「丸暗記」するより、「これは何の切り口でMECEを実現しているか」 を理解すると応用が利きます。

フレーム分割対象MECEな切り口
3C市場環境Customer / Competitor / Company(主体の3者)
4PマーケミックスProduct / Price / Place / Promotion(売る要素)
SWOT戦略要因内部×プラス/マイナス、外部×プラス/マイナス(2×2)
5F業界構造新規参入/代替/買い手/売り手/競合(脅威の発生源)
PESTマクロ環境Politics / Economy / Society / Technology(外部の力)
顧客ファネル顧客行動認知から興味から比較から購入から推奨(時系列)
フレームの本質

「3Cを覚える」のではなく、「市場分析は3者で切るとMECEになりやすい」と理解すると、自分でフレームを作れるようになる。これがコンサル思考の入口。

4. MECE分解の実例:売上低下の原因

具体例で MECE を体感しましょう。「売上が低下している」というイシューを 売上 = 客数 × 単価 という因数分解からツリーで分けます。

売上低下の MECE 分解
graph TD
  A["売上低下"] --> B["顧客数 減少"]
  A --> C["客単価 下落"]
  B --> B1["新規顧客 獲得減"]
  B --> B2["既存顧客 離脱増"]
  C --> C1["割引/値下げ 拡大"]
  C --> C2["低価格商品 シフト"]
  B1 --> B1a["流入チャネル別"]
  B2 --> B2a["離脱理由別"]
「売上=客数×単価」の式そのものがMECE。最初の分岐がMECEなら、子ノードもMECEを保ちやすい。

このツリーで重要なのは、第1分岐(客数 vs 単価)が MECE であること。式から導かれているので、絶対に漏れもダブりもありません。式に落とせる時はぜひそうしましょう。

もっと詳しく: 「数式分解」と「概念分解」の使い分け

MECE分解は2タイプあります。

  • 数式分解: 売上=客数×単価、コスト=固定費+変動費、のように数式で割れるもの。MECEが保証される強力な分解。
  • 概念分解: 顧客=新規/既存、施策=オンライン/オフライン、のように概念で切るもの。便利だが「漏れ」「ダブり」の検証が必要。

どちらが正解ということはなく、数式に落とせる時は数式、無理な時は概念、と使い分けます。

5. よくある失敗パターン

MECEを意識し始めると、最初は失敗するのが普通です。よくあるパターンを知っておくと回避しやすい。

よくない

「Web」「SNS」「広告」

  • SNSは Web の一部
  • 広告は Web/SNS と重なる
  • Eメールが漏れている
よい

「自社サイト」「広告」「SNS」「Eメール」

  • 媒体ごとに排他
  • 主要4チャネルを網羅
  • その他で漏れも吸収可

失敗の典型 5 パターン

パターン例対策
1. 切り口が混在男性 / 学生 / 高所得者(性別と属性と所得が混ざる)1階層では切り口を1つに絞る
2. レベルが違うアジア / 東京 / 欧州(国/都市/地域の混在)同じ粒度で並べる
3. その他の濫用主要3社 / その他全部その他が大きい時は分解を続ける
4. 重複に気づかない若者 / 学生(若者と学生が大きく重なる)2要素の積集合を確認
5. 視点が偏る東京 / 名古屋 / 大阪(他都市が漏れ)地理MECEなら8地方区分
完璧主義の罠

MECEは絶対条件ではなく実用上MECEで十分。数学的な厳密 MECE を追求すると無限に時間がかかる。「主要なもので8割カバー、残り2割はその他で吸収」で実務は回る。

切り口を「候補3つ」から選ぶ実例

「日本の小売店」を MECE 分解するなら、次の3切り口が考えられます。

切り口例強み
業態別百貨店 / GMS / SM / CVS / SS / EC業界統計と整合、施策別議論しやすい
地域別北海道 / 東北 / 関東 / ... / 九州沖縄地理戦略を語る時に強い
規模別大企業 / 中堅 / 中小競争力分析に向く

どの切り口を選ぶかは 「これからしたい議論」 次第です。出店戦略を語るなら地域別、業界比較なら業態別、デジタル化を語るなら規模別。切り口は目的が決めると覚えてください。

分解すること自体は目的ではない。どう切ると、議論が前に進むかを考えなさい。 — バーバラ・ミント (『考える技術・書く技術』)
Exercise 1
入門

MECE違反を見つける

次の分類のうち、MECEに違反しているものを全て指摘し、修正案を出してください。

  1. 飲食店を「和食 / 洋食 / 中華 / イタリアン」で分類
  2. 社員を「20代 / 30代 / 40代 / 50代 / 役職者」で分類
  3. マーケ施策を「広告 / SEO / SNS / コンテンツ」で分類
  4. 顧客を「新規 / リピート」で分類
模範解答を見る
  • 1: ダブりあり — 「洋食」と「イタリアン」が重なる(イタリアンは洋食の一部)。修正案:「和食 / 洋食 / 中華 / その他」または「和食 / 中華 / イタリアン / フレンチ / アメリカン / その他洋食」など、レベルを揃える。
  • 2: 切り口混在 + ダブり — 年代と役職が混在し、20代の役職者がダブる。修正案: 年代軸と役職軸は別の表に分ける。
  • 3: ダブりあり — SEOはコンテンツ施策の一部、SNSも広告とコンテンツに跨る。修正案: チャネル軸(自然/有料)と形態軸(検索/SNS/広告)の2軸MECE。
  • 4: OK — 「初回購入かそれ以降か」で排他的かつ網羅的。実務でも頻出のシンプルなMECE。
Exercise 2
応用

自社の課題を3つの切り口でMECE分解する

あなたの職場で今、解決したい課題を1つ選び、3つの異なる切り口で MECE 分解を試みてください。最後に、最も議論を進めやすい切り口を1つ選び、選んだ理由を書いてください。

  1. 課題:
  2. 切り口A(例:プロセス別):
  3. 切り口B(例:組織別):
  4. 切り口C(例:時系列別):
  5. 選んだ切り口と理由:
模範解答を見る

正解はありませんが、「離職率が高い」を例にすると:

課題: 離職率が業界平均より5pt高い

切り口A (タイミング別):
  入社1年以内 / 1-3年 / 3-5年 / 5年以上

切り口B (理由別):
  給与 / 業務内容 / 人間関係 / キャリア / 生活環境

切り口C (組織別):
  営業 / 開発 / バックオフィス / 経営層

選んだ: B (理由別)
理由: 打ち手を考える時、原因別の方が施策に直結するため。
ただし B で大きな塊が見えたら、その中で A や C で再分解する。

3つ書く理由は、「最初に思いついた切り口」が最良とは限らないから。複数試して比べる癖が、コンサル思考の核心です。
