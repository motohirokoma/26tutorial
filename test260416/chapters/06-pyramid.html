ピラミッド構造

バーバラ・ミントが体系化した「結論ファースト」の伝え方。SCQAでつかみを作り、メッセージと根拠を階層化する。60分でプレゼン資料の骨格が作れるようになります。

所要時間: 60分 Phase: Core 難易度: ★★★

1. ピラミッド原則とは

ピラミッド原則は、伝えたい内容を頂点に 1つのメインメッセージ を置き、それを支える根拠を階層的に並べる構造化技法。McKinsey でバーバラ・ミントが体系化し、今やコンサル業界の標準です。

ピラミッド構造の3階層
頂点1つ・中間3つ・底辺9つの「1-3-9」構造が美しいピラミッドの型。

ピラミッドの本質は「結論を先に出し、根拠を後ろに置く」という順序にあります。多くの日本人ビジネスパーソンが苦手な構造ですが、慣れれば最強の伝達ツールです。

2. SCQA — つかみの定型

ピラミッドの頂点(メインメッセージ)を提示する前に、聞き手の関心を引く「つかみ」が必要。SCQA はその定型です。

段意味例(離職率改善)
S: Situation聞き手が知っている前提当社の離職率は業界平均の1.5倍
C: Complicationそこに生じている問題採用コストが年間2億円に達し、採用側の人手も逼迫
Q: Question聞き手の頭に湧く問い「どうやって離職率を下げるか?」
A: Answer答え = メインメッセージ3つの施策で2年で業界平均水準に下げる
SCQAの順序の意味

聞き手は S と C を聞いた瞬間に Q が頭に浮かぶ。Q を口に出して確認した後で A を出すことで、「自分の問いに答えてくれた」という納得感が生まれる。

3. 結論ファースト vs 起承転結

日本の伝統的「起承転結」と、コンサル流「結論ファースト」は構造が真逆です。

起承転結

結論が最後に来る

  • 「起 → 承 → 転 → 結」の順
  • 聞き手は最後まで結論不明
  • 意思決定者は途中で離脱しがち
  • 物語には強いがビジネスには弱い
結論ファースト

結論が最初に来る

  • 「結論 → 根拠 → 詳細」の順
  • 30秒で結論が伝わる
  • 意思決定者が判断しやすい
  • ビジネス文脈に最適

同じ内容を2方式で書き換えた例

起承転結結論ファースト
「先月、競合A社が値下げを発表しました。当社も追随を検討しましたが、利益率が圧迫されます。一方、当社の独自技術は競合より3倍長持ちします。よって、値下げではなく『長寿命をアピールするマーケ強化』を提案します。」 「結論: 値下げではなくマーケ強化を提案します。理由: ① 利益率を守れる ② 当社の独自強み(寿命3倍)を活かせる ③ 価格競争に巻き込まれない。具体策は3つあり、まず...」

4. ピラミッドの2型 — 演繹型 / 帰納型

ピラミッドの「メインメッセージ ← キーメッセージ」のつながり方には2型あります(Ch.03 の演繹/帰納と直結)。

演繹型ピラミッド (大前提→結論)
graph TD
  C["結論: 当社は新興市場Xに参入すべき"]
  C --> P1["大前提: 市場成長率15%以上の業界に投資すれば3年でROI 30%超"]
  C --> P2["小前提: 市場Xの成長率は18%"]
  C --> P3["導出: よって参入により ROI 30% 超を期待"]
3要素が三段論法で結論を導く。論理的に強いが、聞き手が大前提に同意していないと崩れる。
帰納型ピラミッド (並列の根拠→結論)
graph TD
  C["結論: 当社は新興市場Xに参入すべき"]
  C --> R1["理由1: 市場が魅力的(規模10兆/成長15%)"]
  C --> R2["理由2: 当社に勝ち筋(独自技術)"]
  C --> R3["理由3: 投資が回収できる(5年でROI 25%)"]
複数の独立した根拠を並列で並べる。実務では帰納型の方が圧倒的に頻出。
型強み使う場面
演繹型論理が締まる、反論されにくい規制対応、判断ロジックが固い時
帰納型柔軟、複数視点で説得力戦略提案、複合判断、ほぼ全業務

5. キーメッセージの3要件

ピラミッド第2層のキーメッセージには3要件があります。

  • 1. メインを支える — そのキーが正しいなら、メインも一歩近づく関係。
  • 2. キー間がMECE — キー同士で漏れもダブりもない(Ch.02のMECEがここで効く)。
  • 3. メッセージとして読める — 「市場規模」ではなく「市場規模は十分」のように、主張の形にする。
もっと詳しく: 「名詞」と「メッセージ」の違い

初学者が陥りがちな失敗が「キーが名詞になっている」こと。

  • 名詞だけ: 「市場規模」「競合」「コスト」 → 何が言いたいか不明
  • メッセージ化: 「市場規模は十分大きい」「主要競合は3社のみ」「コストは2年で回収可」 → 結論が分かる

スライドのタイトルは 必ずメッセージ形式 で書く、と覚えてください。

キーメッセージのパターン

キーメッセージの並べ方は、テーマに応じて使い分けます。

時系列順
短期 / 中期 / 長期
構造順
市場 / 当社 / 競合 (3C)
影響順
大→中→小
プロセス順
準備 → 実行 → 検証

7. ピラミッドからスライドへ

ピラミッドが完成すれば、それは ほぼスライドの目次 です。1メッセージ = 1スライドの原則に従えば、自動的に資料骨格ができます。

ピラミッドの階層スライドの位置づけ
SCQA表紙 + エグゼクティブサマリー
メインメッセージ結論ページ
キーメッセージ × 3章の見出し3つ
根拠データ × 9各章の本文スライド3枚ずつ

つまり 10〜15枚程度のコンサル提案資料 は、ピラミッド1枚分の構造で十分カバーできます。

良いピラミッドは、聞き手が30秒で結論を理解し、5分で根拠を理解し、30分で全体を理解できる。 — バーバラ・ミント
議論中のラフピラミッド
完成形にする前は、こんな手描きスケッチで十分。議論で更新しながら磨く。
Exercise 1
入門

SCQA を分解する

次の話を S / C / Q / A に分解してください。

「コロナ後、当社のオフィス出社率は40%程度で安定しています。一方、新入社員の3年定着率がコロナ前の85%から70%に低下しました。これでは採用投資が回収できません。原因を調べたところ『先輩との偶発的な雑談機会』が激減していました。よって、月2回の対面ハドルを必須化することを提案します。」

模範解答を見る
  • S (Situation): コロナ後、出社率は40%で安定している(聞き手も知っている前提)
  • C (Complication): 新入社員の3年定着率が85%→70%に低下、採用投資回収が困難
  • Q (Question): なぜ低下した?(聞き手の頭に湧く問い)
  • A (Answer): 「偶発的雑談の減少」が原因 → 月2回の対面ハドル必須化を提案

SCQAは S と C で「ヤバい!」と感じてもらい、Q が口から出る寸前に A を出すのが理想形です。

Exercise 2
応用

自分の提案でピラミッドを組む

あなたが今、上司や顧客に提案したい何かを1つ選び、SCQA + ピラミッド(メイン + キー3本)を作ってください。

  1. SCQA(各1文ずつ):
  2. メインメッセージ(結論を1文で):
  3. キーメッセージ × 3(各1文、メッセージ形式):
  4. 各キーを支える根拠 × 1〜2(箇条書き):

完成したら、メインだけ見せて伝わるか・キーだけ見せて伝わるかを声に出してチェック。

模範解答を見る

例(週次定例の廃止提案):

S: 当チームは毎週水曜10時に定例会議を持っている
C: 直近6回の定例で意思決定が0件、平均所要時間60分(月4時間×8人=32人時)
Q: この会議は本当に必要か?
A: 定例を廃止し、Slack上の非同期共有 + 月1ハドルに移行することを提案

メイン: 週次定例を廃止して非同期+月次ハドルに移行すべき

キー1: 定例の主目的「進捗共有」は非同期で代替可能
   - 直近6回の議事録の80%が進捗報告のみ
   - 共有すべき内容は事前テキスト化が可能

キー2: 残る目的「議論」は月1で十分
   - 議論ネタが出るのは月平均1.2件
   - 月1ハドル1時間あれば対応可

キー3: 廃止により月32人時を創出できる
   - 8人×60分×4週 = 32時間/月
   - 別の高付加価値業務に充当可能

各キーが 「主張の形」 になっているか確認すること。「進捗共有について」のように名詞で止めると弱いピラミッドになります。
