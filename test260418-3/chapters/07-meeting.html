会議と議事録

AI議事録が議事録を書く時代。じゃあ新人は何をするのか。"何が決まったか"の本質を掴む側に回る。

所要時間: 30min Phase: Core 難易度: 初級

会議は "合意形成の場"

多くの新人が勘違いしている。会議は "話し合う場" ではない。会議とは、合意形成の場である。雑談で終わる会議は、ただのコストだ。

ただし、会議には4つのタイプがあり、それぞれ目的も、君の振る舞いも、議事録の粒度も違う。最初にこの見取り図を持つと、どの会議でも迷わない。

タイプ目的新人の振る舞い議事録の粒度
意思決定Yes/Noや方針を確定する黙って聴く。聞かれたら即答細かく(決定と根拠)
情報共有状況をチームで揃えるメモ中心。質問は最後にまとめて要点のみ
ブレストアイデアを広げる遠慮せず発言。質の前に量全発言を拾う
定例報告進捗確認・リズム維持自分の進捗は簡潔に事前準備テンプレに沿って
4つの会議を2軸でマッピング
同じ "会議" という言葉でも、振る舞いは全く違う。
この章のゴール

AI議事録が普及した時代に、新人が会議で果たすべき役割を再定義する。議事録を書く力と、決定事項を掴む目、そして 発言する勇気。この3つをセットで身につける。

新人が会議でやること

「会議に呼ばれたけど、何すればいいの?」— 最初はみんなこうなる。結論から言うと、入室・着席・傾聴・メモが全てだ。発言は二の次。

  • 入室は5分前、末席に座る — 時間厳守は最低ライン。席は 下座(ドア側)。上司が「こっち座れ」と言うまで勝手に上座に行かない
  • 呼ばれない限り黙る — 特に意思決定会議では、新人の発言は求められていないことが多い。"黙って聴く"は立派な仕事
  • 聞かれたら即答 — 「〇〇さんはどう思う?」と振られたら3秒以内に口を開く。答えが分からなければ「分かりません、確認します」でOK
  • メモは "全部書かない" — 議事録担当でも、逐語記録はAI任せ。君は 決定事項・アクション・宿題 の3つに集中する
  • 退室後すぐに清書 — 記憶が鮮明なうちに、15分以内に議事録をまとめる。時間が経つほど文脈は蒸発する
会議中の新人の動き(シーケンス)
sequenceDiagram participant S as ["新人 (自分)"] participant L as ["上司・先輩"] participant C as ["取引先・他部署"] L->>S: ["会議開始。議題共有"] Note over S: ["まずメモ: 議題/参加者"] L->>C: ["本題の議論"] C->>L: ["提案・質問"] Note over S: ["決定事項のみ書く"] L->>S: ["〇〇さん、進捗は?"] S->>L: ["結論先出しで30秒"] L->>C: ["では方向性は△△で"] Note over S: ["決定を朱書き。誰が・何を・いつまで"] L->>S: ["議事録お願い"] S->>L: ["承知しました。30分以内に"]
新人の動きは "聴く→メモ→聞かれたら即答→議事録" の4ループ。
もっと詳しく: オンライン会議で特に気をつける5つ
  1. カメラはオン — 顔が見えないと "いないのと同じ"。発言のタイミングも掴みづらい
  2. ミュート徹底 — 発言しない時は必ずミュート。家のチャイム・洗濯機の音が入ると致命的
  3. 名前表示を整える — 「iPhone」「ゲスト」で入らない。「部署 姓名」で
  4. チャットは議事録の敵 — 盛り上がると流れて消える。決定事項は口頭で復唱させる
  5. AI議事録ボットの扱い — 参加者全員の同意を得てから起動。機密会議では使わない判断も必要

議事録の基本構造

議事録の目的はただ一つ。あとで読む人が「何が決まったか」を3秒で理解できること。その目的さえ押さえれば、構造は自然に決まる。

議事録テンプレート(汎用)
濃いピンク = 必ず自分で確認する箇所。薄いグレー = AI任せでOK。
セクション記入項目書き方のコツ
Header日時/場所/参加者/議題参加者は敬称揃える。肩書は初出のみ
決定事項「〇〇は△△とする」必ず体言止めか断定文。曖昧語NG
議論要旨発言者 + 要点1発言1行。感情語は削る
アクションアイテム担当者・内容・期限必ず3点揃える。1つ欠けたら確認
黄金ルール: WWW

アクションは必ず Who(誰が) / What(何を) / When(いつまでに) の3つが揃っているか確認。「検討します」「追って連絡」は ゴミ。誰の宿題でもない = 誰もやらない。

AI議事録との付き合い方

AI議事録は、もう "あるのが普通" の時代だ。敵対するのは無駄。ただし、丸投げすると必ず事故る。君の役割は、AIの穴を埋める人間になること。

AI議事録

得意なこと

  • 全発言の逐語記録
  • キーワード検索ができる
  • あなたは会議に集中できる
  • 人物別に発言量を可視化
  • 多言語の自動翻訳
AI議事録

苦手なこと

  • 要約の温度感(本気か冗談か)
  • 決定と "雑談" の区別
  • ホワイトボード・図の文脈
  • 機密・個人情報の扱い
  • 誰が・何を・いつまでにの抽出

つまり、AI議事録 + 自分の"要点メモ"の併用が正解。AIが全発言を拾い、あなたは決定事項・アクションだけを書く。この ハイブリッドが現時点での最適解だ。

議事録作業の内訳(新人の実務)
もっと詳しく: AI議事録を使うときの注意点
  • 開始前に全員の同意 — 録音相当の行為。無断は NG。社内ルールも確認
  • 機密情報の境界 — M&A・人事・顧客固有名詞などは、そもそもAIに投げない選択肢もある
  • 要約を鵜呑みにしない — AIの要約は "平均的に正しい"。重要会議では自分で聴き直す
  • 保存場所を統一 — AI議事録 / 自分のメモ / 清書版 が散らばると検索できない。1箇所に集約
  • 退職・異動時の継承 — AI議事録のログも引き継ぎ対象。アカウント管理を意識する

総務省の「AI利用者の留意点」も一読しておくと、社内説明で役に立つ。

公式ドキュメント

AIを業務利用する上での倫理・法的論点は、経済産業省や総務省が整理している。会議音声の取り扱いは個人情報保護法にも絡むので目を通しておこう。 経済産業省: 生成AI関連政策 →

発言の勇気

最後に、一番重要な話。場の空気を読んで発言することは、AIに代替できない。君がその場にいる意味は、最終的にここに集約される。

とはいえ、いきなり意見を言えとは言わない。新人の発言には 3つの型 がある:

  • 質問 — 「すみません、〇〇について確認ですが」で始める。恥ずかしがらない。分からないまま帰る方が罪
  • 確認 — 「つまり△△という理解で合っていますか」。これを言えるだけで、"聴いていた人"になる
  • 決定事項の復唱 — 「決まったのは、□□さんが〇〇を来週までに、で合っていますか」。議事録担当の神ムーブ
発言を起こす4ステップ
1
違和感
「あれ?」と感じる瞬間を逃さない
2
整理
頭の中で30秒、結論を一文にまとめる
3
タイミング
議論の切れ目、上司のひと呼吸を狙う
4
発言
定型句 + 結論 + 根拠の順で短く
新人の発言は、内容の質より発言したという事実が評価される。最初の半年で「あの子はちゃんと参加している」と認知されれば、後の1年がずっと楽になる。
Exercise 1
入門

15分会議のメモから "3行の決定事項" を抽出する

以下は、新商品企画の15分会議のメモ(抜粋)です。決定事項3つを、箇条書き3行で書き出してください。

・Aさん「新商品の発売は6月がいいのでは」
・Bさん「在庫を見ると6月は厳しい、7月にしたい」
・上司「では7月第2週で。最終案は来週金曜までに」
・Aさん「パッケージ色は赤2案 vs 白2案、どちらで?」
・Bさん「コーポレートに合わせて白基調で」
・上司「白基調でいく。サンプルは水曜までに」
・Aさん「予算は300万で収まりそう」
・上司「OK、予算300万で確定」
模範解答を見る

決定事項3行はこうなる:

【決定事項】
1. 発売日は7月第2週(最終案は Aさんが来週金曜までに提出)
2. パッケージは白基調(Aさんがサンプルを水曜までに提示)
3. 予算は300万円で確定

ポイント:

  • 議論の経緯は省く。結論だけ書く
  • 誰が・何を・いつまでに(WWW)を必ず付ける
  • 「〜がいい」「〜のでは」など意見は全部削る。"決まったこと"だけ
Exercise 2
応用

AI議事録の要約に潜む "3つの抜け" を見つける

下記は、AI議事録が自動生成した要約です。しかしこの要約は、会議の重要な情報を3つ落としています。実際の発言録(下段)と照らし合わせ、抜けている3点を特定し、補ってください。

【AI議事録の要約】

プロジェクトZの進捗について議論した。
スケジュールに遅れが出ており、追加リソースが
検討された。品質面の懸念もあった。
次回は来週予定。

【実際の発言(抜粋)】

・上司「プロジェクトZ、2週間遅れてる。山田さん、立て直せる?」
・山田「増員1名と、機能Xの後ろ倒しで間に合わせます」
・上司「じゃあ機能Xは次フェーズに。増員は人事の佐藤さんに今週中に依頼」
・鈴木「品質の件、今のバグ数だとリリース基準未達です」
・上司「バグゼロ目標。鈴木さんが毎週金曜に進捗レポート」
・上司「次回は来週水曜15時、同会議室で」
模範解答を見る

AI要約が落としている3点は次の通り:

#抜け補うべき内容
1アクション担当者・期限(WWW)が全消滅「増員依頼:山田→人事佐藤、今週中」「品質レポート:鈴木、毎週金曜」
2具体的な決定事項(機能Xの後ろ倒し)「機能Xは次フェーズに移動することを決定」
3次回の日時・場所「次回:来週水曜15時、同会議室」(「来週予定」では足りない)

教訓: AI議事録は "雰囲気はあってる要約" を出すのが得意だが、具体的な固有名詞・期限・数値をよく落とす。そこを埋めるのが新人の仕事。ここに価値がある。
