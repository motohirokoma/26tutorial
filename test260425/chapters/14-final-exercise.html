シナリオ 1 — 大手生命保険会社の戦略相談

Q1
統合問題

「保険ビジネスがAIで根本的に変わる」と聞いた経営層への戦略提案

顧客: 大手生命保険会社(売上3兆円、契約者数1,000万人、エージェント1万名)
状況: 経営層から「AIで保険ビジネスが根本的に変わると聞いた。どう対応すべきか?」と相談を受けた。
あなたの課題: 以下5点について構造化して回答してください。

  1. 2029年の保険業界がAIでどう変わるか(マクロ・技術・業界の3層で)
  2. 同社が直面する3つの主要リスク
  3. 3つの戦略オプション(攻め・守り・両立)
  4. 推奨案 + 理由
  5. 初年度に取り組むべきPoC 3件と、その規模・期待効果
模範解答を見る

① 2029年の保険業界

  • マクロ: 少子高齢化、健康寿命延伸、Z世代の保険離れ、気候変動による災害頻発
  • 技術: テレマティクス + ウェアラブル + ゲノムでパーソナライズ保険、生成AI による加入時の対話、フィジカルAI による介護関連保険ニーズ
  • 業界: 大手3社の寡占構造に Apple Insurance や AIネイティブ新興保険(Lemonade型)が参入、エージェント業務がAI支援に純化

② 主要リスク3つ

  1. 個別最適化に失敗する(画一商品で価格競争に巻き込まれる)
  2. エージェント業務の AI シフトに失敗(コスト構造改革できず競争劣後)
  3. 新規領域(健康・予防・介護)で他社・異業種に取られる

③ 戦略オプション3つ

  • 守り: 既存商品+販売プロセスのAI最適化、コスト圧縮で価格競争力維持
  • 攻め: ヘルスケア新領域(予防・健康データ事業)へ進出、保険外サービス事業化
  • 両立: 守りで原資を作り、攻めに3年で1,000億円投資。エージェント業務を AI 支援で30%効率化、ヘルスケア事業で年500億円規模を狙う

④ 推奨案: 両立。理由: 守りだけでは規模ある新興にやられる、攻めだけでは既存事業の崩壊に耐えられない。3兆円規模の体力を活かし両立投資が現実解。

⑤ 初年度PoC 3件

  1. エージェント支援AI(顧客マッチング+提案書生成): 3億円、エージェント1,000名で実証。提案件数 +30%、成約率 +15%
  2. ウェアラブル連動 健康促進型保険: 4億円、ベンダー連携で会員獲得 5万人、年間データ事業化の基盤
  3. クレーム自動審査AI(画像 + 自然言語): 2億円、軽微案件の自動処理80%、審査コスト40%削減

シナリオ 2 — 中堅電子部品メーカーのDX転換

Q2
統合問題

「中国メーカーの追い上げ」に直面した中堅企業の戦略

顧客: 中堅電子部品メーカー(売上500億円、工場3箇所、創業60年)
状況: 中国メーカーから30%安い類似製品が出始め、主要顧客から値下げ圧力。「3年で営業赤字」と試算。
あなたの課題: 以下を考えてください。

  1. 同社が活かせる「眠れる資産」(60年の蓄積から)を3つ仮説
  2. その資産を起点とした、AI/DXによる差別化戦略
  3. 3年ロードマップと投資額/期待効果
模範解答を見る

眠れる資産3つ

  1. 60年分の不良発生データと修正ノウハウ → 「製造プロセスの暗黙知」
  2. 顧客との品質要求・要件議事録 → 「顧客業界別の品質基準データ」
  3. 独自の精密加工技術と治具設計 → 「希少な技能と装置設計知識」

差別化戦略

  • 方向性: 「価格で戦わない」ニッチ高品質ポジション。中国メーカーがコスト重視、日本大手が大量生産に注力する間で「少量多品種で品質要件最厳格な領域」(医療機器、航空宇宙、半導体製造装置部品)に特化
  • AI/DX 軸: 60年の不良データを学習した「品質予測AI」、独自加工技術の「AI 設計支援エンジン」、顧客との品質基準を結晶化した「品質コンサルサービス」
  • 横展開: 蓄積した品質予測モデルを、同業他社向けにSaaS提供する新規事業

3年ロードマップ

  • Year 1: 不良データ集約 + 品質予測AI構築、医療機器分野で実績作り(投資3億円)
  • Year 2: 航空宇宙・半導体製造装置部品で受注拡大、品質コンサルサービス試行(投資3億円)
  • Year 3: 同業他社向け SaaS 化、年5億円新規売上の立ち上げ(投資2億円)
  • 3年累計: 投資8億円、新規売上累計15億円(SaaSは年30億円規模が次フェーズ)、既存事業の利益率改善で年20億円

シナリオ 3 — 政府機関のサイバー防衛強化

Q3
統合問題

政府機関への商業AIサイバー防衛提案

顧客: ある国の政府機関(公開情報のみ前提)
状況: 国家レベルのサイバー攻撃に対する AI 防衛強化を検討中、商業AI企業からの提案を募集。
あなたの課題:

  1. 提案する技術構成(AIによる脅威検知・対応自動化等)
  2. 商業AIスタートアップが政府案件で押さえるべき特殊論点5つ
  3. 提案書の構成(セクション構成と各章の主旨)
模範解答を見る

技術構成

  • ① 異常通信検知 AI(教師なし学習・異常検知ベース)
  • ② LLM による脅威レポート自動分析(膨大な脅威情報の構造化)
  • ③ SOAR + AI による初動対応自動化
  • ④ 脆弱性スキャン AI(コードベース全数解析)
  • ⑤ ディープフェイク検出 + ソーシャルエンジニアリング対策
  • 主要連携: NIST CSF、ISO 27001、CIS Controls

政府案件で押さえる特殊論点5つ

  1. セキュリティクリアランス: 担当者・データ取扱者の身元確認
  2. データ主権: 国内データセンター必須、海外API利用は機微情報除外
  3. 調達制度: 政府特有の調達ルール、技術評価方法、価格交渉プロセス
  4. 第三者監査: 独立第三者による監査受け入れの体制
  5. 事業継続: 戦時・大規模災害時の継続稼働保証

提案書構成

  1. エグゼクティブサマリー(1ページ)
  2. 脅威環境分析(国家レベル攻撃の最新動向)
  3. 技術ソリューション(5要素の詳細)
  4. セキュリティ・データ主権対応
  5. 運用体制とSLA
  6. 段階的導入ロードマップ
  7. コストと期待効果
  8. リスクと前提条件
  9. 会社紹介・実績(該当範囲のみ公開可情報)
  10. Appendix(技術詳細・参考資料)

シナリオ 4 — 業界横断: SaaSスタートアップへの戦略助言

Q4
統合問題

「自社プロダクトをエージェント時代に対応させたい」相談

顧客: 中堅 BtoB SaaS スタートアップ(売上70億円、社員200名、人事SaaS)
状況: 「LLM/エージェント時代の到来で、既存 SaaS のUI が通用しなくなる」という危機感。1年で何をすべきか?
あなたの課題:

  1. SaaSビジネスがエージェント時代にどう変わるか(仮説)
  2. 同社が1年で取るべき3つの戦略アクション
  3. 注意すべき「Agent 過剰設計」の罠
模範解答を見る

SaaS のエージェント時代変化

  • UI主体 → API主体: ユーザーが画面を操作するのではなく、エージェントが API を叩く時代に
  • メニュー → 対話: 「画面のメニューを覚える」必要なくなり、自然言語で指示
  • ワークフロー → 自律実行: 複数ステップの業務を AI が一気に実行
  • 競争軸: 「機能の多さ」より「データの独自性」+「エージェントへの最適化」

1年でとるべき3つの戦略アクション

  1. Agent-Ready API の整備: 自社SaaSの全機能を構造化されたAPIで公開、Anthropic MCP / OpenAI Function Callingに対応
  2. 「自社AI Agent」の開発: 自社SaaS を中心としたタスクを自動化するAgent。差別化のための主役機能
  3. データ独自性の強化: 自社が保有する顧客データの独自性を価値化、他社AI Agentから「使いたい」と思わせるデータレイヤーを構築

Agent 過剰設計の罠

  • 全部 Agent に任せる:Anthropic 「Building Effective Agents」によれば、ワークフロー(プログラム的フロー)で済むなら Agent にしないのが鉄則。Agent は最終手段
  • 監視・制御の不足: Agent が無限ループに陥る、誤った操作をするリスク。サンドボックス・操作承認・監査ログを設計
  • UX の崩壊: 自然言語のみのUIでは、ユーザーが何を操作したか分からない混乱。状況説明・進捗表示が必須
  • コストの暴走: Agent は何度も LLM を呼ぶ、トークン消費が予算を破壊する。コスト監視と上限設定

シナリオ 5 — 自分のキャリア戦略

Q5
自分への問い

あなた自身が AI/DX 時代に何を強化するか

本コースで触れた多様なスキル(コンサル思考・LLM・ML・フィジカルAI・業界知識・規制・営業)の中で、あなたが今後3年で重点強化したい領域を3つ選び、それぞれのアクションプランを書いてください。

  1. 強化領域 (3つ)
  2. 各領域での3年後の到達目標
  3. 各領域での具体的な学習・実践計画
  4. これらを自身のキャリアにどう接続するか
参考解答(あくまで例)

例: ある若手社員(元営業バックグラウンド)の場合

強化領域3つ

  1. LLM/RAG の実装力(技術側の理解)
  2. 戦略コンサル思考(イシュー設定・構造化)
  3. 製造業の業界知識(主要顧客がここに集中するため)

3年後の到達目標

  • ① RAG システムを社内顧客向けにフルスタックで設計・指導できる
  • ② 案件の初回ヒアリングで、その場で30秒のイシュー定義ができる
  • ③ 自動車・半導体・電子部品 各業界の経営層と「業界の未来」を議論できる

学習・実践計画

  • ① 平日朝1時間、Anthropic/OpenAIドキュメント読書 + 週末に手を動かす。社内案件で2-3件は技術リードを買って出る
  • ② Ch02 の書籍3冊を再読、ブログで毎週1ケース分析を書く、上司・先輩との壁打ちを月2回
  • ③ 業界紙2誌定期購読、四半期に1度業界カンファレンス参加、業界経験者にインタビュー(月1人)

キャリア接続

  • 3年後にエンタープライズアカウント担当(製造業)を希望、技術 + 戦略 + 業界の3点を持つ唯一の人材を目指す
  • 5年後には自分の業界専門ポジションを確立、独立 or 役員候補のキャリアを開く

共通ポイント: 「全部やる」では伸びない。3つに絞る、定量的な到達目標、毎日・毎週のリズムを設計すること。
