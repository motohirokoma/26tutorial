Webで調べる魔法

秘書に「ちょっと調べておいて」と頼む感覚で、Claudeにインターネットを往復してもらおう。

所要時間: 45min Phase: Core 難易度: 初級
ようこそ

ここまでで、Claudeはあなたのパソコンの中を自由に行き来できるようになりました。この章では、ついに外の世界（インターネット）へ出る魔法を覚えます。難しい仕組みは要りません。「調べておいて」と頼むだけです。焦らずに進めましょう。

2026年1月時点の情報

この章は Claude Code 2026年1月版を前提に書かれています。Web関連のツール名（WebFetch / WebSearch）は安定していますが、細かい振る舞いは今後改良される可能性があります。最新の挙動は 公式ドキュメント で確認してください。

Claudeがインターネットとつながる、ってどういうこと？

これまでのClaudeは、あなたのパソコンの中だけを動き回る新入社員でした。ファイルを読んで、書いて、直して。でも「最近の天気って？」「あの本の発売日って？」には答えられませんでした。なぜなら、知識カットオフより後の出来事は知らないし、何より勝手にネットを見に行かない設計だからです。

この章で覚える2つの道具を使うと、Claudeが安全な手続きを踏んでインターネットに出かけます。イメージは「秘書にお使いを頼む」感覚。あなたが命令すると、秘書が外に出て、情報を持ち帰って、手元で要約してくれる。それだけのことです。

あなた → Claude → インターネットの流れ
あなたが頼む → Claudeが取りに行く → 要約して返す。この4ステップだけ。

道具は全部で2つ。名前だけ覚えてください。「WebFetch（ウェブフェッチ）」と「WebSearch（ウェブサーチ）」です。

  • WebFetch — URLを指定して「このページを取ってきて」と頼む魔法。ページのHTMLを取得して、中身を読んでくれる。
  • WebSearch — 検索したいキーワードを渡して「探してきて」と頼む魔法。検索エンジンを叩いて、ヒットしたページの要点を拾ってくる。

2つの違いはこの後じっくり見ます。今は「URL知ってるならFetch、探したいならSearch」くらいのぼんやり感覚でOK。ここで戸惑っても大丈夫。もう一度読めば必ず掴めます。

WebFetch — 「このページを読んでおいて」

まずはシンプルな方から。WebFetchは「このURLを開いて、中身を読んで、ポイントを教えて」とお願いする魔法です。あなたが知っている特定のページ――たとえば公式ドキュメント、ニュース記事、ブログ――を要約したい時に大活躍します。

WebFetchの実プロンプト例
https://docs.claude.com/en/docs/claude-code/overview
このページを読んで、Claude Codeで何ができるかを
箇条書き5つでまとめて。
URLをそのまま貼る → 何をしてほしいかを1行で。これだけ。

実際にやってみましょう。Claude Codeのターミナルで、上のプロンプトをそのまま打ってEnter。数秒後、ページの要点が手元に返ってきます。これが最初の「小さな成功」です。

動いた？ 自分を褒めてOK

いま、あなたはClaudeにWebページを読ませて要約させることができました。これはExcelの達人でも、ブラウザを30個開いて情報収集していた頃の自分でも、絶対に到達できなかった領域です。拍手。

WebFetchが得意なこと / 苦手なこと

得意苦手
公式ドキュメントの要約 ログインが必要なページ
ブログ記事の要点抽出 JavaScriptで動的に出るコンテンツ
ニュース記事の翻訳・要約 PDFや画像中心のページ
仕様書の該当箇所を探す 有料会員限定の記事
もっと詳しく: WebFetchが裏で何をしてるか

WebFetchは裏側で3つのことをしています。

  1. 指定されたURLへアクセス（あなたがブラウザでページを開くのと同じ）
  2. 返ってきたHTMLから、広告やナビゲーションを除いた本文を抜き出す
  3. その本文をClaudeが読み、あなたの指示に従って要約・翻訳・抽出する

つまり「ブラウザで開いて、読み取って、まとめる」をワンコマンドでやる感じ。途中で失敗したら正直に「取れませんでした」と教えてくれます。

WebSearch — 「探してきて」

もう片方の道具、WebSearchは「URLまでは知らないけど、これについて調べて」と頼む時に使います。あなたが普段Googleで検索するのと同じ感覚。違いは、結果を自分で読まなくていいこと。Claudeが複数のページを見て、まとめてくれます。

WebSearchの実プロンプト例
2026年に話題になっているAIアシスタントツールを
WebSearchで探して、3つピックアップして特徴を教えて。
キーワード + 「どうまとめてほしいか」の2点で伝わる。

ここでのコツは、「こう調べて」+「こうまとめて」の2段セットで頼むこと。単に「調べて」だと、Claudeは10個リンクを返してきて、結局あなたが読むはめになります。

WebFetch と WebSearch、いつどっち？

WebFetch

URLを知っている時

  • 公式ドキュメントを要約してほしい
  • このニュース記事の論点を教えて
  • このブログの手順をまとめて

合言葉: 「このページ読んで」

WebSearch

キーワードしか手元にない時

  • 〇〇のおすすめツールを3つ教えて
  • 最近の〇〇業界のトレンドを調べて
  • 〇〇と〇〇の違いを比較して

合言葉: 「ちょっと探してきて」

この2つは対立するものではなく、コンビで使います。実務のよくあるパターンは：

WebSearch → WebFetch の連続技
graph LR
  A["キーワードで検索"] --> B["WebSearch"]
  B --> C["ヒットした記事のURL"]
  C --> D["WebFetch"]
  D --> E["気になる記事だけ精読"]
まず広く探して、良さそうな記事だけ深く読む。人間の調べ物と同じ。

「自分で検索してからまとめて」と頼む魔法

ここが今日のメインです。これまでの2つを組み合わせて、Claudeに「調査員」になってもらいましょう。あなたがただ1文で依頼すれば、Claudeが検索 → 記事読み込み → 比較 → 要約まで自分で判断して進めてくれます。

魔法のプロンプトの型
【テーマ】について、自分でWebSearchして情報を集めた後、
良さそうな記事を2〜3つWebFetchで精読して、
【こういう形式】でまとめてほしい。
「調べて」→「読んで」→「まとめて」の3段ぶち抜き指示。

例を見てみましょう。あなたが「最近人気のキャンプ道具を知りたい」と思ったら：

実プロンプト例（キャンプ道具調査）
2026年に人気のソロキャンプ道具について、自分でWebSearchして、
評価が高い記事を2〜3つWebFetchで読み込んだ上で、
初心者におすすめのアイテムを5つ、
「名前・用途・だいたいの値段・選ぶ理由」の形で表にしてまとめて。

これだけで、Claudeは勝手に検索し、記事を読み、表にまとめて返してくれます。所要時間30秒。自分でやったら1時間コースです。

30秒
Claudeの調査時間
↓ 自力の1/120
5記事
同時に参照できる情報源
↑ 並行処理の強み
1個

もっと伝わるプロンプトのコツ

Ch03で習った「お願いする魔法の型」がここでも生きます。調査系のプロンプトに3つの要素を入れると、返ってくる質が跳ね上がります。

  • ① ゴール — 何のために調べるか。「プレゼン資料用」「買い物の参考」「論文の下調べ」など。
  • ② 形式 — どう出してほしいか。「表で」「箇条書き5つ」「3段落以内」「Markdownで」など。
  • ③ 制約 — やらなくていいこと・避けたいこと。「英語記事は除外」「2024年以降の情報のみ」「初心者向けに」など。
もっと詳しく: なぜ「形式」を指定すると精度が上がるか

「表で」と言われると、Claudeは無意識に「列名は何にしよう」「この情報はどこに入れよう」と構造化しながら書きます。形式の指示は、Claudeに考える枠組みを与えるので、散漫な文章になりにくいのです。逆に「ざっくりまとめて」と言うと、文体も長さもバラバラな要約が返ってきます。

同じ理由で「箇条書き5つ」のような数の指定も効きます。「5つ」と言われたら、Claudeは5つに絞るための取捨選択をします。

注意：Webから来た情報との付き合い方

便利な魔法ですが、大切な注意点が1つあります。Webに書いてあることは、必ずしも正しくない。個人ブログ、古い記事、誤情報、広告まみれのページ――Claudeは善意で取ってきますが、内容の真偽までは保証できません。

公式ドキュメント必読

医療・法律・金融・安全性に関わるテーマでは、必ず公式情報源（厚労省、法務省、公式サイト等）のURLを指定してWebFetchしましょう。WebSearchの結果だけで判断するのは危険です。詳しくは 公式ドキュメント のセキュリティ項を。

そこでおすすめの聞き方がもう1つ。

「出典付き」プロンプト
【テーマ】について調べてまとめた後、
それぞれの情報の出典URLも併記して。
情報が複数の記事で一致しているかもコメントして。
出典を見れば自分で確認できる。「一致してるか」を聞けば裏取りになる。

こうすると、Claudeは「3つの記事で共通していた情報」と「1つの記事にしかなかった情報」を分けてくれたりします。人間のリサーチャーと同じ仕事を、数秒でやってくれるわけです。

「情報を疑う力」は、AI時代の新しい教養です。Claudeは賢いですが、あなたが最終判断者です。肩の力を抜いて、でも鵜呑みにしないで。

使いどころのイメージを広げよう

最後に、日常のいろんな場面で「Webで調べる魔法」を使う例を見てみましょう。Claudeは「これに使うもの」と決まっているわけではありません。あなたの日々の「ちょっと調べたい」をぜんぶ代行してくれます。

あなたの1日で使える場面（Claudeの出番は何回？）
場面お願いの例道具
朝のニュースチェック 「今日の主要ITニュースを3つ、各3行で」 WebSearch
買い物の下調べ 「在宅ワーク用イスのおすすめを予算5万円で比較」 WebSearch + WebFetch
旅行プラン 「京都2泊3日、お寺巡り中心のモデルコースを」 WebSearch
仕事の調査 「競合A社の最新プレスリリースを3件要約」 WebFetch（URL直指定）
学習・勉強 「○○という概念を小学生にもわかるよう解説」 WebSearch
レシピ検索 「冷蔵庫にキャベツと鶏肉。30分で作れる料理3つ」 WebSearch
もっと詳しく: Webに出さないほうがいい情報

Claude自身は個人情報を勝手に送信しませんが、プロンプトに書いた内容はClaudeが受け取ります。以下はプロンプトに書かない方がいい情報の目安です。

  • クレジットカード番号、銀行の暗証番号
  • 本名と住所の組み合わせ（個人特定につながる）
  • パスワード、APIキー（APIキーなど）
  • 社内の機密情報（契約書、顧客リスト）

「セキュアな」情報を扱う時は、会社のポリシーと公式ドキュメントを確認しましょう。

3分間ハンズオン：今すぐ自分の「興味」を調べてもらう

この章で一番大事な時間です。Claude Codeを開いて、あなた自身の興味について、Claudeに調べてもらいましょう。何でもOK。好きな音楽、行きたい国、育てたい植物、気になっている仕事――何でも。

1
テーマを決める
自分がわくわくするもの1つ
2
プロンプトを書く
「調べて+まとめて」の型で
3
実行
Enterを押す
4
結果を読む
足りなければ追加質問
「調べる」って、自分の好奇心を手放さないってこと。Claudeはそれを全力でサポートしてくれる。

プロンプト例のテンプレはこれ：

【自分の興味ある話題】について、最新情報をWebSearchで調べて、
初心者の私にわかりやすく5つのポイントにまとめて。
できれば出典URLも付けて。

やってみました？ 何か返ってきましたね。おめでとうございます。あなたは今日から、手元に専属のリサーチャーがいます。

Exercise 1
入門

今週末の予定をClaudeと相談する

今週末に行けそうな近場のイベントやスポットを、Claudeに調べてもらってください。

  1. 自分の住んでいる街または近隣の大きな駅名を決める
  2. 「今週末の〇〇エリアのイベントをWebSearchで調べて、3つおすすめを教えて」と頼む
  3. 返ってきた内容が気になったら、URLを指定して「このイベントの詳細をWebFetchで読んで」と追撃する
模範解答を見る

たとえば横浜住みの人なら、こう打てばOK：

今週末の横浜エリアのイベントをWebSearchで調べて、
大人2人で楽しめるものを3つ、
場所・時間・だいたいの予算付きで教えて。

追撃のプロンプト例：

https://example.com/event-detail-url
このページをWebFetchで読んで、
必要な持ち物と混雑予想を教えて。

ポイント: 1回目はWebSearchで候補出し、2回目はWebFetchで深掘り。この2段コンビが鉄板です。

Exercise 2
応用

興味あるテーマで「ミニ調査レポート」を作らせる

自分が最近気になっているテーマについて、ClaudeにMarkdownの調査レポートを作らせてください。これは実務でもそのまま使えるテクです。

  1. テーマを1つ決める（例：ZEV車の最新動向、在宅ワーク機器、発酵食品の健康効果など）
  2. 「WebSearchで検索 → 上位3記事をWebFetch → Markdownレポートにまとめ」をプロンプトで指示
  3. レポートには「要約・ポイント3つ・出典URL」の3セクションを要求
  4. 出てきた結果を実際にファイル保存してみる（report.md など）
模範解答を見る

プロンプト例（発酵食品の健康効果で調査するケース）：

テーマ: 発酵食品の健康効果（2026年版）

以下の手順で調査して、Markdownレポートを作成して report.md に保存して：

1. WebSearchで「発酵食品 健康効果 2026」などのキーワードで検索
2. 信頼性が高そうな記事を3つ選ぶ（公的機関・大学・大手メディア優先）
3. 選んだ3記事をそれぞれWebFetchで本文取得
4. 以下の構成でレポート化：
   ## 要約（3行）
   ## 主要ポイント（箇条書き5つ）
   ## 出典（URL + 発信元名）

日本語で、初心者にもわかる言葉遣いでお願いします。

このプロンプトの勘所：

  • 信頼性の基準を明示（公的機関・大学・大手メディア優先）することで、個人ブログや広告記事を自動で外してくれる
  • 構成を具体的に指定することで、期待通りのレポートになる
  • ファイル保存まで含めると、あとで読み返せる資産になる

返ってきたレポートが物足りなければ、「もっと専門的に」「もっと簡単に」「グラフも作って」など、追撃のリクエストで磨いていけばOKです。

いまのあなたは

Claude Codeを使って、インターネットから必要な情報だけを引き出して、きれいにまとめさせることができるようになりました。WebFetchとWebSearchを使い分け、出典付きで調査レポートまで作れます。これは、会社で「リサーチ係」を1人雇ったのと同じくらいの戦力。自信を持っていい。全ユーザーの中でも上位に食い込む実力です。次の章からはいよいよ、自分のWebアプリ作りが始まります。深呼吸して、楽しみに進みましょう。
