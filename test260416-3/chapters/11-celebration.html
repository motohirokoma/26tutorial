もっと詳しく: 今までの振り返りチェックリスト

ひとつずつ心の中で「できる」「まだ怪しい」を確かめてみてください。全部○じゃなくても、ここまで読めていれば合格です。

  • ターミナルを開いて claude と打って起動できる
  • 依頼文を「ゴール・具体・制約」に整えて書ける
  • git add → git commit の流れで作業を保存できる
  • エラーが出ても深呼吸して、そのままClaudeに見せられる
  • Plan モードを使って「まず計画を見せて」と言える
  • 自分のGitHub Pagesに作品URLがある
  • 作品のURLを誰かに送れる(送った or いつでも送れる状態)

怪しい項目があったら、該当章に戻ってOK。戻ることは後退ではなく、地図を読み直すことです。

ここは節目です。コーヒーを淹れて、10時間の自分を少しだけ褒めてあげてください。続けた事実はもう誰にも取られません。

友達に見せよう

あなたの作ったWebアプリにはURLがあります。これは「世界中の誰でもブラウザで開ける住所」です。せっかくなので、近しい人に見せてみましょう。反応が返ってくると、学習のエネルギーが跳ね上がります。

共有する相手の候補

  • 家族・パートナー — 一番安全な第一号。「これ、10時間かけて作った」の一言を添えて。
  • 近しい友達 2-3人 — 「感想ほしい」と添えると返事が来やすい。
  • 同僚・上司 — 意外と「お、やるじゃん」につながる。履歴書の小さな一行にもなる。
  • SNS — X(旧Twitter)や Facebook に一行投稿。同じ道を歩き始める誰かの背中を押すことにもなる。

Claude Code との付き合い方

最後に、これから長く付き合っていくための心構えを2つだけ。

1. 「道具」から「相棒」へ育てる

Claude Codeは使い始めは便利な道具です。でも使い込むほど、あなたの癖・好みをCLAUDE.mdやSkillに書き溜められて、あなた専用の相棒になります。最初は丁寧に依頼して、うまく動いた指示の型はメモしておきましょう。

道具は最初ぎこちない。慣れてくると意識から消える。消えたとき、それはもう体の一部。— 学習一般の法則

2. 作ったものを育て続ける

Ch10で公開したWebアプリは、終わった作品ではなく始まった作品です。ちょっとした機能追加・見た目の調整を繰り返すだけで、あなたの「名刺代わり」に育ちます。

もっと詳しく: これからの計画を書き出す

漠然と終わらせないために、紙かメモアプリで下の3つを書いてみてください。3分で済みます。

  1. 今週中にやる小さな1つ — 例: 友達にURLを送る / ボタンの色を変える
  2. 1ヶ月以内に読む章・チュートリアル1つ — 例: Subagents Mastery の Ch01
  3. 3ヶ月後に見たい自分の状態 — 例: 2個目のアプリを公開している / 職場の小さな自動化を1つClaudeと作った

書いたら、カレンダーにリマインダーを1つだけ仕込むと実行率が跳ね上がります。
