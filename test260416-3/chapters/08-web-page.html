実践: 自己紹介Webページ

これまで学んだ全部を使って、1ページ完結の自己紹介サイトを作る。完成品はGitHub Pagesで公開する。

所要時間: 1h Phase: Practice 難易度: 実践

ゴール: 何を作るのか

1ページの自己紹介サイトを作ります。必要な機能は全部 Claude が書く。あなたは「方向を決める人」と「レビュアー」に徹します。

完成イメージ
ヘッダ + プロフィール + スキル + コンタクト の典型的な1ページ構成。

進め方の段取り

5ステップで完成まで
1
準備
フォルダ作成 & git init
2
骨組み
HTMLの構造
3
スタイル
CSSで見た目
4
調整
細部を磨く
5
公開
GitHub Pages
1ステップ20分以内。詰まったらCh07へ戻る。

Step 1: 準備

# 新しい作業フォルダ
mkdir ~/Desktop/my-profile
cd ~/Desktop/my-profile
git init
claude

Claude起動後、まずプロジェクトの方向性を伝えます。

> このフォルダで自己紹介の1ページサイトを作ります。
  使うのは index.html と style.css の2ファイルだけ。
  まずは方針を3案提案してください。

Step 2: 骨組み

方向性を1つ選んだら、HTMLの構造から作ります。

> 案Aで進めて。まずは index.html だけ、スタイルなしで構造を作って。
  必要なセクション:
  - ヘッダ (名前 + キャッチコピー)
  - プロフィール (写真位置 + 自己紹介文)
  - スキル (3〜5個のタグ)
  - コンタクト (GitHub / Twitter リンク)
なぜ骨組みから?

いきなりスタイルまで頼むと「思ってたのと違う」が起きやすい。骨組みを先に作って構造を確認 → スタイルを後から足す方が、修正が局所的で済む。

Step 3: スタイル

骨組みができたら、CSSを足します。

> OK、構造は完成。次は style.css を作って、以下のスタイル:
  - 背景: ほぼ白 (#fafafa)
  - 文字: 濃い灰色 (#222)
  - アクセント: 1色だけ (お任せ)
  - フォント: sans-serif
  - スマホでも崩れないようレスポンシブ
  外部CSSフレームワーク (Tailwind等) は使わない。

ブラウザで index.html を開いて、見た目を確認しましょう。

Step 4: 調整

「もう少しここをこうしたい」を遠慮なく伝えます。

  • 「もう少し落ち着いた色にして」
  • 「見出しのフォントを大きく」
  • 「写真を中央寄せに」
  • 「スマホで見たときヘッダが切れてる、直して」

Step 5: GitHub Pages で公開

Ch06で覚えたフローでサクッと公開。

> このフォルダを GitHub の "my-profile" リポジトリ (Public) に push して、
  GitHub Pages を有効化してください。完了したら公開URLを教えて。

1〜2分待つと https://ユーザー名.github.io/my-profile/ でアクセスできます。家族や友人に送ってフィードバックを集めるのもいい体験。

もっと詳しく: 「写真をどこから持ってくる?」

本物の自分写真でなくてOK。UnsplashやPexels のフリー写真を使うか、 SVGのアバター をClaudeに作らせるのも手。「シンプルなアバターSVGを丸い枠で作って」と頼めば、それっぽいものが返ってきます。

Exercise 1
入門

本章の手順通りに自己紹介ページを完成させる

5ステップを実際に通します。

  1. 新しい作業フォルダを作る
  2. 本章の依頼例をそのまま使ってStep 1〜4を実行
  3. ブラウザで開いて意図通りか確認
  4. 気になる箇所を1回だけ調整依頼
  5. Step 5 で GitHub Pages に公開
模範解答を見る

「自分の手で完走する」ことが目的なので、模範解答というよりチェックリストです。

[完走チェック]
□ ローカルで index.html がブラウザで開ける
□ 名前 / プロフィール / スキル / コンタクト の4セクションがある
□ スマホサイズで崩れていない
□ GitHub Pagesで公開URLが取れた
□ 公開URLをスマホでも開ける

5項目クリアできれば、もう「Webサイト作って公開できる人」です。これは履歴書に書ける成果。

Exercise 2
応用

ダークモード切替ボタンを追加

1機能だけ「自分で考えて頼む」体験。

  1. 「右上に🌙ボタンをつけて、押すとダークモードに切り替わるようにして。設定はlocalStorageに保存して次回も覚えてて」と依頼
  2. diff を確認 → 承認
  3. ブラウザで実際に切り替えて確認 (再読込しても保持されているか)
  4. うまく動かなければエラー文をClaudeに渡して修正依頼
  5. commit + push で公開版にも反映
模範解答を見る

JavaScriptで document.documentElement.dataset.theme を切り替え + localStorageに保存、CSSで [data-theme="dark"] セレクタでスタイル定義、というのが定番実装。

// 概念だけ抜粋
const btn = document.querySelector('#theme-toggle');
const saved = localStorage.getItem('theme') || 'light';
document.documentElement.dataset.theme = saved;

btn.addEventListener('click', () => {
  const next = document.documentElement.dataset.theme === 'dark' ? 'light' : 'dark';
  document.documentElement.dataset.theme = next;
  localStorage.setItem('theme', next);
});

これが 「自分で機能を考えて、Claudeに頼んで実装する」 最初の体験。Ch09 ではこれを発展させたミニアプリを作ります。
