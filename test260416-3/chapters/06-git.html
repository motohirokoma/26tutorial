Gitとちょっと連携

変更履歴を残し、いつでも巻き戻せるようにする「セーブポイント」の仕組み。GitHubに公開するところまで。

所要時間: 1h Phase: Core 難易度: 中級

Git は「セーブポイント」

Git をひと言で言うと、ゲームのセーブ機能。ファイルの状態をスナップショットで残しておけるので、失敗しても戻れます。Claude Code と組み合わせると、安心して試行錯誤できます。

Gitの全体像
add → commit → push の3段階で「セーブ → クラウド保管」が完了します。

必要な準備

  • Gitをインストール — Macは git --version で既に入ってる場合が多い。Windowsは git-scm.com から
  • 名前とメールを設定 — git config --global user.name "Your Name" など
  • GitHubアカウント作成 — github.com で無料登録
  • gh CLI をインストール — GitHub操作を楽にする公式ツール (任意だが推奨)

最頻出の5コマンド

コマンド意味目安頻度
git status今の状態を確認頻繁(困ったらまずこれ)
git add .変更を全部ステージに乗せるcommit前に1回
git commit -m "..."セーブする区切りごとに
git pushGitHubに送るcommit後に
git log --oneline過去のセーブ一覧を見る履歴確認時
Claude Code に頼める!

これらのコマンドも全部 Claude に頼めます。「変更をコミットして、メッセージは『プロフィール追加』で」と言うだけ。コマンドを覚える必要はありません。

典型ワークフロー

Claude × Gitの黄金パターン
sequenceDiagram
  participant U as あなた
  participant CC as Claude Code
  participant G as Git
  participant H as GitHub
  U->>CC: 「変更前にコミット作ってから直して」
  CC->>G: git commit (バックアップ)
  CC->>CC: ファイル編集
  CC->>U: 差分表示
  U-->>CC: 承認
  CC->>G: git commit (新版を保存)
  U->>CC: 「OK、GitHubにプッシュ」
  CC->>H: git push
  H-->>U: 公開完了
「直す前にコミット」を習慣化すれば、常に「直前の安全地点」がある状態になる。

巻き戻しの技

「あ、やっぱり前の方が良かった」という時の戻し方。3段階あります。

状況コマンド影響範囲
編集中だがcommit前git restore ファイルそのファイルだけ
1つ前のcommitに戻すgit revert HEAD履歴を残しつつ巻き戻し
もっと前のセーブに戻すgit log + git checkout コミットID任意の時点

GitHub Pagesで公開

HTMLファイルをGitHubに置けば、誰でも見られるWebサイトとして公開できます。料金は無料。

  • GitHubに新しいリポジトリを作る (公開設定)
  • ローカルで git remote add origin URL でつなぐ
  • git push -u origin main でアップロード
  • GitHubの Settings > Pages で「main ブランチ」を選ぶ
  • 1〜2分待つと https://ユーザー名.github.io/リポジトリ名/ で公開!
もっと詳しく: なぜ GitHub に置くだけで Web サイトになるの?

GitHub Pages という GitHub の機能で、リポジトリの index.html を Web サーバー経由で配信してくれます。サーバー契約もドメイン取得も不要。練習・ポートフォリオ・小規模サービスにはこれで十分です。

commit メッセージのコツ

commit メッセージは 未来の自分への手紙。「何のために」が分かるように書きます。

悪い

意味不明メッセージ

  • fix
  • update
  • あれ
  • WIP
良い

未来に伝わるメッセージ

  • プロフィール画像を中央寄せに
  • ヘッダの色を青→黒に変更
  • localStorageに対応
  • Mobile Safariで崩れる問題を修正
Exercise 1
入門

練習用フォルダを Git 管理にする

初めての git init から最初の commit までを体験します。

  1. 練習用フォルダで claude を起動
  2. 「このフォルダを Git 管理に初期化して、最初のコミットを作って」と依頼
  3. Claudeが git init → git add . → git commit を提案
  4. コミットメッセージを「Initial commit」に
  5. 「git log --oneline で履歴を見せて」で確認
模範解答を見る

Claude は順番にコマンドを提案して承認を求めます。1つ1つ Enter で進めばOK。

$ git init
Initialized empty Git repository in ...

$ git add .

$ git commit -m "Initial commit"
[main (root-commit) abc1234] Initial commit
 3 files changed, ...

$ git log --oneline
abc1234 Initial commit

これでこのフォルダが Gitで履歴管理される状態 になりました。以降、Claudeが何をしてもコミット履歴に残ります。

Exercise 2
応用

GitHub Pages で profile.html を公開する

Ch05で作った profile.html を世界に公開します。

  1. GitHubで新しいリポジトリ my-profile を作成 (Public)
  2. Claudeに「このフォルダを my-profile リポジトリにプッシュして」と依頼 (gh CLIがあれば自動)
  3. GitHubの Settings > Pages で main ブランチを選択
  4. 1〜2分後、https://ユーザー名.github.io/my-profile/ にアクセス
  5. 表示されたら成功!URLをスマホで開いてみる
模範解答を見る

gh CLI が入っていれば、Claudeは1コマンドでリポジトリ作成→push までやってくれます。

# Claudeが実行する典型コマンド
$ gh repo create my-profile --public --source=. --push
$ gh api --method POST /repos/USER/my-profile/pages \
    -f "source[branch]=main" -f "source[path]=/"

# 1〜2分待ってから
$ open https://USER.github.io/my-profile/

これが Claude × Git × GitHub の黄金パターン。10分かからずに自分のWebサイトを公開できます。Ch08〜10ではこの仕組みを使って実践プロジェクトを公開していきます。
