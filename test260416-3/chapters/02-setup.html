セットアップ：15分で相棒がやってくる

Mac/Windows どちらでもOK。15分後にはあなたのPCでClaudeと会話できます。

所要時間: 45分 Phase: Basics 難易度: 入門
ようこそ

焦らなくて大丈夫です。この章ではあなたのPCにClaude Codeをお迎えします。途中で画面が真っ黒になって驚くかもしれませんが、それは「相棒を呼び出す部屋」なだけ。終わる頃には、自分のPCで「こんにちは」と会話できる状態になります。

2026年1月時点の情報

本章は2026年1月時点の Claude Code CLI (v2.x系) を前提にしています。インストーラや画面は細部が変わる可能性があるので、コマンドが通らない時は公式セットアップガイドも参照してください。

15分の全体像 — まず地図を見よう

これからやることは、たった4つのステップだけです。いきなり手を動かす前に、地図を見て安心してから進みましょう。

どんなにPC操作に慣れていなくても、このフローに沿えば必ずゴールに着きます。

ここまで読んで「本当に15分で終わるの?」と疑う気持ち、よく分かります。大丈夫。途中でつまずいても、この章には「こんなエラーが出たらこう対処」のガイドが入っています。

黒い画面(ターミナル)ってなに?

これから何度も出てくる「黒い画面」。正式名はターミナルといいます。怖そうに見えますが、実はPCに文字で命令する部屋でしかありません。

アナロジーで言うと、普段のPC操作が「レストランでメニューを指差す」なら、黒い画面は「厨房に直接『これを作って』と伝える」感じです。慣れると直接のほうが速くて正確。

黒い画面の中身(イメージ)
$ のあとに文字を打って Enter を押す——それだけ。

Mac と Windows で使う黒い画面の名前が違います。この章では両方を並べて案内していきます。

項目MacWindows
黒い画面の名前ターミナル (Terminal)PowerShell または Windows Terminal
標準搭載?はい(最初から入ってる)はい(Windows 10以降)
開き方Spotlight で「ターミナル」と入力スタートメニューで「PowerShell」と入力
この先やる事コマンド貼り付け→Enterコマンド貼り付け→Enter
もっと詳しく: 「コマンドプロンプト」と「PowerShell」の違い

Windowsには2つの黒い画面があります。コマンドプロンプト(cmd.exe)は昔からある古いやつ、PowerShellは新しくて高機能な方です。このチュートリアルでは PowerShell を使います。もしスタートメニューで「Windows Terminal」が見つかればそちらのほうがモダンで快適ですが、どちらでも同じコマンドが動きます。

Mac / Windows セットアップ手順

ここからが本番です。自分のPCのほうだけ読んでください。Mac使いはMac欄、Windows使いはWindows欄。両方読む必要はありません。

手順MacWindows
1. 黒い画面を開く Command+Space で Spotlight を開き「ターミナル」と入力→Enter スタートボタン→「PowerShell」と入力→Enter
2. Node.js の有無を確認 node -v と打つ node -v と打つ
3. Node.js 未インストールなら nodejs.orgのLTS版をダウンロード→インストーラ実行 nodejs.orgのLTS版をダウンロード→インストーラ実行
4. Claude Code をインストール npm install -g @anthropic-ai/claude-code npm install -g @anthropic-ai/claude-code
5. 動作確認 claude --version claude --version
6. 初回ログイン claude と打つ→ブラウザで認証 claude と打つ→ブラウザで認証

気づきましたか? Node.js を入れた後は、Mac も Windows も同じコマンドで進みます。OSによる違いは「黒い画面の開き方」だけなんです。

Tips

黒い画面へコマンドを貼り付ける時はコピー後にそのままペースト。Macは Command+V、Windowsは右クリック(PowerShell)または Ctrl+V(Windows Terminal)。手打ちすると typo するので、必ずコピペで。

ステップ1-2: Node.js の確認

黒い画面で次を打ってください。v20.18.0 のようにバージョン番号が返ってきたらOK。

node -v

もし command not found や 認識されていません と出たら、Node.js がまだ入っていないサインです。nodejs.org で LTS 版(左側の緑)をダウンロードして、インストーラの指示通り「次へ」を押していけば5分で終わります。

ステップ3: Claude Code を迎え入れる

Node.js が確認できたら、Claude Code 本体を入れます。この1行だけ:

npm install -g @anthropic-ai/claude-code

npm がお弁当を宅配する感じで、CLI を運んできてくれます。30秒〜2分くらい待ちます。最後に added 42 packages みたいな表示が出たら成功です。

ステップ4: バージョン確認 = 小さな成功1

claude --version

2.1.0 (Claude Code) のように数字が返ってきたら、おめでとうございます! あなたのPCに相棒が住み始めました。自分を褒めていい瞬間です。

ここまでで一度深呼吸

ここまで来たあなたは、もう「黒い画面でソフトを入れられる人」です。日本の大人の中でも、自分のPCに開発者向けのCLIを入れた経験がある人は多くありません。小さな一歩ですが、確実に一歩です。

認証：Claudeにあなたを覚えてもらう

初めて claude と打つと、「どのアカウントで使う?」と聞かれます。これを認証といいます。アナロジーで言えば、ビルに入るときの入館証の発行です。

公式ドキュメント必読

認証(認可)まわりはセキュリティの根幹です。誤った扱いでアカウント情報が漏れると金銭的被害が出ることもあります。API key(文字列)を他人やSNSに貼らない、公開リポジトリにコミットしない——この2点だけは厳守してください。詳しくは公式を。Claude Code 認証の公式ドキュメント →

認証の選択肢

方式向いてる人手順メモ
Claude Max / Pro サブスク 月額で使い放題にしたい人 claude → ブラウザで claude.ai にログイン 推奨。このチュートリアルはこれ前提
Anthropic API key 従量課金で使いたい人・会社貸与のキー 環境変数 ANTHROPIC_API_KEY を設定 使った分だけ請求。キーは他人に絶対見せない
Bedrock / Vertex 経由 AWS/GCP で社内契約済みの会社員 管理者に設定値をもらう 非エンジニアなら基本スルーでOK

ステップ6: 実際にログインする

黒い画面で次を打ちます:

claude

初回だけ「Open this URL in your browser」のようなメッセージが出ます。URLをコピーしてブラウザに貼ると、Anthropic のログイン画面が開きます。普段 claude.ai で使っているのと同じアカウントでログインしてください。

ブラウザで「Allow(許可)」を押すと、黒い画面に戻って認証完了のマークが出ます。

もっと詳しく: API key 方式を選ぶ場合

Anthropic Console で API key を発行できます。発行した sk-ant-... で始まる文字列を、環境変数 ANTHROPIC_API_KEY に設定します。

Macなら ~/.zshrc に次の1行:

export ANTHROPIC_API_KEY="sk-ant-xxxxx"

Windows(PowerShell)なら:

setx ANTHROPIC_API_KEY "sk-ant-xxxxx"

このキーは絶対に他人に見せないでください。GitHub に誤ってアップしてしまうと数分で悪用される事例も報告されています。

はじめての対話：「こんにちは」

認証が終わったら、claude と打って、画面が対話モードに切り替わるのを待ちます。> や │ のようなプロンプトが出たら、そのまま日本語で話しかけましょう。

> こんにちは

Enter を押すと、数秒待って返事が返ってきます。

「はい、こんにちは！今日は何をお手伝いしましょうか?」 — あなたのPCの中のClaude

この瞬間、あなたと相棒の関係が始まりました。ここが本当のゴール。

使い終わる時

対話を終えるには /exit と打つか、Ctrl+C を2回押します。また呼び出したい時は claude と打てばOK。

うまくいった時

画面に「こんにちは」の返事

  • 数秒以内に返事
  • 日本語でスムーズに会話できる
  • /exitで終了できる
もしエラーが出ても

Exercise — やってみよう

Exercise 1
入門

「こんにちは」の往復をしてみる

Claude Code を起動して、日本語で「こんにちは」と話しかけ、返事をもらいましょう。そしてそのあと、もう一つだけ質問を続けてみます。

  1. 黒い画面を開く
  2. claude と打ってEnter
  3. 「こんにちは」と入力→返事を受け取る
  4. 続けて「今日の天気に合う一言を教えて」などと話しかけてみる
  5. /exit で終了
模範解答を見る

こんな流れになれば成功です:

$ claude
> こんにちは
こんにちは！何をお手伝いしましょうか?

> 今日の天気に合う一言を教えて
晴れやかな気分で新しい一歩を踏み出す日ですね。

> /exit

返事の内容は毎回違うのでOK。返事が返ってくる事実が成功の証です。

Exercise 2
応用

認証状態と設定を確認してみる

Claude Code がどのアカウントで動いているかを確認しましょう。他のOSに移った時に役立ちます。

  1. 黒い画面で claude --version を打つ
  2. 次に claude を起動し、対話モードで /status と打つ
  3. 表示されたアカウント/モデル名をメモする
  4. /exit で終了
模範解答を見る

/status を打つと、現在のアカウント・使用モデル・作業ディレクトリなどが一覧表示されます。

> /status
Account:  your-email@example.com
Model:    claude-sonnet-4-7
Working:  /Users/you/
Auth:     OAuth (Max)

ここで Auth: OAuth (Max) や Auth: API key が見えれば、認証は正しく通っています。もし Not authenticated と出たら、もう一度 claude を起動してログインをやり直してください。
