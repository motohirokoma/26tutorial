基本コマンドと会話の流れ

スラッシュコマンドと「依頼→レビュー→確定」の対話リズム。Claude Code との上手な付き合い方。

所要時間: 1h Phase: Basics 難易度: 入門

会話のリズム — 3拍子で動く

Claude Code との会話は、ほぼ 3拍子の繰り返し です。これを身体で覚えてしまえば、もう怖いものはありません。

Claude Code 会話の3拍子
1
依頼
日本語で頼む
2
レビュー
提案と差分を見る
3
確定
承認 or 修正依頼
この3拍子を100回繰り返すうちに、自然と「上手な依頼の仕方」が身につきます。

スラッシュコマンド最低限セット

「/ で始まる魔法の命令」が スラッシュコマンド。覚えるのは最初は5つで十分です。

コマンド役割こんなとき
/helpヘルプを表示使い方を忘れた時
/clear会話履歴を全部消す新しいテーマを始める時
/compact会話を要約して圧縮履歴が長くなって重い時
/initCLAUDE.md を作るプロジェクト開始時
/exitClaude Code を終了終わる時
もっと詳しく: /clear と /compact はどう違う?

/clear は 過去の会話を完全に捨てる ので、Claudeは「初対面」状態になります。テーマを変える時に使う。/compact は 過去のやりとりを要約して保持 するので、コンテキストを残しつつ容量(=料金)を節約できます。長丁場のセッションでは /compact が便利です。

承認のフロー

ファイル変更やコマンド実行の前、Claudeは必ず確認してきます。「これでいいですか?」と問われたら、3つの選択肢があります。

変更承認の分岐
flowchart TD
  C["Claudeが提案"] --> Q{"内容は適切?"}
  Q -- "はい" --> Y["Enter で承認"]
  Q -- "微修正" --> M["『○○を変えて』と返す"]
  Q -- "却下" --> N["Esc でキャンセル"]
  Y --> D[ファイル更新]
  M --> C
  N --> R[元の状態を維持]
Enter / 自然言語修正 / Esc の3択を覚えれば、暴走の心配なし。

キーボードショートカット

マウスを使わない世界なので、ショートカットを少し覚えると体感速度が一気に上がります。

Esc
処理を中断
Ctrl+C
強制終了 (2回押し)
↑ / ↓
過去の入力履歴
Shift+Enter
改行(複数行入力)

CLAUDE.md という「常駐メモ」

プロジェクトのルートに CLAUDE.md ファイルを置くと、Claudeが 毎回最初に読んでくれる 常駐メモになります。「このプロジェクトは○○です」「××のスタイルで書いて」といった指示を恒常化できます。

# /init で雛形を生成
> /init

Claude が自動でプロジェクトをスキャンし、CLAUDE.md の雛形を作ってくれます。

CLAUDE.md に書くと便利な情報

使用言語(例: 「日本語で返事して」)、コーディング規約、プロジェクトの目的、よく使うコマンド。毎回入力している指示は CLAUDE.md に逃がす のが鉄則。

使用量の体感

1セッションでだいたい何回くらい依頼すればいいか?平均的な学習者の依頼数の内訳を見てみましょう。

1時間セッションの依頼内訳 (平均)

圧倒的に多いのは 「修正依頼」。一発で完璧なものは出ないので、「ここをこう変えて」を何度も繰り返すのが普通です。これは下手なのではなく、これが正しい使い方 です。

Exercise 1
入門

/help と /clear を実際に試す

スラッシュコマンドの感覚を掴みます。

  1. Claude Code を起動
  2. /help でヘルプ一覧を表示
  3. 適当な会話を3往復する(「自己紹介して」「もっと短く」「英語で」など)
  4. /clear で履歴をリセット
  5. もう一度「自己紹介して」と聞き、前回の会話を覚えていないことを確認
模範解答を見る

/clear 後は完全な「初対面」状態。「英語で」と頼んだ前の文脈は完全に消えるので、再び日本語で答えてきます。これが セッションの文脈リセット 体験です。

> /help
... (ヘルプ表示)

> 自己紹介して
... (日本語で返答)

> もっと短く
... (短くなる)

> 英語で
... (英語で返答)

> /clear
[History cleared]

> 自己紹介して
... (再び日本語で初対面の挨拶)
Exercise 2
応用

CLAUDE.md を /init で作って育てる

常駐メモの威力を体感します。

  1. 練習用フォルダで /init を実行
  2. 生成された CLAUDE.md を確認
  3. 末尾に「以後、すべての応答は **絵文字を1つ含めて** 返答してください」を手動で追記
  4. Claudeに「お疲れさま」と話しかけてみる(絵文字が含まれていれば成功)
  5. 追記行を消して保存し、もう一度同じ会話で挙動が戻ることを確認
模範解答を見る

CLAUDE.md は セッション開始時に毎回読み込まれる ので、追記した直後の挙動には反映されない場合があります。/clear で履歴をリセット → 再質問するのが確実。

# CLAUDE.md (例)
このプロジェクトは Claude Code チュートリアルの演習用です。

## 応答ルール
- 日本語で返答する
- 以後、すべての応答は **絵文字を1つ含めて** 返答してください  ← 追記

「常駐指示の力」 を体感するのが目的。これを応用すれば「常に PEP8 に沿った Python を書いて」「コミットメッセージは英語で」といった恒常ルールを設定できます。
