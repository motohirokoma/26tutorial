Claude Codeって何?

まずは全体像から。Claude Codeが何者で、なぜ非エンジニアでも使えるのかを掴みます。

所要時間: 45min Phase: Basics 難易度: 入門
ターミナルやAPIキーに馴染みが薄い方は

先に はじめての方へ (前提知識) を流し読みしておくと、この章以降がスムーズに読めます。

Claude Codeを一言でいうと
あなたが日本語で頼むだけで、Claudeがあなたのファイルに直接手を入れて成果物を作ります。

一言で言うと

Claude Codeは「ターミナルの中に住んでいるAI開発アシスタント」です。あなたが日本語で「こんなWebページを作って」と話しかけると、Claudeがあなたのパソコンのファイルを直接読み書きして、アプリやページを作り上げてくれます。

  • ターミナルで動く — ブラウザやアプリではなく、文字ベースの画面で会話する
  • あなたのファイルを直接操作 — コピペ不要、Claudeが読んで書く
  • 対話しながら進む — 一発勝負ではなく、何度でも調整できる

動作ループの全体像

Claude Code は「依頼 → 実行 → 確認 → 調整」の 無限ループ で成果物に近づけていきます。

Claude Codeの動作ループ
flowchart LR
  U["あなた (日本語で依頼)"] --> C[Claude Code]
  C -->|"読む / 書く / 実行"| F["ファイル・コマンド"]
  F --> C
  C -->|結果と提案| U
  U -->|調整・追加の依頼| C
依頼 → 実行 → 確認 → 調整、を何度でも回せる。これがエージェント型AIの特徴。

ChatGPT / Copilot との違い

「AIにコードを書かせるならChatGPTでもいいのでは?」と思うかもしれません。違います。Claude Code は エージェント型 AIで、ChatGPTやGitHub Copilotとは役割が違います。

ツール関係性主な動作あなたの仕事
ChatGPT / Claude.ai会話型質問に答える回答をコピペしてエディタに貼る
GitHub Copilot補完型次の1行を提案運転席に座って受け入れる
Claude Codeエージェント型ファイルごと作る・直す方向を決めてレビューする

たとえるなら、ChatGPTは「相談相手」、Copilotは「助手席」、Claude Codeは「ハンドルごと渡せる運転手」。「横浜まで行って」と頼めるタイプです。

もっと詳しく: なぜ「ターミナルで動く」ことが重要なのか

ターミナルはOSのすべての機能に触れる「素の窓口」です。ブラウザ経由のAIはサンドボックスに閉じ込められていて、あなたのファイルを本当の意味では操作できません。Claude Codeはターミナルで動くので、git、npm、ls といった既存のツールをそのまま呼び出せます。結果、プロのエンジニアが日頃使っている道具箱を、そのままあなたの道具にできます。

非エンジニアでも使える3つの理由

理由1

日本語でOK

「赤いボタンを追加して、クリックでアラート」で通じる。コードを1行も書けなくても、言葉にできれば成果物が返ってくる。

理由2

書くのはClaude

あなたはレビュアー。diff 表示で差分が色付きで出るので、読まなくても雰囲気で判断できる。

理由3

失敗してもすぐ戻せる

Claudeは変更前に必ず確認。Gitで履歴も残るので revert で巻き戻せる。挑戦のコストが激低。

向いてる仕事/向かない仕事

万能ではありません。Claude Code が得意な仕事と、苦手な仕事があります。

Claude Codeの得意・苦手マップ
「やることが言葉で書ける」かつ「コードで解ける」領域がClaudeの主戦場。

次の章では

実際にClaude Codeをあなたのパソコンにインストールして、「こんにちは」と話しかけてみます。

Exercise 1
入門

公式ドキュメントの目次を眺める

Claude Codeの「地図」を頭に入れます。全部読む必要はありません。どんな機能があるかをざっと見るだけです。

  1. ブラウザで docs.claude.com/en/docs/claude-code を開く
  2. 左サイドバーの目次を上から下まで一度スクロール
  3. 気になった機能・名前を3つメモ(例: 「Hooks って何?」「MCP ってどこかで聞いた」「Sub-agentsって便利そう」)
模範解答を見る

「読んで理解する」ではなく「見出しを眺めて存在を知る」のが目的。例えば以下のようなメモが書ければ十分です。

気になった機能メモ:
1. Slash commands  → /help /clear /compact など。まず最初に覚える
2. CLAUDE.md       → プロジェクトに恒常的な指示を書いておくファイル
3. MCP             → 外部ツール (Gmail / Driveなど) と接続できる仕組み

この先の章で出てきたときに「あ、目次で見たやつだ」と気づければ学習効率がぐっと上がります。

Exercise 2
応用

自分の業務でClaude Codeを使える場面を5つ書き出す

学習のモチベーションは「自分ごと化」で決まります。Claude Codeを自分の日常業務・副業・趣味に引き寄せて考えてみましょう。

  1. 紙でもメモアプリでも良いので5行書く欄を用意
  2. 「Claude Codeで楽になりそうな場面」を5つ書く(業務効率化/副業/趣味なんでもOK)
  3. 各場面の横に「期待する効果」を一言(時間短縮、品質向上、学習加速 など)
模範解答を見る

非エンジニアでも十分使えます。以下は一例です。自分のワークフローに置き換えて考えるのがポイント。

1. 週次レポートの雛形作成      → 毎週30分 → 5分に短縮
2. Excelの関数や数式のデバッグ → 検索とトライの往復が消える
3. 簡単なWebサイト(LP)を自作 → 外注コストゼロ、修正も即反映
4. 議事録からタスク抽出・整形  → 1時間 → 10分、抜け漏れ減
5. 副業の顧客向け提案書の叩き台 → 白紙恐怖症が消える

コツは「大きな仕事」ではなく「細かく繰り返している作業」に目を向けること。Claude Codeは10分×5回の省略が得意です。
