ミニ Web アプリ作り：後編 + 公開

動く機能を足して、世界に公開するURLを手に入れる

所要時間: 90分 Phase: Advanced 難易度: 中級
ようこそ、最後の大仕事へ

Ch09で「見た目」まで作ったお気に入りリスト、いよいよ動く機能と世界への公開です。焦らずにいきましょう。わからなくなったらいつでも前に戻ってOK。今日マスターするのは「JavaScriptで追加・削除・保存 → Gitでアップ → GitHub Pagesで公開」の4ステップ。ゴールは「自分の作品に世界中からアクセスできるURL」を手に入れることです。

バージョン注記

本章の手順は 2026年1月時点 の GitHub と GitHub Pages のUIに基づきます。GitHub認証の方法や Pages 設定画面の文言はアップデートで変わる可能性があります。画面が違ったら慌てず GitHub Pages公式ドキュメント を確認しましょう。

① JavaScriptで機能を足す（40分）

Ch09で作ったのは、いわば「ショーウィンドウに飾られた商品」。見えるけど触れない。ここに JavaScript を足して、ボタンが効くアプリに変えていきます。

何を足すか：3つの機能

  • 追加機能 — フォームに入力して「追加」ボタンでリストに新しい項目が入る
  • 削除機能 — 各項目の横の「×」ボタンで、その項目を消せる
  • 保存機能 — localStorage に保存して、ページを閉じて開き直してもリストが残る

Claudeへのお願い（コピペでOK）

Ch03で習った「魔法の3要素（ゴール・具体的・制約）」の型で頼みます。Claudeに投げるプロンプトの実例はこちら:

app.js を作って、index.html に読み込んでください。
機能は3つ:
1. 入力フォーム(タイトルとURL)の「追加」ボタンで、リストに項目が追加される
2. 各項目に「×」ボタンを置いて、押すとその項目が消える
3. localStorage に保存して、リロードしても内容が残る

制約:
- 外部ライブラリは使わない(素のJSだけ)
- コメントを日本語で丁寧に入れてほしい
- ファイル1つで完結させて

Claude Code はこれを受けて app.js を作ってくれます。作ってもらったコードの骨格はだいたいこんな感じになります:

// app.js — お気に入りリストの機能
const STORAGE_KEY = "my-favorites";

// 保存されているリストを読み込む（なければ空配列）
function loadFavorites() {
  const raw = localStorage.getItem(STORAGE_KEY);
  return raw ? JSON.parse(raw) : [];
}

// リストを保存する
function saveFavorites(list) {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(list));
}

// 画面に描き直す
function render() {
  const list = loadFavorites();
  const ul = document.getElementById("fav-list");
  ul.innerHTML = "";
  list.forEach((item, i) => {
    const li = document.createElement("li");
    li.innerHTML = `
      <a href="${item.url}" target="_blank">${item.title}</a>
      <button data-i="${i}" class="del">×</button>
    `;
    ul.appendChild(li);
  });
}

// 追加ボタン
document.getElementById("add-btn").addEventListener("click", () => {
  const title = document.getElementById("title").value.trim();
  const url   = document.getElementById("url").value.trim();
  if (!title || !url) return;
  const list = loadFavorites();
  list.push({ title, url });
  saveFavorites(list);
  render();
  document.getElementById("title").value = "";
  document.getElementById("url").value = "";
});

// 削除ボタン（イベント委譲）
document.getElementById("fav-list").addEventListener("click", (e) => {
  if (!e.target.classList.contains("del")) return;
  const i = Number(e.target.dataset.i);
  const list = loadFavorites();
  list.splice(i, 1);
  saveFavorites(list);
  render();
});

// 最初の描画
render();
今すぐ試す（5分）

作った app.js を Ch09 の index.html に <script src="app.js"></script> で読み込ませ、ブラウザで開き直します。フォームに「好きなサイト」を入れて追加 → ブラウザを閉じて開き直しても残っているか確認。残っていたら自分を褒めてOK。この瞬間、あなたの作品は「動くWebアプリ」になりました。

用語の正体

用語平たく言うとどう使う
addEventListener「○○が押されたら××する」を予約する命令ボタンに動作をくっつける
localStorageブラウザの小さなメモ帳データを保存して次回も使う
JSON.stringifyオブジェクト→文字列に変換localStorageに入れる前
JSON.parse文字列→オブジェクトに戻す取り出す時
innerHTML「中身のHTMLを書き換える」リストを描き直す時
もっと詳しく: localStorage ってどれくらい保存できるの?

ブラウザごとに違いますが、だいたい5MB程度が上限です。お気に入りリストのような軽いテキストなら、何千件入れても全然余裕。ただしユーザーが「ブラウザのデータを削除」すると消えます。永久保存ではないので、大事なものは後日「クラウドのデータベース」に移す、という発想が必要です（この章では触れません）。

もっと詳しく: 「イベント委譲」って何?

上のコードで、「×」ボタンそれぞれにリスナーを付けるのではなく、親の <ul> に1個だけリスナーを付けて、どれがクリックされたかを判定しています。これが「イベント委譲」。項目が増えたり減ったりする場面ではこちらの書き方が素直で、コードもスッキリします。Claudeに「イベント委譲で書いて」と頼めば自然にこうなります。

—— コードの中身が全部わからなくても大丈夫。「追加」「削除」「保存」の3ブロックがあること、それだけ感じ取れればOK。細かい文法は、作りながら少しずつ掴まっていきます。

② GitHubに上げる（20分）

できたアプリをパソコンの中から GitHub のクラウドに引っ越しさせます。これを push と言います。Ch05で触った Git の応用編ですね。

GitHub認証の整理

公式ドキュメント必読

GitHubへの認証はパスワード直書きではなく「Personal Access Token (PAT)」か「SSH鍵」を使います。間違えると作業が全部止まるので、ここだけは公式を読んでおくのが安全。GitHub認証の公式ドキュメント →

初心者には HTTPS + PAT が一番簡単。macOS なら Keychain が自動で PAT を覚えてくれます。

GitHubリポジトリを作る

GitHubリポジトリ作成の画面フロー
GitHub画面の3ステップ + ターミナルの3行。これだけで作品がクラウドに旅立ちます。

実際のコマンド（コピペでOK）

アプリのフォルダでターミナルを開いて、最初の一回だけ:

# まだGit管理していない場合だけ
git init
git add .
git commit -m "お気に入りリスト v1 完成"

# GitHub側でリポジトリを作ったら、そのURLを登録
git remote add origin https://github.com/あなたのユーザー名/my-favorites.git
git branch -M main
git push -u origin main

2回目以降、変更したら毎回:

git add .
git commit -m "どんな変更をしたか一言"
git push
今すぐ試す（5分）

push が成功したら、GitHubのリポジトリページをブラウザで開いてみましょう。index.html / style.css / app.js が並んでいるはず。自分が書いたファイルがクラウドに並んでいる瞬間、ちょっと感動します。まだURLで動作はしませんが、世界に一歩近づきました。

もっと詳しく: push が「rejected」と言われた時は?

よくあるのは「リモート側に最初のコミット（READMEなど）が既にある」パターン。git pull --rebase origin main で一度取り込んでから git push すれば解決。迷ったらそのエラー文をそのままClaudeに見せて「これどういう意味?」と聞くのが最速。Ch07で習った通り、エラーは友達です。

—— ターミナルで英語のメッセージが流れて不安になっても大丈夫。ほとんどは「うまくいった報告」です。怪しいものだけClaudeに見せれば翻訳してくれます。

③ GitHub Pagesで公開ONにする（15分）

クラウドに上がっただけでは、まだ「倉庫に置いてある」状態。ここで 公開スイッチをON にします。たった数クリックで、あなた専用のURLが発行されます。

GitHub Pages設定画面の手順
設定画面は英語ですが、押すボタンは上の4つだけ。怖くありません。

公開URLの形

URLの土台
https://あなた.github.io/
リポジトリ名
my-favorites/
完成形
https://あなた.github.io/my-favorites/
今すぐ試す（3分）

上記4ステップ完了後、Settings > Pages の上部に緑のチェックマークと共にURLが表示されます。そのURLをクリックしてみましょう。あなたが作ったお気に入りリストが、世界のどこからでも見られる状態になっています。スマホからも、友達のPCからも、地球の裏側からも。

もっと詳しく: なぜ GitHub Pages は無料なの?

GitHubは「世界のエンジニアが使うプラットフォーム」を維持するため、静的サイト（HTMLとJSだけ、サーバー側で複雑な処理をしないもの）の公開機能を無料で提供しています。あなたのリストは「静的サイト」なのでピッタリ。容量もアクセス数も、個人の作品レベルなら十分な制限があります。公式の制限事項 →

次はこんな改良ができる（発展課題）

公開できた今、もう「完成」ではなく「育てられる対象」になりました。アイデアが浮かんだらClaudeに頼むだけ。2つ、すぐ試せる改良を紹介します。

発展①: カテゴリ機能を足してみる（30-45分）

お気に入りに「仕事」「趣味」「学習」などのタグを付けられるようにする改良です。Claudeへのお願い文の例:

app.js に「カテゴリ」機能を追加してください。
- 追加フォームに「カテゴリ」のセレクトボックス(仕事/趣味/学習/その他)
- リスト表示時、カテゴリを小さいバッジで表示
- 「仕事だけ表示」等のフィルタボタン
- localStorage のデータ構造も合わせて更新
既存の追加・削除・保存は壊さないで。

やる前に Ch06 で習った Plan モードで「どう変更するか」見せてもらうと、既存コードが壊れる不安が減ります。できたら git commit -m "カテゴリ機能追加" & git push。1〜2分後にはもう公開URLに反映されています。

発展②: 見た目をダークモード対応にする（15-30分）

ブラウザの設定に合わせて自動で暗い色調にする改良。Claudeへのお願い:

style.css にダークモード対応を追加してください。
- OSのダーク設定を自動検出 (prefers-color-scheme)
- 背景黒、文字白、リンクはアクセント色を保つ
- CSSだけで実装、JS変更なし

CSSの @media (prefers-color-scheme: dark) を使う、クラシックなパターン。非エンジニアの人が作ったとは思えない見た目に一段上がります。pushすれば夜中にスマホで開いた時の体験もグッとよくなります。
