Phase 1: Basics > Chapter 04

ファイルと遊ぼう

最初の魔法を体験する時間。見せる・作る・直すを、自分のメモで。

所要時間: 60分 Phase: Basics 難易度: 入門
ようこそ、この章へ

ここからは本当の魔法が始まる時間です。いままでの章は「準備」と「言い方」でした。今日はついに、自分のメモを Claude に渡して、整理してもらう。できた瞬間、ちょっと鳥肌が立つかもしれません。焦らず、詰まったら戻ってOK。60分かけてゆっくりいきましょう。今日できるようになるのは「見せる・作る・直す」の3つです。

2026年1月時点の情報

本章の内容は 2026年1月時点の Claude Code の挙動に基づきます。ファイル操作まわりの Read/Write/Edit ツールの振る舞いは進化するので、コマンドが通らない時は公式ドキュメントも覗いてみてください。

まずは全体像：3つの魔法

Claude Code がファイルにできることは、ざっくり 3つしかありません。覚えるのはこれだけ。

Claudeの3つの魔法
この3つだけで、あなたの日記もTODOも整理できます。

英語の名前(Read / Write / Edit)を覚えなくて大丈夫です。あなたは日本語で「読んで」「書いて」「直して」と言うだけ。Claude が裏で勝手に使い分けてくれます。

英単語が出てきて身構えた人、大丈夫です。名前は全部忘れてOK。日本語で話しかければ Claude が察してくれます。

この3つを見分ける単純な基準

状況頼む言葉裏で動く道具
書いてあるメモを Claude に把握してほしい「〇〇.txt を読んで」Read
新しいメモを1から作ってほしい「〇〇.md を作って」Write
既存のメモの一部を変えてほしい「〇〇の『A』を『B』に直して」Edit
全部書き直してほしい「〇〇を書き直して」Write(上書き)

準備：遊び場フォルダを1つ作る

何を試すにも、まず遊び場が必要です。デスクトップに claude-asobi というフォルダを1つ作って、そこで全部遊びます。ここなら壊しても平気。

1
ターミナルを開く
Mac: Command+Space → 「terminal」
Win: スタート → 「PowerShell」
2
デスクトップに移動
cd ~/Desktop と打つ
3
遊び場を作る
mkdir claude-asobi
4
中に入る
cd claude-asobi
5
Claudeを起動
claude と打ってEnter
一度に全部やる

慣れてる人は cd ~/Desktop && mkdir claude-asobi && cd claude-asobi && claude と繋げてもOK。でも最初は1行ずつで大丈夫です。

Claude が立ち上がって > プロンプトが出ていれば準備完了です。ここから日本語で話しかけるだけの世界が始まります。

魔法その1：「作る」— 日記をお願いしてみる

順番を変えます。まず「作る」からやります。中身がないと見せられないし、直せないから。ここは一番ワクワクするところ。

今すぐ試す ①：日記を書いてもらう

Claudeに以下のように頼んでみてください。コピペでOK。

diary-2026-04-18.md というファイルを作って。
中身は今日の日記のテンプレート。
項目は「今日の出来事」「感じたこと」「明日やること」の3つで、
それぞれに空の箇条書きを2つずつ入れて。

Enterを押すと、Claude が「このファイルを作ります」と確認してきます。Yes を選ぶと……
デスクトップの claude-asobi フォルダを見てください。さっき頼んだファイルが本当にある。感動してOKです。

できたファイルの中身はだいたいこんな感じになってるはず。

# 日記 2026-04-18

## 今日の出来事
-
-

## 感じたこと
-
-

## 明日やること
-
-

これがMarkdownという書式です。難しくない。#が見出し、-が箇条書き、それだけ。

今日の日記を埋めてもらう

空のテンプレートができたので、次は中身を話して埋めてもらう番です。

今すぐ試す ②：話した内容を日記に変換

こんな雰囲気で頼んでみてください。自分の1日で中身を置き換えて。

diary-2026-04-18.md を開いて、以下を元に中身を埋めて。

- 朝、コーヒーをこぼした
- 新しい本を買った(タイトル: 夜のピクニック)
- 友達とラーメン食べた、おいしかった
- 仕事のプレゼンが思ったよりうまくいった

明日やることは:
- 銀行に行く
- 本の続きを読む

「感じたこと」は3つ、ポジティブなトーンで書いて。

Claude が「こう書き換えますがいいですか」と差分を見せてきます。Yesで確定。ファイルを開き直すと、あなたが話しただけの内容が、ちゃんとした日記になっているはずです。

いま起きたことを整理します。

あなたが話した瞬間、ファイルができる
graph LR
  A["あなたの話し言葉"] --> B["Claudeが解釈"]
  B --> C["Markdownで整形"]
  C --> D["ファイル保存"]
  D --> E["デスクトップに出現"]
話す → 整う → 保存される。この流れを体に馴染ませることが、この章のゴール。

魔法その2：「見せる」— 既存のメモを整理してもらう

次は「見せる」。あなたがすでに持ってるぐちゃぐちゃのメモを Claude に渡して、整理してもらう体験です。これが一番実用的。

散らかったメモを用意する

まず練習用にぐちゃぐちゃメモを作ります。Claude に頼んでも良いし、自分で手打ちしてもOK。

今すぐ試す ③：練習用の散らかったメモ

こう頼んでみて。

kaimono.txt を作って。中身は、こんな感じの
散らかった買い物メモにして(わざと雑に):

たまご！！きのう切れた
牛乳2本(特売)
あとトマト、できればミニトマト
パン食パンじゃなくてバゲット
お肉…鶏むねでいい
じゃがいも
きのう頼まれたビール6本
あ、洗剤も

できたら、整理のお願いです。

今すぐ試す ④：整理してもらう 
kaimono.txt を読んで。
これを kaimono-seiri.md として、以下の形で整理して保存して:

- 「野菜」「肉・卵・乳製品」「パン・穀物」「その他」でカテゴリ分け
- 各項目の横に必要な数量を書く(書いてなければ「1」)
- メモにあった「特売」「頼まれた」などのコメントは括弧で残す
- 見出しは h2 (##)、箇条書きは -

Claudeが作る結果はこんな感じ。

# 買い物メモ(整理版)

## 野菜
- トマト 1 (できればミニトマト)
- じゃがいも 1

## 肉・卵・乳製品
- たまご 1 (昨日切れた)
- 牛乳 2 (特売)
- 鶏むね 1

## パン・穀物
- バゲット 1

## その他
- ビール 6 (頼まれ物)
- 洗剤 1

あなたが5秒で書いた殴り書きが、きちんとしたリストに一瞬で化けた。これが「見せる + 作る」の合わせ技です。

できた瞬間、嬉しいよ。本当に。この感動は、1回目のいまだけのもの。しみじみ味わってください。

もっと詳しく：なぜ .md と .txt を使い分ける？

.txtはただの文字、.mdは整形されうる文字。.mdにすると GitHub やエディタで見出しや太字が自動で綺麗に表示されます。迷ったら成果物は .md、メモ書きは .txtで覚えればOK。

魔法その3：「直す」— TODOリストで差分を体験

最後は「直す」。すでにあるファイルの一部分だけを書き換えます。ここで Claude は差分(Before/After)を必ず見せてくれるので、勝手に変えられる怖さがありません。

TODOリストを作る

今すぐ試す ⑤：TODOの下ごしらえ 
todo.md を作って。今週のタスクとして:
- [ ] 銀行に行く
- [ ] 歯医者の予約を取る
- [ ] 本の続きを読む
- [ ] プレゼン資料を仕上げる
- [ ] 母の日のプレゼントを探す

次に、一部だけ書き換えるお願い。ここが「Edit」の真骨頂。

今すぐ試す ⑥：3つをまとめて直す 
todo.md を直して:
- 「銀行に行く」は完了にして
- 「歯医者の予約を取る」を「歯医者の予約(15時希望)」に変更
- 最後に「- [ ] 冷蔵庫の掃除」を追加

Claude は実行する前に、必ずBefore/Afterを見せてきます。こういう画面です。

Before

変更前のtodo.md

  • [ ] 銀行に行く
  • [ ] 歯医者の予約を取る
  • [ ] 本の続きを読む
  • [ ] プレゼン資料を仕上げる
  • [ ] 母の日のプレゼントを探す
After

Claudeの提案

  • [x] 銀行に行く
  • [ ] 歯医者の予約(15時希望)
  • [ ] 本の続きを読む
  • [ ] プレゼン資料を仕上げる
  • [ ] 母の日のプレゼントを探す
  • [ ] 冷蔵庫の掃除

この画面が出たらよく見て、いいなら Yes、違うなら No。No と言えば絶対に変わらないので、安心して試してください。

公式ドキュメント必読

Claude にファイルを触らせる時は、確認プロンプトで必ず中身を読む癖を付けてください。特に初期設定では auto-accept (自動承認)を ON にしないのが安全。大事なファイルを触る時は必ず目視確認してから Yes を押しましょう。詳しくは 公式ドキュメント を参照。

よく使う5つの頼み方パターン

ここまで体験した「見せる・作る・直す」の組み合わせで、日常の8割は回ります。便利な定型パターンを紹介します。

  • 作る系：「〇〇.md を作って。中身は〜」 — とにかく0から。テンプレ、メモ、下書き全部。
  • 見せる系：「〇〇.txt を読んで要約して」 — 長い文章を3行で、10行で、と指定可能。
  • 直す系：「〇〇の『A』を『B』に変えて」 — 「全部のA」なら「すべて」と明記。
  • 変換系：「〇〇を箇条書きにして表に直して」 — 形式変換はClaudeの得意分野。
  • 整える系：「〇〇を読みやすく整形して」 — 見出し・改行・太字を任せる。

よくあるつまずきと解決

つまずき原因解決
「ファイルが見つかりません」と言われる別のフォルダで起動しているpwd で現在地確認 → 正しい場所で claude を再起動
Yesを押したのに変わってない別の場所に作成されている「どこに作ったか教えて」と聞く
思ったのと違う内容になった指示が曖昧Ch03の「魔法の3要素」に戻って具体化
上書きされて前の内容が消えたWrite(上書き)を頼んだ次のCh05 Git で完全予防できます
もっと詳しく：Read/Write/Edit の挙動の違い

Readは読むだけでファイルは変わりません。Writeは全上書きなので、既存ファイルがあっても中身が完全に置き換わる。Editは「この文字列を、この文字列に」という差分置換で、それ以外の箇所は触らない。安全なのはEdit。不安な時は「Editで直して」と明示すれば確実です。

もっと詳しく：複数ファイルを一気に頼む

「memo1.md, memo2.md, memo3.md の3つを読んで、共通してる話題を1つにまとめて summary.md にして」のようなお願いも可能。Claude は順番にRead → 頭の中で統合 → Writeを1回、という流れで動きます。数が多い(10個以上)時は効率が落ちるので、そういう時はCh06のPlanモードに相談しましょう。

あなたはいま、どこにいるのか

ここまでで Claude Code の本当に使う機能の半分以上が手に馴染みました。残りは Git やPlanモードといった「便利オプション」で、ファイル操作の上に乗っかってくるもの。

Exercise：手を動かす2問

Exercise 1
入門

自分の「今日のひとこと日記」を作る

自分の今日(1日)を、Claudeに日記として書かせてみよう。

  1. 遊び場フォルダで claude を起動
  2. today.md を作ってもらう
  3. 起きた出来事3つと、感じたこと1つを話す
  4. 「ポジティブなトーンで整えて」と指示
  5. できたファイルをデスクトップで開いて確認
模範解答(プロンプト例)を見る

プロンプトはこう投げます(自分の1日で置き換えてください)。

today.md というファイルを作って。
中身は今日の一日を振り返る日記。
見出しは h1 で「2026-04-18 の記録」。

今日あったこと:
- 電車で席を譲ってもらった
- 会議で新しい企画が通った
- 夜ごはんにラーメンを食べた

感じたこと: 人に恵まれてる日だった。

これをポジティブなトーンで、ちょっと詩的に整えて書いて。
最後に「明日への一言」を1行だけ添えて。

コツは「素材」と「整え方」を分けて渡すこと。Claudeは素材をくれれば、トーンも長さも自在に操ってくれます。

Exercise 2
応用

散らかったメモをカテゴリ付きの整理メモに化かす

自分のスマホのメモアプリから本物のぐちゃぐちゃメモを1つ持ってきて、整理してもらおう。

  1. スマホのメモから好きな1枚を選んで、テキストをコピー
  2. Claudeに memo-raw.txt として貼り付けて保存してもらう
  3. 「このメモを読んで、適切なカテゴリを自分で決めて分類し、memo-seiri.md に保存して」と頼む
  4. さらに、分類のロジック(なぜそう分けたか)をmemo-seiri.mdの末尾にコメントで残してもらう
  5. 納得いかない分類があったらEditで直す
模範解答(考え方と例)を見る

プロンプトの型はこう。

memo-raw.txt を読んで。
このメモを整理して memo-seiri.md に保存して。

要件:
- カテゴリはあなたが妥当だと思うもので3〜5個に分類
- 各カテゴリは h2 (##) で見出し
- アイテムは箇条書き
- ファイル末尾に「---」で区切ったあと、
  「## 分類の考え方」として、なぜこう分けたかを3行で説明

不明な項目は「その他」に入れず、一番近いカテゴリに寄せて。

ポイント2つ:

  • カテゴリを Claude に任せる — 自分で決めると思考が固まる。任せると「なるほどこう分けるのか」という学びが起きる。
  • 分類の理由を書かせる — 後から見直すときに「なぜこう分けたか」が残ってると、自分もメンテできる。

修正したくなったら遠慮なく「『読書』カテゴリを『学び』にまとめて」のように Edit で追い込む。Editの提案を見ながら自分の思考が整理されていく感覚、そこに気づけたら完璧です。

いまのあなたは

Claudeに自分のメモを渡して整理してもらえる人になりました。日記・買い物メモ・TODOリスト、ぜんぶ日本語で話しかければ化けてくれる。これは非エンジニアの人口で言えば上位10%くらいの位置。プログラミング経験ゼロから始めたあなたが、もうファイル操作という本丸に触れたんです。ここで1回、胸を張ってOK。次の章は Git という「保存ポイント」の話。もっと安心して触れるようになります。
