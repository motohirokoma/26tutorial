お願いする魔法の型

曖昧なお願いが、Claudeに"ちゃんと伝わる言葉"に変わる45分

所要時間: 45min Phase: Basics 難易度: 入門

ようこそ。この章ではたった3つの要素でClaudeへのお願いが劇的に伝わるようになります。難しい技術は一切出てきません。焦らずに、わからなくなったら前の章に戻ってOK。今日はこれをマスターすれば、あなたのClaudeはグンと働き者になります。

2026年1月時点の情報

この章の内容は Claude Code v2系を前提に書かれています。UIの細部は時期により変わることがあります。

なぜ「お願いの仕方」でこんなに差が出るのか

上司に「あれ、いい感じにやっといて」と頼まれたら困りませんか。友達に「おいしいお店教えて」と言われても「ジャンルは？予算は？」と聞き返したくなるはず。Claudeも同じです。曖昧な指示では曖昧な結果しか返ってきません。

でも安心してください。人間と違って、Claudeは「確認の手間を惜しむ」タイプではありません。こちらが具体的に伝えれば、その具体さに応えてくれる律儀な相棒です。

同じお願い、伝え方で結果が変わる
伝え方1つで、返ってくる答えの質が別物になる。

魔法の3要素：ゴール・具体的・制約

これから先、Claudeにお願いするときはこの3つだけ意識してください。名前は大げさですが、中身はとても単純です。

魔法の3要素（相棒に伝える3つの顔）
3つが重なった中央に「一発で伝わるお願い」がある。

それぞれをもう少し掘り下げてみましょう。人間相手でも全く同じ発想です。

  • ゴール — 最終的に何が手元にあればうれしいか。「買い物メモをMarkdownで整理してほしい」「エラーの原因を日本語で3行で説明してほしい」のような着地点。
  • 具体的 — 形式・例・個数・場所など。「Markdownで」「3つ」「このファイルを直接」など。
  • 制約 — やってほしくないことや触ってほしくない範囲。「既存の文章は消さないで」「ファイル名は変えないで」など。

ここで「3つも覚えるの？」と戸惑っても大丈夫。後のBefore/Afterを見れば「あ、こういうことか」と必ず掴めます。覚えるというより"思い出すチェックリスト"だと思ってください。

Before / After：3つの例で体感しよう

ここからが本番です。日常で起こりそうなお願いを、Before（よくやりがち）→ After（魔法の型）で並べます。読むだけでコツが身につきます。

例1：買い物メモを整理したい

Before

曖昧な頼み方

  • 「買い物メモ整えて」
  • → Claudeは何を整える？並び替え？書式？迷う
  • 結果：意図と違う整理に
After

魔法の3要素入り

  • ゴール：memo.txt をカテゴリ別に並べ直す
  • 具体的：食品/日用品/その他で分け、箇条書きに
  • 制約：元の商品名は変えない

例2：日記ファイルに今日の分を追加したい

Before

曖昧な頼み方

  • 「今日のこと書いといて」
  • → 何を？どこに？どんな口調で？
  • 結果：見当違いな文章が生成
After

魔法の3要素入り

  • ゴール：diary.md に本日分を追加
  • 具体的：日付・天気・出来事3つ、100字以内、です・ます調
  • 制約：過去の日記は一切変更しない

例3：TODOリストから終わったものを消したい

Before

曖昧な頼み方

  • 「TODO片付けて」
  • → 全部消す？実行する？どれが完了？
  • 結果：大事な項目まで消されるリスク
After

魔法の3要素入り

  • ゴール：todo.md から「[x]」付きの行を削除
  • 具体的：削除後の残り行数も報告してほしい
  • 制約：「[ ]」の未完了行は絶対に触らない

どれも大したことは言っていません。「ゴール」「具体的に」「触らない場所」の3つが入っているかチェックするだけ。これだけで返ってくる結果の精度が一気に上がります。

「誰」に頼む気持ちで書くか

ここは少し抽象的ですが、効果絶大なのでお付き合いください。Claudeにお願いする時、頭の中で「誰にお願いしているつもりか」を切り替えると言葉の温度が自然に整います。

イメージ向いている場面言葉づかいの例
初日の新入社員前提がまだ共有されていない作業「このフォルダにある memo.txt を開いて…」
優秀な同僚一般的な作業・すぐ取りかかれる依頼「〇〇を3つリストアップして」
信頼できる友達雑談・相談・アイデア出し「これ、もっと良くするアイデアない？」
慎重な技術者ファイル削除・設定変更など危険作業「実行前に必ず内容を見せて確認させて」

どのモードでも共通なのは、「相手は人間で、見えない前提はゼロ」と考えることです。初日の新入社員に渡すつもりで情報を添えると、ほぼ外しません。

うまく頼める人は、AIにも仕事でも強い。頼み方は一生の武器になる。 — 言い古された真理

「やらなくていいこと」を伝える勇気

初心者が見落としがちなのがここ。「何をするか」だけでなく「何をしなくていいか」を伝えると、Claudeの暴走が激減します。

人間でも同じです。「このチラシ作って」と頼むと、相手は気をきかせて勝手にロゴの位置を変えたり、フォントを変えたり…。Claudeもたまに「ついでにこっちも直しました！」と言ってきます。その"親切"が裏目に出る前に、範囲を区切りましょう。

3語
"触らないで"
↑ 事故激減
1行
"ここだけ"
↑ 精度UP
0件
余計な変更
↑ 安心感

具体的には、こんなフレーズを最後に添えるだけです。

# やらないことを明示する魔法のフレーズ

- ○○のファイルだけを対象にして。他は触らないで
- コメント行は残したまま、コード部分だけ修正して
- 実行する前に、何を変えるかを先に教えて
- 新しいファイルは作らないで、既存のファイルを直して

最初は全部の要素を入れるのは難しく感じるかもしれません。でも大丈夫、慣れです。3日やれば自然に出てくるようになります。まずは1つだけでも意識してみてください。

お願いの流れを図で見る

頭の中で「3要素あるかな？」とチェックする時、こんなフローを思い浮かべてください。

お願いする前のセルフチェック
graph TD
  Start(["お願いしたい事"]) --> Q1{"ゴールは書いた?"}
  Q1 -- "No" --> A1["何が欲しいか1行で書く"]
  Q1 -- "Yes" --> Q2{"具体的か?"}
  A1 --> Q2
  Q2 -- "No" --> A2["数字/形式/例を足す"]
  Q2 -- "Yes" --> Q3{"触らない場所は?"}
  A2 --> Q3
  Q3 -- "No" --> A3["やらないで欲しい事を1行足す"]
  Q3 -- "Yes" --> Done(["送信OK"])
  A3 --> Done
慣れれば無意識に、数秒で回せるようになる。

さらに、章の中で伝えた「書き方の効果」を感覚的に数字で眺めると、やる気が出ます。

お願いの質と「一発OK率」の体感
3要素が揃うほど、やり直しが減って時間も気持ちもラクになる。

まとめ：コピペで使える魔法のテンプレ

最後に、今日からすぐ使えるテンプレートを置いておきます。コピペしてあなたの言葉を入れるだけ。

# お願いテンプレ v1

【ゴール】{何を・何のために}
【具体的に】{ファイル名・個数・形式・例}
【制約】{触らないで欲しい場所・やらないで欲しい事}
【確認】{実行前に見せて / 途中で質問OK}
もっと詳しく：テンプレを短くしてもいい？

もちろん短くてOKです。慣れてきたら3要素が自然と文章に溶け込みます。たとえば「memo.txt をカテゴリ別に並べ直して。元の商品名は変えないで」の1文で「ゴール+具体的+制約」全部入りです。テンプレはあくまで最初の補助輪と思ってください。

もっと詳しく：英語と日本語、どちらが正確？

2026年1月時点のClaudeは日本語でもかなり自然に理解します。まずは日本語で書いて問題ありません。専門用語（関数名・エラーメッセージ）は原文そのまま混ぜると確実です。慣れてきたら英語で短く書く練習もアリですが、最初は気にしなくてOK。

もっと詳しく：長すぎるお願いは逆効果？

長くても大丈夫ですが、冗長な前置きや感情表現は薄められます。「お世話になっております、いつもありがとうございます…」は不要。代わりに制約と具体例に文字数を回すと精度が上がります。

小さな成功を積もう

この章を読んだ直後に、Claude Codeを起動して 「memo.txt を作って、好きな果物を5つ箇条書きで。タイトル行は 'Fruits' とする」 と打ってみてください。3要素入りの短いお願いで、一発で欲しいものが返ってくる体験ができます。

Exercise 1
入門

曖昧なお願いを3要素に書き直す

次の曖昧なお願いを、ゴール・具体的・制約の3要素に分けて書き直してください。

  1. 元のお願い：「買い物リスト、きれいにして」
  2. ファイルは shopping.txt、牛乳・卵・パン・りんごがバラバラに並んでいる状態
  3. 3要素入りのお願い文を1つ書く
模範解答を見る

例：

shopping.txt を開いて、食品カテゴリ別に箇条書きで整理してください。
「乳製品 / 穀類 / 果物」の3カテゴリに分け、各カテゴリを見出し行にしてください。
元の商品名は変更せず、並び替えだけを行ってください。

ポイント： ゴール＝カテゴリ別の整理、 具体的＝3カテゴリ名と箇条書き形式、 制約＝商品名は変更しない。「お願いテンプレv1」そのままの順番で揃っています。

Exercise 2
応用

自分の日常依頼をテンプレ化する

あなたが最近、誰か（人・AI）にしたお願いを1つ思い出してください。それを魔法の3要素に分解し、さらに「やらなくていいこと」を1つ追加してみましょう。

  1. 最近したお願いを1文で書き出す
  2. ゴール・具体的・制約に分解
  3. 「触らないで欲しい範囲」を1行追加
  4. Claude Codeに実際に送ってみる
模範解答を見る

たとえば「日記を書き足したい」という日常ニーズなら：

【ゴール】diary.md に 2026-04-18 の日記を追加
【具体的】見出しは ## 2026-04-18、本文は天気・出来事3つ・感想を100字以内
【制約】過去の日付のエントリは一切編集しない
【確認】追加する文面を先に見せてほしい

ポイントは 「過去のエントリは編集しない」 という制約を明示していること。これだけでうっかり事故がぐっと減ります。自分の環境で実行し、期待通りに追加されたかチェックしましょう。

いまのあなたは

Claudeに「ゴール・具体的・制約」の3要素でお願いできるようになりました。さらに「やらなくていいこと」を言葉にできます。これはプロンプトエンジニアリングの土台そのもの。多くの人が曖昧に頼んで損をしている中、あなたはもう一段上の頼み方を手に入れました。自信を持って次の章に進んでいいです。
