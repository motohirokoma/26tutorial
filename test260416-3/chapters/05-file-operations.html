ファイル操作

読む・書く・直すをClaudeに任せる。Read / Write / Edit ツールの仕組みと、安全に使うための作法。

所要時間: 1h Phase: Core 難易度: 中級

3つのツール: Read / Write / Edit

Claude Code がファイルを操作する道具は、シンプルに 3つだけ。

ツール役割承認が必要?
Readファイルを読む不要 (読むだけなので安全)
Writeファイルを新規作成または上書き必要
Edit既存ファイルの一部を書き換える必要 (差分が表示される)
3ツールの使い分け
flowchart TD
  Q{"何をしたい?"} --> R["読みたい"]
  Q --> N["新しく作りたい"]
  Q --> E["既存ファイルの一部を直したい"]
  R --> Read["Read ツール"]
  N --> Write["Write ツール"]
  E --> Edit["Edit ツール"]
  Read --> Done["承認不要、即実行"]
  Write --> Approve1["差分表示 → 承認"]
  Edit --> Approve2["差分表示 → 承認"]
Claudeが自動で適切なツールを選びます。あなたは「やりたいこと」を日本語で言うだけ。

diff の読み方

Claudeが変更を提案するとき、ターミナルに diff が色付きで表示されます。これを読めるようになると、安心感が違います。

diff の見た目
赤(-) が削除行、緑(+) が追加行。両者を見比べれば「何がどう変わるか」が一目瞭然。

ファイル指定の3パターン

Claude にどのファイルを操作してほしいかを伝える方法は3つ。明確に指定するほど精度が上がります。

  • 絶対パス — 「~/Desktop/practice/index.html を読んで」一番確実。
  • 相対パス — 「./src/main.js の中身を確認して」現在のディレクトリ基準。
  • あいまい指定 — 「ToDoアプリのHTMLを直して」Claudeが自分でファイルを探す。
あいまい指定の罠

同名ファイルが複数あったり、想定外のファイルを編集してしまうリスクがあります。重要な変更ほど パスを明示 しましょう。

安全なファイル操作の作法

推奨

慎重派ワークフロー

  • 作業フォルダを Git 管理にしておく
  • 大きな変更前に git commit
  • diff を1回ずつ目視確認
  • 小さく頼んで小さく承認
注意

暴走を招くやり方

  • 「全部一気にリファクタして」
  • diff を見ずに即承認
  • Git未管理のフォルダで作業
  • 大事な書類があるフォルダで起動

典型タスクと頼み方

やりたいこと依頼例
ファイルの中身を見る「config.json の中身を見せて」
新しいファイルを作る「todo.html を作って、ToDoリストUIを書いて」
一部だけ書き換える「style.css の --bg を白に変更して」
全置換「全ファイルで「foo」を「bar」に置換して」
ファイルを消す「old.html を削除して」
名前を変える「tmp.txt を memo.txt にリネーム」

ファイル操作の進化タイムライン

Step 1
手動でコピペ (旧来)
ChatGPTが書いたコードを自分でファイルに貼る。手間と転記ミスが多い。
Step 2
Copilot で次の行を補完
エディタの中で1行ずつヒントが出る。運転は自分。
Step 3
Claude Code が直接編集 (現在地)
日本語で頼むだけで、ファイルが自動で更新される。あなたはレビュアー。
もっと詳しく: .gitignore や .claudeignore って?

.gitignore は Git に追跡させたくないファイルのリスト(機密ファイル、ビルド成果物など)。Claude Code はこのファイルを尊重するので、.env のような秘密ファイルは原則読み込みません。明示的に「.env を読んで」と頼まない限り安全。さらに .claudeignore を置けば、Claude専用の除外リストも設定できます。

Exercise 1
入門

3つのファイルを連続で操作する

Read → Write → Edit の流れを体験します。

  1. 練習用フォルダで claude を起動
  2. 「hello.txt を作って、中身に『Hello, Claude!』と書いて」(Write)
  3. 「hello.txt の中身を見せて」(Read)
  4. 「hello.txt の『Claude』を『World』に変更して」(Edit)
  5. もう一度Readで結果確認
模範解答を見る

3ツールの違いを体感する練習です。Editでは差分(diff)が表示され、「Claude → World」の変更が緑+赤で示されます。

# 期待する遷移
hello.txt = "Hello, Claude!"
↓ Edit「Claude → World」
hello.txt = "Hello, World!"

Edit はファイル全体を書き換えるのではなく 該当箇所だけをピンポイントで置換 するので、長いファイルでも安全です。

Exercise 2
応用

HTMLファイルを読み込んでスタイルを変える

「読んでから直す」の典型ワークフローを体験します。

  1. Ch02のExerciseで作った README.md があるフォルダで claude 起動
  2. 「profile.html を作って、自分の名前と趣味を書いた1ページサイトに。背景白・文字黒のシンプル版」
  3. ブラウザで開いて確認
  4. 「profile.html の背景を薄いグレーに、見出しだけ青色にして」
  5. diff を見て承認 → ブラウザを再読込で確認
模範解答を見る

2回目の依頼でClaudeはまず profile.html を Read し、必要箇所だけ Edit します。差分は style タグの数行のみ。これがエージェント型の効率の良さです。

# 想定される Edit の差分
- body { background: white; color: black; }
+ body { background: #f5f5f5; color: black; }

- h1 { color: black; }
+ h1 { color: #2563eb; }

「読んでから直す」を体感したら、もうあなたは Claude Code をファイル操作の道具として使いこなせています。
