最終演習

これまでの全部を統合して、自分の課題を1つ Claude と一緒に解決する。Short Edition の卒業課題。

所要時間: 30min Phase: Practice 難易度: 統合
あなたが10時間で学んだことを、あなた自身の1つの課題で 試す時間です。完璧を目指さない。動けば勝ち。 — 卒業課題のスピリット

問題提示

あなたの日常 or 業務にある「ちょっと面倒な作業」を1つ選び、Claude Code で30分以内に自動化または改善するツールを1つ作って GitHub Pages に公開する。

ルール

① 完璧でなくていい(動けば合格)
② 既存のWebサイト・アプリ・スクリプト、何でもOK
③ 30分の予算で「動くもの」をリリース
④ 公開URLとコードはGitHubに残す

テーマのアイデア

「何を作ろう」と迷う人向けの、30分で作れるアイデア集。1つ選んでも、組み合わせてもOK。

家計メモアプリ

金額+カテゴリだけのシンプル家計簿。月間集計付き。

名刺リンク集

1ページのリンクツリー(LinkTree風)を自作。SNS / 仕事 / ブログを整理。

ポモドーロタイマー

25分 + 5分のサイクルを自動で進めるタイマー。通知音付き。

勉強記録

科目+時間を記録、今週の累計を表示するシンプルログアプリ。

レシピ検索

自分が知ってるレシピをタグで分類して検索できるカード集。

アイデアメモ

思いついたら即書き込めるメモ帳。タグ + 日付 + 検索付き。

30分の使い方

時間配分テンプレート

段階的ヒント

ヒント1: 何を作るか決まらない

「自分が今週、3回以上やった『手動の繰り返し作業』」をリストアップ。一番面倒なものを選ぶ。例: 「家計の入力」「タスク忘れ防止」「学習進捗の管理」。テーマがあれば6割勝ったも同然。

ヒント2: 機能を絞る

30分なので、機能は最大3つに絞る。「入力 / 表示 / 保存」だけでもアプリは成立する。Ch09のToDoアプリのような構造をベースにすると、応用が楽。

ヒント3: Plan を1分かける

いきなり「実装して」と頼まず、必ず「方針を3案出して」 → 「決まったら手順を書いて」 → 「OKそれで実装して」の順。Ch04で学んだPlan→Design→Buildフロー。

ヒント4: 詰まったら

「動かない」なら、エラー全文をClaudeに貼って原因と修正を聞く(Ch07)。「設計が悪そう」なら、/clearして別アプローチを最初から提案させる。絶対に1人で抱え込まない。

合否の基準

合格

これがあれば卒業

  • 公開URLにアクセスすると動く
  • 「自分が解きたかった問題」が部分的にでも解けている
  • GitHubにコードが残っている
不合格

こっちじゃダメ

  • 動かない / 公開できなかった
  • 本に載ってる例題そのまま (自分の課題感ゼロ)
  • 30分のプランがそもそも壮大すぎ

模範解答 (例: ポモドーロタイマー)

30分でこんな感じに作れる(クリックで展開)

例として「ポモドーロタイマー」を選んだ場合の依頼&完成までを再現。

# Plan (3分)
> ~/Desktop/pomodoro でポモドーロタイマーを作りたい。
  まず方針3案出して(機能の優先順位ベースで)。

# Design (2分)
> 案A (シンプル: 25分タイマー + 通知音 + 履歴) で進めて。
  index.html 1ファイルで実装する手順を書いて。

# Build (15分)
> OKその手順で実装して。

# テスト (5分): ブラウザで実際に動作確認
> タイマーが3秒で終わる「テストモード」を追加して、デバッグ後に消して

# 調整 (3分)
> モバイルで通知音が鳴らない、Web Audio APIに切替えて

# 公開 (2分)
> このフォルダを GitHub の "pomodoro" リポジトリに push、
  GitHub Pages 有効化して URL 教えて

これだけで30分以内に動くポモドーロタイマーが世界に公開できます。これが 「Claude Code を使える人の生産性」。

応用課題

30分でできた人向けの追加チャレンジ。

  • ダークモード対応 — Ch08の手法を移植 (5分)
  • 統計画面 — 過去のデータをグラフ表示 (Chart.js使用、15分)
  • 共有リンク — URLパラメータで状態を共有できるように (10分)
  • 第2版を別レポジトリで — 同じテーマで設計をやり直す (30分)
