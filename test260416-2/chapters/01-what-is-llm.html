LLMとは何か

AI / 機械学習 / ディープラーニング / 生成AI / LLM。似ているけれど違うこれらの言葉を、一枚の地図に整理します。記事で「LLMが〜」と書かれたとき、自分がいまどこの話をしているかが分かるようになるのがゴール。

所要時間: 30分 Phase: Basics 難易度: 入門

1. まず「地図」を描く

「AI」「機械学習」「ディープラーニング」「生成AI」「LLM」。ニュースで毎日見る言葉ですが、ほとんどの人は 包含関係を取り違えています。最初にやるべきは、「どれがどれの中に入っているのか」を絵で押さえることです。

用語のロシアンドール構造
AI ⊃ 機械学習 ⊃ ディープラーニング ⊃ 生成AI ⊃ LLM。LLMは生成AIの「テキスト担当」。
  • AI — 「機械が知的に振る舞う」全領域。ルールベース(if文の塊)も、学習型も両方含む。
  • 機械学習 — データから自動でパターンを学ぶ手法。AIの一部だが「if文の塊」は含まない。
  • ディープラーニング — 多層のニューラルネットを使う機械学習。画像・音声・言語で爆発的成功。
  • 生成AI — 新しいコンテンツ(テキスト・画像・音声・動画)を 作る AI。分類や予測ではなく 生成 が目的。
  • LLM — 「テキスト生成に特化した、巨大なディープラーニングモデル」。ChatGPT・Claude・Gemini・Llama がこれ。

3. 何ができて、何ができないのか(現時点)

仕組みを学ぶ前に、まず「現在地」を押さえます。LLM が 得意 なことと 苦手 なことを比較カードで:

得意

パターンの読み取り・生成

  • 文章の要約・翻訳・書き換え
  • コード生成・レビュー
  • 「それっぽい」回答の即時提示
  • 大量の文章からの抽出・整理
  • ロールプレイ・対話
苦手

正確性・最新性・推論

  • 事実確認(ハルシネーション)
  • 学習後に起きた出来事
  • 大きな数値の正確な計算
  • 長い論理の厳密な追跡
  • 「分からない」と認めること

この「得意 / 苦手」の境界が なぜそうなるのか を、Ch02〜Ch06 で仕組みから理解していきます。

4. 5世代の歴史を一気に見る

「なぜ今、急に LLM が騒がれるのか?」は、過去20年の流れを 1本のタイムライン で見れば一目です。

2003 - 2012
統計的機械学習の時代
手作業で特徴量を設計。SVM・ロジスティック回帰など。文脈の長い文章は苦手。
2013
Word2Vec — 単語に「意味のある数字」
単語を数百次元のベクトルに埋め込む手法が登場。Ch02 で詳しく扱う。
2017
"Attention Is All You Need" — Transformer 登場
Google が発表した1本の論文。今のすべての LLM はこの上に立つ。Ch04 で扱う。
2020
GPT-3 — 1750億パラメータの衝撃
「大きくすれば質が変わる」(Scaling Law)が証明される。
2022
ChatGPT 公開 — 一般人がLLMに触れる
RLHF でチャット用に整えられた GPT-3.5 が公開。2ヶ月で1億ユーザ。
2023 - 現在
マルチモーダル / エージェント時代
画像・音声・動画も扱う Gemini / GPT-4o / Claude。LLM が 道具を使う エージェントへ。Ch06 で扱う。
もっと詳しく: なぜ「大きさ」がそんなに効くのか?

2020年に OpenAI が発表した論文 "Scaling Laws for Neural Language Models" では、モデルサイズ・データ量・計算量と性能の間に 滑らかなべき乗則 があることが示されました。簡単に言えば「10倍大きくすれば性能はこれくらい上がる」という予測可能なカーブが見つかったのです。

さらに驚きだったのは 創発(Emergence)。あるサイズを超えると、それまでできなかったタスク(例: 算数の繰り上がり、推論、コード)が 急に できるようになる現象が観察されました。これが「とにかく大きくすれば賢くなる」の根拠であり、現在のスケール競争の出発点です。詳しくは Ch05 と Ch06 で。

用語整理: クローズド vs オープンウェイト

クローズド(GPT-4, Claude, Gemini)= モデル本体は非公開、API経由で使う。オープンウェイト(Llama, Mistral, DeepSeek)= モデルファイルが公開され、自分のPC・サーバで動かせる。「オープンウェイト」と「オープンソース」は厳密には別物です。

Exercise 1
入門

「AI / 機械学習 / DL / 生成AI / LLM」を自分の言葉で書く

5つの用語について、それぞれ 1行 で「これは何か」を、誰かに口頭で説明するつもりで書き出してください。包含関係(どれがどれに含まれるか)も書く。

  1. 白紙の紙(またはメモアプリ)を用意
  2. 5つの用語を縦に書く
  3. 右側に1行の定義を書く
  4. 下に「⊃」記号で包含関係を書く(例: AI ⊃ 機械学習 ⊃ ...)
模範解答を見る
  • AI — 機械が知的に振る舞う領域全部(if文ベースも含む)
  • 機械学習 — データから自動でパターンを学ぶ手法
  • ディープラーニング — 多層ニューラルネットを使う機械学習
  • 生成AI — 新しいコンテンツを「作る」AI(分類ではなく生成)
  • LLM — テキスト生成に特化した巨大なディープラーニングモデル

包含関係: AI ⊃ 機械学習 ⊃ ディープラーニング ⊃ 生成AI ⊃ LLM

このシンプルな関係を 口で説明できる なら、ニュース記事の「生成AI規制」「LLM倫理」「機械学習の偏り」がそれぞれ どのレベルの議論 なのかが瞬時に分かるようになります。

Exercise 2
応用

身近なAIサービスを5つ「分類」する

あなたが過去1ヶ月で触ったAI的サービス(ChatGPT、Google翻訳、Netflixのレコメンド、スマホの顔認識、Siri など)を5つ挙げ、それぞれが「LLM か、生成AIだがLLMでないか、ディープラーニングだが生成AIでないか、機械学習だがDLでないか、AIだが機械学習ですらないか」を判定してみてください。

  1. サービスを5つ書き出す
  2. それぞれが何の入力 → 何の出力かを書く
  3. 上記5レベルのどこに属するかを判定
  4. 判断に迷うものはその理由をメモ
模範解答を見る

判定例(諸説あり、目的は「考えること」):

  • ChatGPT(対話) → テキスト→テキスト → LLM
  • DALL·E(画像生成) → テキスト→画像 → 生成AI(LLMではない)
  • スマホ顔認識 → 画像→人物識別 → ディープラーニング(生成ではない)
  • Netflixレコメンド → 視聴履歴→推薦 → 機械学習(DLではない場合が多い)
  • 古いSiriの定型応答 → ルールベース → AI(機械学習ではない)

判定で迷ったものこそが学習の宝物です。「Google翻訳って今はLLM?」のような疑問は、実は 歴史的な変遷 を反映しています(昔は統計翻訳→今はTransformer系)。これがニュースの読み方を一段引き上げます。
