Transformer 〜注意機構という発明〜

2017年に発表された1本の論文「Attention Is All You Need」。今のAIブームはすべてこの論文から始まりました。この章では、Transformerの核心 — 「Attention(注意機構)」 — を図と日常的な例えで理解します。数式は一切使いません。

所要時間: 45分 Phase: Deep Dive 難易度: 中級

1. Transformer 以前の世界 — RNN の限界

2017年以前、文章を扱うAIは RNN や LSTM という構造が主流でした。これらは 「単語を1つずつ順番に読む」 方式で動きます。人間が左から右に読むのに近い。

しかしこの方式には致命的な弱点がありました:

RNN/LSTMの弱点

1単語ずつしか処理できない

  • 並列計算ができない → 学習が遅い
  • 長文だと最初の方を忘れる
  • 文末から文頭への参照が苦手
  • GPUの恩恵を受けにくい
Transformerの解決

全単語を同時に処理

  • 並列計算可能 → GPU使い倒せる
  • 「Attention」で任意の単語間を直結
  • 長文でも遠い単語の関係を保持
  • 大規模化とスケーリング法則の道を開いた

2. Attention の核心メタファ: 「文の中での会話」

Attentionを一言で説明するメタファ: 「文中の単語が、互いに『あなたのこと、どれくらい気にすべき?』を聞き合う仕組み」。

例えば文「銀行に行ったら、川が氾濫していた」を読むとき、「銀行」 という単語の意味を確定するには、後ろの「川」「氾濫」を見る必要があります(英語の "bank" が「銀行」か「川岸」かが決まる)。Attentionはこの 双方向参照 を1ステップで可能にします。

「銀行に行ったら、川が氾濫していた」のAttention重みイメージ
「銀行」が「川」を強く見ている。これで「金融機関の銀行」じゃなく「川岸の bank」だと判別できる。

3. Q, K, V — 検索エンジンのメタファ

Attentionの仕組みは 図書館の検索 に例えるとつかめます。各トークンは3つの「役割」を持ちます:

Query (Q) — 質問
「私は◯◯を探してる」
Key (K) — 札
「私は△△の本です」
Value (V) — 中身
「私の内容はこれ」

各トークンは Q を出して全トークンの K と照合 し、似ているもの(内積が大きいもの)の V を多めに混ぜて 自分の新しい表現を作ります。これが Self-Attention(自己注意)。

入力ベクトル列が、3つの行列(Q,K,V)に分裂し、互いに参照し合って混ざり、新しいベクトル列が出力される。

4. Multi-Head Attention — 複数の視点で見る

Self-Attentionを 並列に複数本 走らせる。これが Multi-Head Attention。

1つのヘッドが「主語と動詞の対応」を見るとき、別のヘッドは「指示語の参照先」、また別のヘッドは「同じトピックの単語のクラスタ」を見る、というように 分業 します。GPT-4 では概ね 96 ヘッド、Claude も同程度。

Multi-Head の役割分担イメージ
複数の視点から同時に文を見ることで、豊かな関係性を捕まえる。

5. Transformerブロック全体像

Transformer 1ブロック = Multi-Head Attention + Feed Forward(各トークンを個別に処理する全結合層)+ 残差接続(入力をそのまま足す)+ 正規化。これを 何十段 も積みます。

Transformerブロック1段の構造
graph TD
  X["入力ベクトル列"] --> A1["LayerNorm"]
  A1 --> A2["Multi-Head Attention"]
  A2 --> R1(("+"))
  X --> R1
  R1 --> F1["LayerNorm"]
  F1 --> F2["Feed Forward
(各トークン個別)"]
  F2 --> R2(("+"))
  R1 --> R2
  R2 --> Y["出力ベクトル列
(次のブロックへ)"]
"+" は残差接続。深く積んでも勾配が消えにくくする工夫。

6. なぜ計算量がコンテキスト長の2乗なのか

Self-Attentionで各トークンが 他の全トークン を見ます。トークンが N 個あれば、ペアの数は N × N = N²。コンテキスト長が2倍になれば計算量は4倍、10倍になれば100倍。

計算量はコンテキスト長の2乗で増加

これが「長文を扱うLLMが高い」と「200K以上のコンテキストはまだ高価」の根本理由。Flash Attention や Sparse Attention など、この2乗を緩和する研究がホットトピック。

もっと詳しく: なぜ Position Encoding が必要なのか

Self-Attentionは「全トークンを並列に見る」ので、順序の情報がない という弱点があります。「猫が犬を追う」も「犬が猫を追う」も、Attentionだけだと区別できない。

これを補うのが Positional Encoding(位置エンコーディング)。各トークンの埋め込みに「これは1番目」「これは2番目」を表すベクトルを足してから入力します。最近は RoPE(Rotary Position Embedding)が主流で、長文外挿性能が良いとされています。

7. Decoder-only / Encoder-only / Encoder-Decoder

Transformer家系には3種類あります。用途で使い分けられます:

Encoder-only

BERT 系

  • 双方向に読む
  • 分類・抽出が得意
  • 生成はしない
  • 例: 検索, NER, 感情分析
Decoder-only

GPT / Claude / Llama

  • 左から右に生成
  • 後ろを見ない (Causal Mask)
  • 対話・生成に最適
  • 現代LLMの主流
Encoder-Decoder

T5, 翻訳系

  • 入力を encode → 出力を decode
  • 翻訳・要約に伝統的
  • 最近はDecoder-onlyに押されぎみ
  • 例: T5, BART

8. まとめ

  • Attention — 各トークンが他の全トークンに「注目」できる仕組み
  • Q, K, V — 質問・札・中身。検索エンジンのメタファで考える
  • Multi-Head — 複数の視点を並列で。文法・参照・話題などを分業
  • 多段に積む — 12〜126段。深くするほど抽象度が上がる
  • 計算量はN² — 長文ほど高くつく。Long-context技術が要る理由
Exercise 1
入門

「文脈で意味が変わる」例を10個集める

Attention の存在意義は「文脈で単語の意味が変わる」を捕まえること。同形異義語・多義語を10個挙げて、各単語が「何を見て」意味を確定しているかを書き出します。

  1. 同じ単語で意味が複数ある語を10個リストアップ
  2. 各単語について、2つの異なる文を作る
  3. 各文で、その単語が文中の どの語 を見ることで意味が決まるかを矢印で書く
模範解答を見る

例:

  • はし — 「川にはしを架ける」(→「川」を見る = 橋) / 「はしで食べる」(→「食べる」を見る = 箸)
  • くも — 「空に黒いくも」(→「空」 = 雲) / 「くもの巣を払う」(→「巣」 = 蜘蛛)
  • あつい — 「お茶があつい」(→「お茶」 = 熱い) / 「友情があつい」(→「友情」 = 厚い)
  • bank — "go to the bank for money" (→ money = 銀行) / "river bank" (→ river = 川岸)
  • apple — "I bought an apple" (→ bought = 果物) / "Apple released iPhone" (→ released = 会社)

これがまさにAttentionが解く問題。Word2Vec などの 静的 埋め込みは「あつい」に常に同じベクトルを割り当てます。Transformerは 文脈に応じてベクトルを書き換える ので、同じ「あつい」でも文ごとに違う意味を持てます。

Exercise 2
応用

BertViz で実際のAttention重みを可視化する

論より証拠。実物のAttention重みをブラウザで眺めて、「Attention がどんな関係性を捕まえているか」を見る。

  1. BertViz のColabリンクを開く(無料・ブラウザだけでOK)
  2. 用意されたデモ文(英語)で、各ヘッドのAttentionパターンを観察
  3. 自分の文を入れて(英語のほうが分かりやすい)、特徴的なヘッドを3つ見つける
  4. (余裕があれば) 「The animal didn't cross the road because it was too tired.」で "it" が何を指すか、Attentionが正しく "animal" を強く見ているか確認
模範解答を見る

典型的な観察結果:

  • あるヘッドは「直前の単語」だけを強く見る → 局所的文脈担当
  • 別のヘッドは「主語」を文末まで強く見る → 主語追跡担当
  • 別のヘッドは「同じ品詞」をクラスタ化する → 構文担当
  • "it" は実際に "animal" を強く参照する → 指示語解消担当

これを目で見ると、「Attention が何かを学んでいる」ことが具体的に納得できます。BERT/GPTの内部解釈研究(Mechanistic Interpretability)はここから始まる広大な領域です。

応用ヒント: Anthropic が公開している "Attention Heads in Pythia" や "Toy Models of Superposition" を読むと、Attention ヘッドの内部解釈の最先端が分かります。
