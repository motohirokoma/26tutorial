LLMの能力・限界・未来

仕組みを知った今、ふたたび「LLMは何ができて、何ができないのか」に戻ります。創発能力・ハルシネーション・バイアス・アラインメント・マルチモーダル・エージェント。ニュースで頻繁に出るこれらの用語を、この章で一気に結びつけます。

所要時間: 30分 Phase: Deep Dive 難易度: 中級

1. 創発能力(Emergent Abilities)

創発能力とは、「あるサイズを超えると、急にできるようになる能力」のこと。Scaling Lawの予測する 滑らかな改善 とは別に、特定タスクで 段差 のような性能ジャンプが観察されました。

典型例: 算数の繰り上がり、複数段階の論理推論、ペルシャ語のことわざ理解。10Bパラメータでは0%、100Bパラメータでは突然70%、のような曲線。

創発能力の典型曲線(模式)
論争: 本当に「創発」なのか?

2023年、「Are Emergent Abilities of LLMs a Mirage?」という論文が物議。「評価尺度が 離散的 だから段差に見えるだけで、連続尺度で測れば滑らかなんじゃないか」という反論。これがまだ決着していないのが現状です。

2. ハルシネーション — 自信満々の嘘

ハルシネーション(Hallucination)はLLMの最大の弱点。Ch03で見たように、LLMは「次のトークンの確率」を出すだけで、事実検証する仕組みを内蔵していない。

典型的なハルシネーションのパターン

パターン具体例原因
架空の論文を引用「Smith et al. 2021 で示されています」論文タイトルの統計的それっぽさ
架空のAPI関数numpy.linspace_safe() を使う命名パターンからの「ありそう」生成
事実の混合「夏目漱石の代表作 走れメロス」関連語の混線
判例・URL捏造架空の判例番号を引用形式に沿った穴埋め生成
計算ミス「12,345 × 678 = 8,365,910」(誤)確率分布での計算は近似

対策: RAG(検索拡張生成)

RAG(Retrieval-Augmented Generation)は、ハルシネーション対策の現実的な解。

RAG の仕組み
graph LR
  Q["ユーザの質問"] --> R["検索
(自社DB / Web)"]
  R --> D["関連文書
(参考資料)"]
  Q --> P["プロンプト組み立て"]
  D --> P
  P --> L["LLM"]
  L --> A["回答
+ 引用元"]
「内部知識だけで答える」のをやめて、外部データを毎回読ませて答える。引用元も付けやすい。

3. バイアスと安全性

学習データはインターネット。インターネットは 人類社会の偏りそのまま。だからLLMは社会の偏りを そのまま、ときに強化して 出力します。

  • 性別バイアス — 「医者」を男性、「看護師」を女性と仮定する傾向
  • 文化バイアス — 学習データの大半が英語 + 西洋中心、非西洋文化の情報が薄い
  • 政治的バイアス — RLHFラベラーの傾向(米国の高学歴若年層)が反映されやすい
  • 世代・古さの偏り — 学習締切日(knowledge cutoff)以降の情報を知らない

これらは RLHFやSystem Promptで緩和 はできるが、根絶は難しい。LLMを使う側が「そういうもの」と理解しておく必要があります。

4. ベンチマーク — どう「賢さ」を測るのか

LLMの「賢さ」は 標準化されたベンチマーク で比較されます。主要なものを覚えておくと、ニュースの「ベンチマーク◯◯で1位」が読めます。

主要モデルのベンチマーク比較(模式)

5. マルチモーダル(Multimodal)

テキストだけでなく 画像・音声・動画 も扱えるLLM。GPT-4o, Claude 3.5 Sonnet, Gemini 2.0 が代表。仕組みは「画像/音声を トークン化 してテキストと同じシーケンスに乗せる」。Ch02のトークン化が「言語以外」にも拡張された形です。

6. エージェント(Agent)

LLMが 道具を使い、行動し、自律的にタスクを完遂する 形態。これが2024-2026年のホットトピック。

エージェントのループ(ReAct パターン)
graph LR
  G["目標
(ユーザ)"] --> T["思考
(Thought)"]
  T --> A["行動
(Tool使用)"]
  A --> O["観察
(結果)"]
  O --> T2{"終了?"}
  T2 -- No --> T
  T2 -- Yes --> R["最終回答"]
LLMが「考える→道具で行動する→結果を見る」を自律的に繰り返す。Claude Code, ChatGPT Operator などがこの系統。

「道具」の典型例

  • Web検索 — 最新情報の取得、ハルシネーション抑制
  • コード実行 — Python/JSを動かして計算、データ分析
  • ファイル読み書き — ローカルファイルの操作(Claude Code, Cursor等)
  • ブラウザ操作 — クリック・入力 (Computer Use, Operator)
  • APIコール — 任意の外部サービスを呼ぶ (MCP, Function Calling)
2026年時点の主要エージェント

Claude Code(Anthropic, ターミナル内エージェント)/Cursor / Windsurf(IDE内エージェント)/Devin(Cognition, 自律コーディング)/ChatGPT Operator(OpenAI, ブラウザ操作)/Manus(中国発、汎用)。それぞれ得意な道具と環境が違う。

8. まとめ

LLMは「創発」によって急に賢くなるが、「ハルシネーション」「バイアス」を持つ。これを RAG・アラインメント・エージェント 等の周辺技術で補い、より マルチモーダル で 自律的 な存在になっていく。 — この章のひとことまとめ

あなたは今、ニュースの主要キーワードをほぼ網羅しました。最終章 (Ch07) で、これらを使って 架空の技術ニュース記事 を読み解いてみましょう。

Exercise 1
入門

ハルシネーションを意図的に起こさせる

LLMの限界を 体感 するために、ハルシネーションを観察します。

  1. ChatGPT または Claude を開く
  2. 「『深層変分注意機構』(Deep Variational Attention Mechanism)という手法について、提案論文と主要な貢献を教えて」と聞く ※架空の用語
  3. 応答に「もっともらしい論文タイトルや著者名」が含まれていないか確認
  4. 含まれていたら、「Google Scholar で実在を確認して」と続けて聞く
模範解答を見る

典型的な結果:

  • 古いモデル/思考なしモデルだと「Smith et al. (2021)」のような 架空の引用 を生成しがち
  • 新しいモデル(Claude 3.5+, GPT-4o, Gemini 2.0+)は「その用語は私が知る限り存在しません」と 不確実性を表明 する傾向が強くなった
  • RAG/検索付きのモード(Perplexity, ChatGPT検索, Claude with web)は、本当は存在しない ことを発見できる

これがアラインメントの進歩。「分からないと言える」モデルが増えてきた。ただし完全ではないので、重要な事実は必ず一次ソースで検証 する習慣が必要です。

Exercise 2
応用

「同じ質問」を3つの異なるモードで比べる

事前学習だけ vs RAG vs エージェント、で同じ質問への答えがどう変わるかを観察。

  1. 質問を1つ用意。例: 「2025年のノーベル物理学賞受賞者は誰?」
  2. モード1: 通常のChatGPT/Claude(検索OFF)で聞く → 学習締切以降ならハルシネートしがち
  3. モード2: Perplexity または ChatGPT (検索ON) で聞く → RAGで実際に検索した結果が返る
  4. モード3: Claude Code または Devin で「2025のノーベル物理学賞をWeb検索して、信頼できるソース3つから引用してまとめて」を実行
  5. 3つの応答の 正確性・引用源・不確実性の表明 を比較
模範解答を見る

典型的な結果:

  • モード1: 学習締切前なら正解、それ以後だと「私の知識は2024年までです」と返るか、ハルシネートする
  • モード2: ほぼ確実に正答。引用URLも付く。但し「検索結果が間違っていれば一緒に間違える」
  • モード3: 複数ソース照合があるので最も信頼性が高い。だが 時間がかかる(エージェントループの分)

このトレードオフ表が、実プロジェクトでLLMを設計する時の判断軸になります:

モード速度正確性コスト用途
素のLLM速い不安定安い創作・要約・書き換え
RAG中高い中FAQ, 社内文書検索
エージェント遅い最高高い調査・自動化・複雑タスク
