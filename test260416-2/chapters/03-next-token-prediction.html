次単語予測 〜LLMの意外とシンプルな原理〜

LLMが長文を書いたり質問に答えたりする姿は複雑ですが、原理は驚くほどシンプル。「次に来るトークンを予測する」を繰り返しているだけです。この章で、その仕組みと、なぜLLMは毎回違う答えを返すのか(Temperature / Top-p)を理解します。

所要時間: 30分 Phase: Basics 難易度: 入門

1. LLMがやっている、たった一つのこと

LLMの中身は、突き詰めれば 1つの関数 です。

(これまでのトークン列) → (次のトークンの確率分布) — LLMの本質はこの1行に集約される

例えば「今日の天気は」と入れたら、辞書(語彙)の中の数万トークンそれぞれに 確率 がつきます:

「今日の天気は」の次に来るトークンの確率(イメージ)

LLMは 「晴れ」 を選ぶ確率が一番高いと判断しました。これを選んだら、次は「今日の天気は晴れ」を入力して、また次のトークンを予測。これを えんえんと繰り返す。

自己回帰(Autoregressive)生成の流れ
graph LR
  A["今日の天気は"] --> B["LLM"] --> C["晴れ"]
  C --> D["今日の天気は晴れ"] --> E["LLM"] --> F["です"]
  F --> G["今日の天気は晴れです"] --> H["LLM"] --> I["。"]
  I --> J["..."]
出力したトークンを次の入力に追加して、また予測。これをEOS(終了)トークンか上限まで繰り返す。
用語: 自己回帰(Autoregressive)

「自己回帰」とは、自分の出した結果を自分の入力に追加していく方式のこと。GPT系LLMは全部これ。だから生成中にストリーミング(文字が1個ずつ出てくる演出)ができる。

2. 「どう選ぶか」でキャラが決まる: サンプリング戦略

LLMは確率分布を出すまで。そこからどう1つ選ぶか は別の話です。これがLLMの「個性」を決める部分。

戦略A: いつも一番高い確率を取る(Greedy / Argmax)

毎回 1位 のトークンを取る。決まりきった答えになるが、創造性ゼロ。同じ入力に対して必ず同じ出力。コードレビューや事実回答にはこれが向く。

戦略B: 確率に従ってサイコロを振る(Sampling)

確率分布に比例してランダムに選ぶ。「晴れ」42%, 「曇り」27% なら、42:27 の比でサイコロを振る感じ。同じ入力でも毎回違う答えが出る。これがChatGPTで「何度か聞き直すと違う答えが出る」理由です。

Temperature(温度)

Temperature は確率分布の 鋭さ を調整するつまみ。

Temperature が確率分布をどう変えるか
  • Temperature 0(冷たい) — Greedy と同じ。常に1位を選ぶ。決定的。
  • Temperature 0.7(中間) — ChatGPTのデフォルト付近。自然な多様性。
  • Temperature 1.5+(熱い) — レアなトークンも選ばれる。創造的だが破綻も増える。

Top-k / Top-p(候補の絞り込み)

確率の低い「変なトークン」を最初から除外する仕組み:

パラメータ仕組み例
Top-k確率上位 k 個の中だけからサンプリングk=40 → 上位40候補だけ
Top-p(Nucleus)累積確率が p になるまでの候補だけp=0.9 → 累積90%に入る候補だけ
使い分けの目安

事実ベース・コード生成 → Temperature 0〜0.3。創作・ブレスト → Temperature 0.8〜1.2 + Top-p 0.95。極端に1.5を超えると意味不明な単語が混じってきます。

3. 「次の1トークンだけ」からなぜ賢く見えるのか

「えっ、次の1トークン予測してるだけで、なんで複雑な質問に答えられるの?」という疑問が当然出ます。答えは 3つの仕組みの合わせ技 です:

1
圧倒的データ
人類の文章を学習
2
巨大モデル
数百億のパラメータ
3
文脈の読み取り
Transformer (Ch04)

「次の1トークン」を予測するには、その時点までの全文脈を理解 しなければなりません。「フランスの首都は」の次に「パリ」と答えるには、地理を知っている必要がある。「だが、結局その仕組みは間違っているのか?」の次に「いいえ」と答えるには、否定の論理を理解している必要がある。

つまり 「次の1トークンを精密に予測できる関数」=「世界をかなり理解している関数」 という構図。これがLLMの本質的なアイデアです。

もっと詳しく: ハルシネーションの根本原因

LLMは「次のトークンの確率」を出すだけで、「事実かどうか」を判断する仕組みを内蔵していません。「アインシュタインは」の次に「ノーベル物理学賞を1921年に」が来やすい確率分布を持っているなら、それを生成する。本人がそれを言ったかどうかは、単に学習データに頻出していたかどうか でしか分からない。

これが ハルシネーション(自信満々の嘘)の根本原因。「事実検証は別レイヤーが必要」というのが、RAG(検索拡張生成)などの技術が要る理由です。Ch06 で扱います。

4. ストリーミング表示の正体

ChatGPTで答えが 1文字ずつタイピングされる ように出てくる、あのアニメーション。あれは 演出ではなく実際の動き です。LLMは1トークン生成するたびに、それをすぐにユーザに送っているのです。

1トークン = 1ステップ。ユーザに即送る
sequenceDiagram
  participant U as ユーザー
  participant S as サーバー
  participant L as LLM
  U->>S: 質問
  S->>L: 推論開始
  L-->>S: トークン1 「は」
  S-->>U: 「は」
  L-->>S: トークン2 「い」
  S-->>U: 「い」
  L-->>S: トークン3 「、」
  S-->>U: 「、」
  L-->>S: ...
  L-->>S: EOS (終了)
  S-->>U: 完了
体感速度を上げるための重要な工夫。「待たされてる感」が劇的に減る。

Exercise 1
入門

「同じ入力で違う答え」を実験する

あなたが普段使うLLMで、Temperatureの効果を体感します。

  1. ChatGPT または Claude を開く
  2. 「『朝のコーヒーは』に続く文を1行で書いて」と頼む
  3. 新しいチャットを開いて、まったく同じプロンプトをもう一度
  4. これを5回繰り返して、答えがどれくらい変わるか見る
模範解答を見る

5回試すと、たぶん毎回違う答えが返ります(完全同じはまず無い)。例:

  • 「朝のコーヒーは、私の1日のスイッチを入れる魔法だ。」
  • 「朝のコーヒーは深い香りで眠気を吹き飛ばしてくれる。」
  • 「朝のコーヒーは欠かせない儀式のようなものだ。」

これがサンプリングの効果。同じ確率分布から異なるトークン列が選ばれている。Temperature を 0 に設定できるAPIなら、毎回完全に同じ答えが出るはずです。

ポイント: LLMが「考えて違う答えを出している」のではなく、確率分布からサイコロを振っている から違う答えになる、という事実を体感してください。

Exercise 2
応用

Temperature を意図的にずらして遊ぶ

OpenAI Playground または Claude Workbench で、Temperatureを変えて違いを観察します。

  1. OpenAI Playground または Claude Workbench を開く
  2. 同じプロンプト「短編小説の最初の一文を書いて」を3回入力 — Temperature 0, 0.7, 1.5 で
  3. 3つの結果を比較して、どう違うか観察
  4. (余裕があれば) Top-p も 0.5, 0.9, 1.0 で試す
模範解答を見る

典型的な結果のパターン:

  • Temp 0: 「窓の外には朝霧が立ち込めていた。」のような 無難で凡庸 な始まり
  • Temp 0.7: 「彼女が郵便受けを開けると、見知らぬ筆跡の手紙が一通だけ入っていた。」のような 適度に意外性のある 始まり
  • Temp 1.5: 「鯨が空から落ちてきた朝、僕は8時に目覚ましをセットしていた。」のような ぶっ飛んだ 始まり(時には文法も怪しくなる)

「創造性」と「安定性」のトレードオフが、たった1つの数字でこんなに変わることを体感してください。

応用: プロダクト開発で「同じ入力に同じ出力が必要」なら Temp=0、「毎回違う提案が欲しい」なら Temp=0.8 のように、用途で使い分けるのがプロの基本です。
