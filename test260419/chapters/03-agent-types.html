Agentタイプの使い分け

Explore / general-purpose / 専門Agent それぞれの適材適所

所要時間: 30分 Phase: Basics 難易度: 初級
この章のゴール

Claude Codeで Agent ツールを呼ぶとき、subagent_type に何を指定すべきか——その判断を「なんとなく」ではなく5秒で決めきれる状態を作ります。3つの主要タイプ(Explore / general-purpose / 専門型)の得意・苦手、ツール権限の違い、選定フローを押さえ、最後に同じ目的を2タイプで走らせて挙動の違いを肌で感じるハンズオンまでやります。本章の記述は2026年1月時点の情報です。

3つの主要Agentタイプ

Ch02で見た通り、Agent ツールには必ず subagent_type を指定します。Claude Codeに2026年1月時点で主に使える3タイプは、大きく次の3系統に分類できます。

3タイプの関係 — 役割と権限で住み分ける
左に行くほど「読むだけで安全」、右に行くほど「できることが広い」。
  • Explore — 読み取り専門のリサーチ専用Agent。コードベース調査、仕様の収集、ドキュメントの横断読みに最適。編集しないので安全。
  • general-purpose — 万能型のデフォルトAgent。読み込みも編集もコマンド実行もできる。1人のミニClaude Codeとして動く。
  • 専門型 — タスク別に調整された専用Agent。Plan(計画立案)、code-reviewer(コードレビュー)、claude-code-guide(Claude Code使用法)など。
ざっくり覚え方

「読むだけ=Explore / 作らせる=general-purpose / 専用タスクあり=専門型」。この3択で95%のケースは即決まります。

タイプ別の得意・苦手とツール権限

「何ができて・何ができないか」は許可されるツールセットで決まります。ここが各タイプの性格を作っています。

観点Exploregeneral-purpose専門型 (例: code-reviewer)

コマンド実行 (Bash)×◎△

重要: Exploreは編集できない

Exploreはファイル編集系のツール(Edit/Write)を持っていないため、どんなに詳細な指示を出しても、コードは1行も書き換えません。「調査のついでに直して」と頼んでも、結果は調査レポートだけ戻ります。書き換えが要るならgeneral-purposeを選ぶ——これが一番多い取り違えです。

もっと詳しく: 「編集不可」が効く場面

Exploreの「編集できない」は一見制約ですが、並列調査で威力を発揮します。5つのExploreを同時に走らせても、互いのファイルを壊し合う心配がゼロ。worktreeを用意する必要もなく、読み取りだけなので安全にリサーチを並列化できます。

一方general-purposeを5並列で走らせると、同じファイルに書き込み合うレースコンディションのリスクがあり、isolation="worktree" 等で隔離する設計が必要になります(Ch04で扱う)。

タスク別の適合度

6つの典型タスクで3タイプがどれくらい向いているかを見ます。相対的な強みを感覚でつかむのが目的。

タイプ×タスクの適合度レーダー
  • コードベース調査: Explore◎ / general-purpose○ — 読むだけならExploreの方が安価で速い。
  • ファイル編集: general-purpose◎ / Explore× — Exploreは物理的に書けない。
  • コードレビュー: code-reviewer◎ / general-purpose△ — 専門型は観点がブレない。
  • 計画立案: Plan◎ / general-purpose○ — Planは段取り化のプロンプト込。
  • Webリサーチ: Explore◎ / general-purpose○ — WebFetchは両方使えるが、Exploreの方が軽い。
  • コマンド実行・検証: general-purpose◎ / Explore× — Bash権限がExploreには無い。

判断が2秒以上かかりそうなら、general-purposeを選んでOK。万能型なので失敗はしません。ただし「明らかに読むだけ」「明らかにレビュー」なら、専用タイプを選ぶほうがプロンプト短縮・コスト減・精度UPの三重メリットが得られます。

もっと詳しく: 「専門型」の中身は何が違うのか

専門型は2つの要素で調整されています:(1) システムプロンプト にタスク固有の指示が埋め込み済み(例: code-reviewerなら「セキュリティ/パフォーマンス/可読性の観点で見る」が既に入っている)、(2) ツール権限 がタスクに最適化(Planならファイル編集なし、計画だけ返す)。このため呼び出し側の prompt は短くても動き、出力の品質も安定します。

自分用の専門型が欲しい場合、~/.claude/agents/ にマークダウンで定義するユーザー定義Agentの機能がありますが、本カリキュラムではビルトインの3タイプに集中します(作成は応用編)。

典型ユースケース対応表

現場でよく出るタスクを挙げて、それぞれどのタイプが正解か。このページをブックマークしておくだけでも判断速度が上がります。

やりたいこと正解タイプ理由
「認証関連のコードがどこに書いてあるか全部探して」Explore読むだけで済む、編集不要
「このPRのセキュリティ観点を見て」code-reviewer観点が定まった専門型あり
「リファクタ計画を立てて」Plan計画立案に最適化済

「Next.js 15の新機能を調べてまとめて」ExploreWebFetch+要約、編集なし

「Claude Codeのhooksの書き方を教えて」claude-code-guide専門的知識の取り出し
「5つのライブラリを並列比較して」Explore × 5並列編集不要・並列しやすい

「ログファイルから異常を探して」Explore読むだけ、GrepとRead

ハンズオン — Explore vs general-purpose

頭で分かっても体感しないと定着しません。ここで同じ目的を2タイプに投げて、出力の違いを見ます。お手元のClaude Codeで実際に試してください。

今すぐ試す

任意のGitリポジトリ(自分のプロジェクトでOK)を開いたClaude Codeで、下の2つを同じメッセージに入れて並列実行します。プロンプトはわざと同一にして、タイプの違いだけ見ます。

Explore版

読み取り専門で調査だけ

Agent(
  subagent_type="Explore",
  description="認証コード調査",
  prompt="このリポジトリで認証に関連するファイルと関数をすべて列挙し、役割を50語以内で要約してください。"
)
general-purpose版

万能型で同じプロンプト

Agent(
  subagent_type="general-purpose",
  description="認証コード調査",
  prompt="このリポジトリで認証に関連するファイルと関数をすべて列挙し、役割を50語以内で要約してください。"
)

観察ポイント

  • 実行時間 — Exploreの方が速く返るはず(ツールが少なく選択が速い)。
  • 出力スタイル — Exploreは調査レポート形式に寄り、general-purposeは自由度が高く文体が揺れがち。
  • 副作用 — Exploreはどう頼んでも1行も変更しない。general-purposeはプロンプト次第で余計な編集が入ることも。
  • コスト — Exploreの方がコストが安くなる傾向。並列で5本走らせるなら差がさらに開く。
2タイプ同時実行のイメージ
sequenceDiagram
  participant U as あなた
  participant M as メインClaude
  participant E as Explore (sub)
  participant G as general-purpose (sub)
  U->>M: 「認証コード調査してほしい」
  M->>E: Agent呼出(同一プロンプト)
  M->>G: Agent呼出(同一プロンプト)
  E-->>M: ファイル一覧+要約(短い)
  G-->>M: ファイル一覧+要約(詳しめ、余計な提案含む)

メインClaudeが同じプロンプトで2タイプを同時呼出。結果を並べて比較できる。
もっと詳しく: 結果をいちばん学び深く見る方法

両方の結果を得たら、次の3点に注目してノートに書き出してください:
(1) どちらがより簡潔だったか、(2) どちらがより網羅的だったか、(3) 余計な改善提案や編集を始めそうになったのはどちらか。この3問を言語化すると、自分のタスクに対する「タイプ選びの直感」が加速度的に育ちます。

さらに気力があれば、同じプロンプトをPlanにも投げてみてください。計画立案系タイプに「調査」を頼むと観点のズレが体感できます。適材適所の肌感覚はこういうミスマッチ実験で育ちます。

ここまで来れば、タイプ選びは「なんとなく」ではなく意志をもって決められる状態です。Ch04ではこの知識を土台に、複数Agentを並列で回すときの設計を学びます。

Exercise 1
入門

10個のタスクを正しいタイプに割り当てる

次の10タスクを、自分の頭で subagent_type に分類してみましょう。答えを出してから模範解答で答え合わせ。

  1. 「プロジェクトの依存ライブラリ一覧を抽出して」
  2. 「package.jsonのscripts欄に新しいlintコマンドを追加して」
  3. 「この関数のセキュリティリスクをレビューして」
  4. 「今週やるべき機能追加の計画を作って」
  5. 「Reactの新しいルーティング方式を調べて」
  6. 「テストを書いて、失敗したら修正して」
  7. 「config.tomlの古いキーを全部grepして」
  8. 「Claude Codeのhooks機能について教えて」
  9. 「未使用のexportを探して削除するPRを作って」
  10. 「ログから404が増えた時間帯を見つけて」
模範解答を見る
  1. Explore — 読むだけ、編集不要
  2. general-purpose — ファイル編集が必須
  3. code-reviewer — レビュー専門型
  4. Plan — 計画立案専門型
  5. Explore — Web+コード横断で読むだけ
  6. general-purpose — 編集+Bash実行の複合
  7. Explore — Grepは読み取り
  8. claude-code-guide — Claude Code固有の知識
  9. general-purpose — 削除+PR作成で編集あり
  10. Explore — ログGrepと要約

分け方のコツ:「書き換えるか?」を最初に問い、書き換え無しならExplore一択。書き換えありで専門領域(レビュー/計画/Claude Code使い方)なら専門型、そうでなければgeneral-purpose。

Exercise 2
応用

自分の環境でExplore vs general-purpose を走らせ、違いを言語化する

本章のハンズオンを実際に自分のプロジェクトで実行し、観察結果を記録します。

  1. 任意のGitリポジトリを開いてClaude Codeを起動
  2. 本章の「今すぐ試す」2つのAgent呼び出しを同じメッセージに並べて実行
  3. 戻ってきた結果を比較し、次の4点を自分の言葉でメモ:
    (a) 応答の長さ / (b) 出力フォーマット / (c) 余計な提案の有無 / (d) 処理時間(体感でOK)
  4. 「次に同じ調査をするなら、自分はどちらを選ぶか」を結論として書く
模範解答を見る

典型的な観察例(筆者の環境, 2026年1月):

  • (a) 応答の長さ — Explore約300語、general-purpose約500語。Exploreは要約に絞る傾向
  • (b) 出力フォーマット — Exploreは「ファイル名: 役割」の箇条書きで整然。general-purposeは散文+時折リファクタ提案が混じる
  • (c) 余計な提案 — general-purposeで「この関数はリファクタ候補です」等の未依頼コメントが2件。Exploreは依頼通りだけ
  • (d) 処理時間 — Exploreが約3割短い(Bash/Editの検討が不要のため)

結論例: 単なる調査ならExplore一択。編集や検証まで含むならgeneral-purpose。「依頼の粒度に応じてタイプを切り替える」と、コストも結果の質も両取りできる。

※ 同じ体験を3回別のタスクでやると、選定フローチャートを見なくても手が動くようになります。Ch04の並列実行設計でさらに活きてきます。
