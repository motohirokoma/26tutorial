Carbon

Carbon is IBM's open-source enterprise design system. It exists to encode the IBM Design Language as code, with rigorous design tokens, a 2x grid, and accessibility baked in. Carbon prefers sharp corners (0px radius), a tight 8px spacing grid, and information density.

Open

Apache 2.0 — used by IBM products and the wider open-source community.

Token-driven

Two tiers — core color tokens + role-based component tokens — power White / Gray-10 / Gray-90 / Gray-100 themes.

Enterprise dense

Productive type ramp, sharp 0px corners, 16px grid for high-information UIs.

Color documented Official ↗

Carbon's swatches step in 10s from 10 to 100 (10 highest tint, 100 darkest). Blue 60 is the primary interactive color; Gray 100 is the deepest UI surface.

Surfaces documented Official ↗

Carbon prefers Tiles to Cards. Variants: Default, Clickable, Selectable, Expandable. All sharp corners.

Default tile

A static tile groups related content with a subtle border on Layer 02.

Clickable tile

Links the entire surface; entire tile is the target.

Selectable tile

Acts as a custom radio. Selected state shown by left rule + interactive border.

Layout documented Official ↗

Carbon's 2x grid: 16 columns max, 16px gutter, 8px (mobile) - 32px (XL) page margins. Spacing tokens follow a 2x scale (2/4/8/12/16/24/32/40/48/64).
